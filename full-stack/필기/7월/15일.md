## 핵심요약

**04_MACHINE_LEARNING/Supervised_Learning/Multiclass_Classfication**에서는 Credit Score 데이터(`train.csv` 10만행×28열 / `test.csv` 5만행×27열)로 고객 신용등급을 `Poor`/`Standard`/`Good` 3개 클래스로 분류하는 다중분류 파이프라인을 실습했다(`ex/01.ipynb`). 14일 완성한 이진분류(`Binary_Classfication`)와 동일한 7단계 스켈레톤(문제정의→가져오기→EDA→전처리→분할·학습→검증→내보내기)을 재사용하되 타겟만 2클래스에서 3클래스로 확장한 버전으로, **05단계(라벨 매핑·원핫인코딩)까지만 작성되어 있고 06(모델 학습·검증지표)·07(모델/제출파일 내보내기)은 아직 빈 셀**이다. 또한 노트북의 `execution_count`를 직접 확인해보니 셀이 위에서 아래로 순서대로 실행된 게 아니라 뒤섞여 있어서(1→44 사이 뒤죽박죽), 화면에 찍힌 `cat_cols`/`num_cols` 결과가 빈 리스트(`[]`)로 나와 있다는 걸 발견했다 — Restart & Run All로 재검증이 필요한 부분. `ex/01-Copy1.ipynb`는 다중분류와 무관하게 이진분류(Adult Census) 노트북을 그대로 복사해둔 흔적임도 확인했다.

## 04_MACHINE_LEARNING — Supervised_Learning · 다중분류 (`Multiclass_Classfication/ex/01.ipynb`, 미완성)

| 단계 | 코드 | 핵심 |
|---|---|---|
| 1. 문제정의 | 타겟 `Credit_Score` | `Poor`/`Standard`/`Good` 3개 클래스 — 이진분류의 `income`(`<=50K`/`>50K`)과 달리 다중분류 |
| 2. IMPORT | `pd.read_csv('train.csv')`/`test.csv'` | `Num_of_Delayed_Payment` 등 26번째 컬럼에서 `DtypeWarning`(혼합 타입) 발생 |
| 3. EDA | `head()`, `Credit_Score.unique()`, `describe()`, `info()`, `isnull().sum()` | `Age`에 `-500`, `Occupation`에 `_______` 같은 오염된 문자열이 그대로 들어있음(전처리 전 반드시 확인해야 할 이상치) |
| 4. 결측치 처리 | `cat_cols`/`num_cols`를 `dtype`+`isnull().any()` 조건의 리스트컴프리헨션으로 뽑아 train 중앙값/최빈값으로 `fillna` | **주의**: 저장된 노트북은 셀 실행 순서(`execution_count`)가 뒤섞여 있어 화면상 `cat_cols`/`num_cols` 결과가 빈 리스트로 찍혀 있음 — 새로 위에서 아래로 실행하면 `Name`/`Type_of_Loan`/`Monthly_Balance` 등 실제 결측 컬럼이 잡히는지 재검증 필요 |
| 5. 라벨 인코딩 + 인코딩 | `y = train['Credit_Score'].map({'Poor':0,'Standard':1,'Good':2})`; `train.drop(columns=['ID','Customer_ID','Credit_Score'])`; `pd.concat([train,test])` 후 `pd.get_dummies` → `X`/`X_test` 분리 | 이진분류와 동일하게 train+test를 합쳐서 원핫 인코딩(더미 컬럼 어긋남 방지) |
| 6. 학습·검증 | *(미작성)* | `train_test_split(..., stratify=y)` → `RandomForestClassifier` → `accuracy_score`/`classification_report` 순서로 이어질 예정(다중분류라 `roc_auc_score`는 `multi_class='ovr'` 옵션 또는 클래스별 확률 필요) |
| 7. 내보내기 | *(미작성)* | 모델 저장(`joblib`) 및 `test_id` 기준 제출 파일(`submission.csv`) 생성 필요 |

> `ex/01-Copy1.ipynb`는 다중분류와 무관하게 `Binary_Classfication/ex/01.ipynb`(Adult Census 이진분류) 내용을 그대로 복사해둔 흔적 — 이 폴더의 실질적인 다중분류 작업은 `01.ipynb` 하나.

## 15일 핵심 정리

---

### 1) 이진분류 → 다중분류, 바뀌는 건 타겟과 6~7단계뿐

```
공통 스켈레톤 : 문제정의 → IMPORT → EDA → 전처리 → 라벨/인코딩 → (학습·검증) → (내보내기)
바뀌는 지점   : y = map({'Poor':0,'Standard':1,'Good':2})  (클래스 3개)
             → roc_auc_score는 그대로 못 씀, multi_class='ovr' 등 옵션 필요
```

### 2) 노트북 `execution_count`로 실행 순서 검증하기

```
셀 옆 [숫자]가 위→아래로 증가하지 않으면 "재실행 순서가 뒤섞였다"는 신호
  → 화면에 찍힌 출력이 현재 코드와 다른 상태를 반영할 수 있음(신뢰 불가)
확인법 : ipynb의 execution_count 필드를 셀 순서대로 나열해 오름차순인지 체크
대응법 : Restart & Run All로 처음부터 다시 실행해 출력을 재검증
```

### 3) cat_cols/num_cols가 빈 리스트로 나온 이유(오늘 발견한 함정)

```
cat_cols = [c for c in train.columns if (train[c].dtype=='object' and train[c].isnull().any())]
→ Name/Type_of_Loan/Monthly_Balance 등 결측 있는 object 컬럼이 실제로 존재하는데도 []로 출력됨
→ 원인 추정: 이 셀이 fillna 이후, 혹은 다른 상태의 데이터프레임을 대상으로 늦게 재실행됨
→ 교훈: fillna 결과("남은 결측치: 0 0")를 곧이곧대로 믿지 말고,
  cat_cols/num_cols가 실제로 채워지는지 먼저 확인한 뒤 전처리 셀을 신뢰할 것
```

### 4) 미완성 구간 이어서 할 일

```
06 : train_test_split(stratify=y) → RandomForestClassifier(n_estimators=100, random_state=0)
     → predict/predict_proba → accuracy_score, classification_report (다중분류 지표 확인)
07 : test_id 보관 여부 확인 후 predict_proba 또는 predict 결과로 submission.csv 생성
```
