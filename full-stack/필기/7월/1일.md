# 7월 1일 필기 — React + Spring Boot 연동 (파라미터 전달 · 유효성 검사 · 예외 처리 · JPA CRUD)

## 핵심요약

오늘은 React와 Spring Boot가 어떻게 HTTP로 통신하는지를 단계별로 학습했다. **01_PARAM**에서 axios로 쿼리 파라미터·경로 파라미터·RequestBody를 전송하는 세 가지 방식을 실습했고, **02_VALIDATION**에서 Spring Boot의 Bean Validation이 오류를 감지하면 React가 에러 메시지를 받아 화면에 표시하는 흐름을 익혔다. **03_EXCEPTION**에서는 서버에서 예외가 발생하면 `@ExceptionHandler`가 에러 응답을 반환하고 React의 `.catch()` 블록이 이를 받아 에러 페이지로 이동하는 패턴을 다뤘다. **04_JPA**에서는 React 폼 → axios POST → Controller → Service → Repository → DB 저장까지 이어지는 전체 CRUD 레이어를 연결했다. 모든 단원에서 CORS 설정(`@CrossOrigin`)이 React(3000) ↔ Spring Boot(8080) 간 통신의 전제 조건이 된다.

---

## React ↔ Spring Boot 기본 연결 구조

---

### CORS 설정 — `@CrossOrigin`

React는 `localhost:3000`, Spring Boot는 `localhost:8080` 으로 포트가 다르기 때문에 브라우저가 CORS 정책으로 요청을 차단한다. Spring Boot 컨트롤러에 `@CrossOrigin`을 달아 허용한다.

```java
@RestController
@RequestMapping("/param")
@CrossOrigin(origins = {"http://localhost:3000", "http://127.0.0.1:3000"})
public class ParamController { ... }
```

> `origins = "*"` 로 전체 허용도 가능하지만 운영 환경에서는 도메인을 명시하는 것이 안전하다.
> ⚠ 인증(쿠키/JWT)을 쓰는 API는 `origins="*"`와 `withCredentials:true`를 함께 쓸 수 없다. 이 문제와 CORS 관련 심화 트러블슈팅은 [[3일.md]]의 07_SPRINGSECURITY 단원 참고.

## 01_PARAM — React → Spring Boot 파라미터 전달 3가지

---

### 방식 비교

| 방식 | React (axios) | Spring Boot | 용도 |
|------|--------------|-------------|------|
| 쿼리 파라미터 | `params: { name: '홍길동' }` | `@RequestParam` | GET, 선택적 값 |
| 경로 파라미터 | URL에 직접 삽입 `/02/홍길동/55/대구` | `@PathVariable` (DTO 자동 바인딩) | 식별자, 필수값 |
| Request Body | `axios.post(url, { ... })` | `@RequestBody` | POST, JSON 객체 전송 |

### Spring Boot — `ParamController.java`

```java
@RestController
@RequestMapping("/param")
@CrossOrigin(origins = {"http://localhost:3000","http://127.0.0.1:3000"})
public class ParamController {

    // 방식 1: 쿼리 파라미터 (?name=홍길동)
    @RequestMapping(value="/01", method={RequestMethod.GET, RequestMethod.POST},
                    produces=MediaType.APPLICATION_JSON_VALUE)
    public ResponseEntity<Map<String,Object>> param01(
            @RequestParam(value="name", required=false) String name) {
        Map<String,Object> map = new LinkedHashMap<>();
        map.put("name", name);
        map.put("status", "success");
        return new ResponseEntity<>(map, HttpStatus.OK);
    }

    // 방식 2: 경로 파라미터 (/02/홍길동/55/대구) → DTO 자동 바인딩
    @RequestMapping(value="/02/{name}/{age}/{addr}", method={RequestMethod.GET, RequestMethod.POST},
                    produces=MediaType.APPLICATION_JSON_VALUE)
    public ResponseEntity<Map<String,Object>> param02(PersonDto personDto) {
        Map<String,Object> map = new LinkedHashMap<>();
        map.put("name", personDto.getName());
        map.put("age",  personDto.getAge());
        map.put("addr", personDto.getAddr());
        return new ResponseEntity<>(map, HttpStatus.OK);
    }

    // 방식 3: JSON Body (POST)
    @PostMapping(value="/03", consumes=MediaType.APPLICATION_JSON_VALUE,
                              produces=MediaType.APPLICATION_JSON_VALUE)
    public ResponseEntity<Map<String,Object>> param03(@RequestBody PersonDto personDto) {
        Map<String,Object> map = new LinkedHashMap<>();
        map.put("name", personDto.getName());
        map.put("status", "success");
        return new ResponseEntity<>(map, HttpStatus.OK);
    }
}
```

### React — `Param.jsx`

```jsx
const Param = () => {

    // 방식 1: 쿼리 파라미터 → params 옵션
    const handleParam1 = () => {
        axios.get("http://localhost:8095/param/01", {
            params: { name: '홍길동' },
            headers: { 'Content-Type': 'application/json' }
        }).then(resp => { console.log(resp) });
    };

    // 방식 2: 경로 파라미터 → URL 템플릿 리터럴에 직접 삽입
    const handleParam2 = () => {
        axios.get(`http://localhost:8095/param/02/${'홍길동'}/${55}/${'대구'}`)
             .then(resp => { console.log(resp) });
    };

    // 방식 3: JSON Body → axios.post 두 번째 인자로 객체 전달
    const handleParam3 = () => {
        axios.post("http://localhost:8095/param/03",
            { name: "하하", age: 56, addr: "서울" },
            { "Content-Type": "application/json" }
        ).then(resp => { console.log(resp) });
    };

    return (
        <>
            <button onClick={handleParam1}>GET /param/01?name=홍길동</button>
            <button onClick={handleParam2}>GET /param/02/홍길동/55/대구</button>
            <button onClick={handleParam3}>POST /param/03</button>
        </>
    );
};
```

## 02_VALIDATION — Spring Boot 유효성 검사 → React 에러 표시

---

### Spring Boot — `MemoDto.java` + `MemoController.java`

```java
// DTO: 필드마다 유효성 조건 선언
@Data
public class MemoDto {
    @NotNull(message="ID를 입력하세요")
    @Min(value=10,  message="최소 숫자는 10이상이어야 합니다.")
    @Max(value=65535, message="최대 숫자는 65535이하이어야 합니다.")
    private Integer id;

    @NotBlank(message="메모를 입력하세요")
    private String text;

    @NotBlank(message="작성자를 입력하세요")
    @Email(message="example@example.com 형식으로 입력하세요")
    private String writer;

    @NotNull(message="날짜정보를 선택하세요")
    @DateTimeFormat(pattern="yyyy-MM-dd'T'HH:mm")
    private LocalDateTime regdate;
}

// Controller: @Valid + BindingResult → 에러 필드명:메시지 Map 반환
@PostMapping(value="/add", consumes=MediaType.APPLICATION_JSON_VALUE,
                            produces=MediaType.APPLICATION_JSON_VALUE)
public ResponseEntity<Map<String,Object>> add(
        @RequestBody @Valid MemoDto memoDto, BindingResult bindingResult) {
    Map<String,Object> map = new LinkedHashMap<>();

    if (bindingResult.hasErrors()) {
        for (FieldError error : bindingResult.getFieldErrors()) {
            map.put(error.getField(), error.getDefaultMessage()); // { "id": "ID를 입력하세요" }
        }
        return new ResponseEntity<>(map, HttpStatus.EXPECTATION_FAILED); // 417
    }
    return new ResponseEntity<>(map, HttpStatus.OK);
}
```

### React — 에러 메시지 수신 후 필드 옆에 표시

```jsx
const handleSubmit = () => {
    axios.post("http://localhost:8080/memo/add",
        { id, writer, text, regdate },
        { "Content-Type": "application/json" })
    .then(resp => {
        setMessage(resp.data);         // 성공 응답 (200)
    })
    .catch(err => {
        setMessage(err.response.data); // 실패 응답 (417): { id: "...", writer: "..." }
    });
};

// JSX: message가 있을 때만 에러 메시지 표시
<input name="id" onChange={e => setId(e.target.value)} />
<span>{ message && message.id }</span>
```

> 핵심 흐름: React POST → Spring `@Valid` 검사 → 실패 시 417 + Map 응답 → React `.catch()` → `err.response.data` 로 에러 메시지 수신

## 03_EXCEPTION — 서버 예외 → React 에러 페이지 이동

---

### Spring Boot — `@ExceptionHandler` (컨트롤러 단위)

```java
@RestController
@RequestMapping("/memo")
@CrossOrigin(origins = {"http://127.0.0.1:3000","http://localhost:3000"})
public class MemoController {

    // 특정 예외 처리
    @ExceptionHandler(FileNotFoundException.class)
    public ResponseEntity<Exception> error1(Exception ex) {
        return new ResponseEntity<>(ex, HttpStatus.EXPECTATION_FAILED);
    }

    @ExceptionHandler(ArithmeticException.class)
    public ResponseEntity<Exception> error2(Exception ex) {
        return new ResponseEntity<>(ex, HttpStatus.EXPECTATION_FAILED);
    }

    // 위에서 잡히지 않은 모든 예외 포괄 처리
    @ExceptionHandler(Exception.class)
    public ResponseEntity<Exception> error3(Exception ex) {
        return new ResponseEntity<>(ex, HttpStatus.EXPECTATION_FAILED);
    }

    // 예외를 의도적으로 발생시키는 테스트 엔드포인트
    @GetMapping("/list1") public void list1() throws Exception { throw new FileNotFoundException("파일이 없다.."); }
    @GetMapping("/list2") public void list2() throws Exception { throw new ArithmeticException("계산똑바로해.."); }
    @GetMapping("/list3/{n1}/{n2}") public void list3(@PathVariable int n1, @PathVariable int n2) { System.out.println(n1/n2); }
    @GetMapping("/list4") public void list4() { throw new NullPointerException(); }
}
```

### React — `.catch()` 에서 에러 데이터를 받아 에러 페이지로 `navigate`

```jsx
// List.jsx
const navigate = useNavigate();

const handlerRequestList1 = () => {
    axios.get("http://localhost:8080/memo/list1")
         .then(resp => { console.log(resp) })
         .catch(err => {
             // err.response.data : 서버에서 반환한 예외 객체
             // err.response.status : HTTP 상태 코드
             navigate('/exception', {
                 state: {
                     message: err.response.data.message,
                     status:  err.response?.status
                 }
             });
         });
};

// Exception.jsx — navigate의 state를 useLocation으로 수신
const Exception = () => {
    const location = useLocation();
    const { message, status } = location.state || {};
    return (
        <div>
            <h1>EXCEPTION</h1>
            {status  && <p>Status Code: {status}</p>}
            {message && <p>Message: {message}</p>}
        </div>
    );
};
```

> 핵심 흐름: axios `.catch(err)` → `err.response.data.message` 추출 → `navigate('/exception', { state: {...} })` → 에러 페이지에서 `useLocation().state` 로 표시

## 04_JPA — React 폼 → Spring Boot → DB (전체 레이어 연결)

---

### 레이어 구조

```
React (Post.jsx)
  ↓ axios.post (JSON)
Controller (MemoController)
  ↓ @RequestBody @Valid MemoDto
Service (MemoServiceImpl)
  ↓ Memo 엔티티 변환 + save()
Repository (MemoRepository)
  ↓ JPA
Database (Memo 테이블)
```

### Spring Boot — Entity + Repository + Service

```java
// Entity: DB 테이블과 매핑
@Entity @Data @NoArgsConstructor @AllArgsConstructor
public class Memo {
    @Id @GeneratedValue(strategy = GenerationType.IDENTITY)
    private long id;
    private String text;
    private String writer;
    private LocalDateTime regdate;
}

// Repository: JpaRepository 상속으로 save/findAll/findById/delete 자동 제공
public interface MemoRepository extends JpaRepository<Memo, Long> { }

// Service: DTO → Entity 변환 후 저장
@Service
public class MemoServiceImpl {
    @Autowired
    private MemoRepository memoRepository;

    @Transactional(rollbackFor=Exception.class)
    public boolean addMemo(MemoDto dto) throws Exception {
        Memo memo = new Memo();
        memo.setText(dto.getText());
        memo.setWriter(dto.getWriter());
        memo.setRegdate(LocalDateTime.now());
        memoRepository.save(memo);  // INSERT
        return true;
    }
}
```

### React — `Post.jsx` (폼 입력 → axios POST → 결과 표시)

```jsx
const Memo = () => {
    const [id, setId]         = useState();
    const [writer, setWriter] = useState();
    const [text, setText]     = useState();
    const [regdate, setRegdate] = useState();
    const [message, setMessage] = useState(null);

    const handleSubmit = () => {
        axios.post("http://localhost:8080/memo/post",
            { id, writer, text, regdate },
            { "Content-Type": "application/json" })
        .then(resp => { setMessage(resp.data); })        // 성공
        .catch(err  => { setMessage(err.response.data); }); // 유효성 에러
    };

    return (
        <div>
            <input name="writer" onChange={e => setWriter(e.target.value)} />
            <span>{ message && message.writer }</span>   {/* 에러 메시지 표시 */}

            <input name="text" onChange={e => setText(e.target.value)} />
            <span>{ message && message.text }</span>

            <input type="datetime-local" onChange={e => setRegdate(e.target.value)} />

            <button onClick={handleSubmit}>전송</button>
        </div>
    );
};
```

## 1일 핵심 정리

---

### 1) React → Spring Boot 파라미터 전달 방식

```
쿼리 파라미터  axios.get(url, { params: { key: val } })
               → @RequestParam

경로 파라미터  axios.get(`/path/${val1}/${val2}`)
               → URL 패턴 매칭 + DTO 자동 바인딩

JSON Body      axios.post(url, { key: val })
               → @RequestBody PersonDto
```

---

### 2) 응답 흐름 — 성공 vs 실패

```
성공 200 OK      → .then(resp)    → resp.data
유효성 실패 417  → .catch(err)   → err.response.data  (필드명: 에러메시지 Map)
서버 예외 417    → .catch(err)   → err.response.data.message
                                   → navigate('/exception', { state: {...} })
                                   → useLocation().state 로 수신
```

---

### 3) 학습 흐름 (01~04)

```
React ↔ Spring Boot 연동 학습 흐름
  │
  ├── 01_PARAM — 파라미터 전달 방식
  │     ├── @RequestParam  (쿼리 파라미터)
  │     ├── @PathVariable  (경로 파라미터 → DTO 바인딩)
  │     └── @RequestBody   (JSON body → POST)
  │
  ├── 02_VALIDATION — 서버 사이드 유효성 검사
  │     ├── DTO 필드: @NotNull @NotBlank @Min @Max @Email
  │     ├── Controller: @Valid + BindingResult
  │     └── React: .catch(err) → err.response.data (에러 Map)
  │
  ├── 03_EXCEPTION — 예외 처리 → 에러 페이지
  │     ├── @ExceptionHandler (컨트롤러 단위 예외 처리)
  │     ├── React: .catch() → navigate('/exception', { state })
  │     └── Exception.jsx: useLocation().state 로 메시지 수신
  │
  └── 04_JPA — CRUD 전체 레이어
        ├── @Entity / @Id / @GeneratedValue
        ├── JpaRepository.save() / findAll() / deleteById()
        ├── @Transactional (예외 시 롤백)
        └── React 폼 → axios → Controller → Service → Repository → DB
```
