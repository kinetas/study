# 7월 20일 필기 — 머신러닝: 선형회귀 파이프라인, 단순회귀 vs 다중회귀 R² 비교, statsmodels로 p-value 검증

## 핵심요약

**04_MACHINE_LEARNING/Supervised_Learning/03_Liner_Regression/01.ipynb**로 BigMart 매출 데이터(`train.csv` 6818행×12열 / `test.csv` 1705행×11열)를 이용한 선형회귀 파이프라인을 완성했다. 타겟 `Item_Outlet_Sales`가 범주가 아닌 **연속 숫자**라는 점에서 지금까지의 이진/다중분류와 근본적으로 다른데, "몇 종류인가"라는 질문 자체가 성립하지 않으니 채점 지표도 정확도/f1/ROC-AUC 대신 **RMSE·MAE·R²**로 통째로 바뀐다. 특성 1개(`Item_MRP`)만 쓴 단순회귀는 R² 0.3123으로 약했는데, 식별자를 뺀 나머지 특성을 전부 원핫 인코딩해 넣은 다중회귀는 R² 0.5669로 크게 개선됐다 — 특성 수가 회귀 성능에 직접 기여한다는 걸 같은 데이터로 직접 대조 확인했다. 마지막엔 `statsmodels`의 `OLS`로 같은 단순회귀를 다시 적합해 sklearn의 계수(15.58)와 정확히 일치함을 확인하고, `Item_MRP`는 p-value 0.000으로 유의미하지만 절편(`const`)은 p-value 0.934로 통계적으로 유의하지 않다는 것까지 짚었다 — sklearn은 계수만 주고 statsmodels는 그 계수를 믿어도 되는지까지 알려준다는 역할 차이를 실습으로 확인한 하루. 참고로 같은 폴더의 `01-Copy1.ipynb`는 내용을 열어보니 이전에 완성한 다중분류(`Multiclass_Classfication`) 노트북과 완전히 동일한 파일이 잘못 복사돼 들어와 있던 것으로, 선형회귀 내용이 아니었다.

## 20일 핵심 정리

---

### 1) 같은 데이터로 특성 1개 vs 35개 — R² 차이를 직접 대조

```
단순회귀 (Item_MRP 1개)        : R² 0.3123
다중회귀 (원핫 인코딩 35열)     : R² 0.5669
```
상관계수가 제일 높은 특성 하나만 써도 회귀가 "되긴" 하지만, 나머지 약한 특성들(`Outlet_Type`, `Outlet_Size` 등)을 다 넣으니 설명력이 거의 2배 가까이 뛰었다. 분류에서 "특성 하나로도 어느 정도 맞는다"는 감각을 회귀 R²로 숫자화해서 확인한 셈.

### 2) EDA에서 상관계수부터 보는 습관 (회귀 전용)

```python
num.corr()['Item_Outlet_Sales'].sort_values(ascending=False)
# Item_MRP  0.567 (최강) / Item_Weight 0.02 / Outlet_Establishment_Year -0.05 / Item_Visibility -0.12
```
분류 EDA에서 `value_counts()`로 클래스 분포부터 보듯, 회귀 EDA에서는 타겟과의 상관계수부터 훑는 게 첫 단추. 상관이 약한 특성(`Item_Weight`)이라고 무조건 버리는 게 아니라, 다중회귀에 넣었을 때 오히려 전체 R²가 올라간 걸 보면 "단일 상관계수"와 "여러 특성을 같이 넣었을 때의 기여도"는 다른 이야기라는 것도 확인됨.

### 3) sklearn 계수 vs statsmodels p-value — 역할이 다르다

```
sklearn  LinearRegression.coef_    : 15.58   (계수만 알려줌)
statsmodels OLS summary            : coef 15.58, P>|t| 0.000  (계수 + 믿을 만한지)
```
두 라이브러리가 계산한 기울기 15.58이 정확히 같다는 걸 확인해 "같은 선형회귀"라는 걸 검증한 뒤, statsmodels에서만 나오는 p-value로 `Item_MRP`는 유의미하지만 절편은 유의하지 않다는 통계적 해석까지 얻었다. sklearn은 예측·서비스용, statsmodels는 "이 계수를 근거로 삼아도 되는가"를 확인하는 분석용이라는 용도 차이로 이해하면 됨.

### 4) 타겟이 "연속값"이 되는 순간 채점 지표 세트가 통째로 바뀜

```
분류(이진/다중) : 정확도, f1(macro/weighted), ROC-AUC
회귀            : RMSE, MAE, R²
```
16일 다중분류에서 "양성 개념이 없어지니 ROC-AUC를 못 쓴다"고 정리했던 것과 같은 맥락 — 이번엔 "몇 종류인가"라는 질문 자체가 없어지니(연속값) 분류 지표 전체가 무의미해지고 회귀 지표로 전면 교체된다는 걸 실습으로 확인. "문제정의 4가지 결정"에서 2)번 답이 바뀌면 3)·4)번도 따라 바뀐다는 원칙이 분류 사이(이진↔다중)뿐 아니라 분류↔회귀 사이에서도 그대로 적용됨.
