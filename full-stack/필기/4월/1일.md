# 4월 1일 필기 — 데이터베이스 (database.md)

## 핵심요약

데이터베이스는 앱·웹의 정보를 영구적으로 저장·접근·공유하는 시스템이며, 현업에서는 관계형 DBMS(MySQL 등)가 주로 사용된다. DDL로 데이터베이스와 테이블을 생성(CREATE)·삭제(DROP)·변경(ALTER)하며, 컬럼에는 INT·VARCHAR·DATETIME 같은 타입과 NOT NULL·PRIMARY KEY 같은 제약조건을 지정한다. DML로 데이터를 삽입(INSERT)·수정(UPDATE)·삭제(DELETE)·조회(SELECT)하며, ALTER TABLE의 ADD/DROP/CHANGE/MODIFY로 컬럼을 동적으로 관리한다.


# 데이터베이스

> **정의**  
> 앱·웹을 위한 정보의 **영구 저장**을 담당하고, 데이터를 **체계적으로 저장·접근·공유**할 수 있게 하는 시스템.

---

## DBMS

**DBMS(Database Management System)**: MySQL, Oracle 등 다양한 제품군.

| 분류 | 설명 |
|------|------|
| 구조 유형 | 계층형 · 망형 · 관계형 · 객체지향형 · 객체-관계형 |
| 일반적 사용 | 현업에서는 **관계형 DBMS** 구조가 널리 쓰입니다 |

## SQL

데이터베이스·테이블을 다루는 기본 명령 정리.

### 데이터베이스

| 목적 | 예시 |
|------|------|
| 생성 | `CREATE DATABASE db_name;` |
| 선택(사용) | `USE db_name;` |
| 삭제 | `DROP DATABASE db_name;` |

### 테이블

**생성**

```sql
CREATE TABLE table_name (
    column_name TYPE CONSTRAINT,
    ...
);
```

**컬럼 타입 예**

- `INT`
- `VARCHAR(허용크기)`
- `DATETIME`
- 기타 필요한 타입

**제약 조건 예**

- `NULL` / `NOT NULL`
- `PRIMARY KEY`
- 기타 제약 조건

**삭제**

```sql
DROP TABLE table_name;
```

**변경 (`ALTER`)**

```sql
ALTER TABLE table_name 변경내용;
```

### DDL 요약 — `ALTER TABLE`에서 자주 쓰는 패턴

| 구문 패턴 | 의미 |
|-----------|------|
| `ADD COLUMN 컬럼명 타입 [제약] …` | 컬럼 추가 |
| `DROP COLUMN 컬럼명` | 컬럼 삭제 |
| `… ADD COLUMN … AFTER 기존컬럼명` | 새 컬럼을 특정 컬럼 **바로 뒤**에 배치 |
| `CHANGE COLUMN 기존이름 새이름 타입 [제약]` | 컬럼 이름·타입·제약 일괄 변경 (MySQL) |
| `MODIFY COLUMN 컬럼명 타입 [제약]` | 이름은 유지하고 타입·제약만 변경 (MySQL) |

> DBMS마다 `CHANGE` / `MODIFY` 지원 문법 차이가 있을 수 있으므로 제품 문서 병행 확인.

---

### DML

**데이터 조작**용 명령. 테이블 **행** 삽입·수정·삭제용.

| 목적 | 핵심 형태 |
|------|-----------|
| 삽입 | `INSERT INTO … VALUES …` |
| 수정 | `UPDATE … SET … WHERE …` |
| 삭제 | `DELETE FROM … WHERE …` |
| 조회 | `SELECT … FROM … WHERE …` |

**삽입**

```sql
INSERT INTO 테이블명 (컬럼1, 컬럼2, …)
VALUES ('값1', '값2', …);
```

**수정**

```sql
UPDATE 테이블명
SET 컬럼1 = '내용', …
WHERE 조건;
```

**삭제**

```sql
DELETE FROM 테이블명
WHERE 조건;
```
