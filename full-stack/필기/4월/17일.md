# 4월 17일 필기 — JAVA

## 핵심요약

데이터는 선저장 후 처리가 원칙이며, 저장할 공간의 형태를 지정하는 것이 자료형이다. 기본 자료형(Primitive)은 값 자체를 저장하고, 클래스 자료형(참조변수)은 데이터가 저장된 메모리 주소값을 저장한다. 변수는 유지보수 편의를 위해 공간에 이름을 붙인 것이며, `=` 연산자는 우변(rv)을 먼저 처리한 뒤 좌변(lv) 공간에 대입한다. 실수는 부동소수점 방식이 기본이며, `float`은 4byte(접미사 `F`), `double`은 8byte로 기본 실수형이다.

### 2) Primitive 자료형 (기본 타입)

| 자료형 | 크기 | 부호 | 범위 | 비고 |
|--------|------|------|------|------|
| `byte` | 1 byte | signed | -128 ~ 127 | |
| `char` | 2 byte | unsigned | 0 ~ 65535 | 단일문자 |
| `short` | 2 byte | signed | -32768 ~ 32767 | |
| `int` | 4 byte | signed | 약 ±21억 | **기본 정수형** |
| `long` | 8 byte | signed | 매우 큰 수 | 접미사 `L` |
| `float` | 4 byte | — | 6~9자리 정밀도 | 접미사 `F` |
| `double` | 8 byte | — | 15~18자리 정밀도 | **기본 실수형** |
| `boolean` | — | — | true / false | 논리형 |

### 3) Class 자료형 (참조 타입)

| 자료형 | 설명 |
|--------|------|
| `String` | 문자열, 참조변수 — 데이터가 저장된 메모리 주소값을 저장 |

> 기본자료형 변수 → 값 자체 저장  
> 클래스자료형 변수(참조변수) → 데이터가 저장된 **메모리 주소값** 저장

### 4) CH01 — C04변수_자료형

**핵심 개념**

| 용어 | 설명 |
|------|------|
| 변수 | 유지보수 편의를 위해 지정한 수(바뀔 예정인 수) |
| 변수명 | 변수 공간에 접근하기 위한 문자 형태의 주소 |
| 자료형 | 저장 공간을 형성하고 저장될 자료의 형태를 제한하는 예약어 |
| `=` 연산자 | `lv = rv` — rv를 먼저 처리(저장/연산) 후 lv에 대입 |
| 리터럴상수 | 코드에 직접 쓴 값, 상수 POOL에 저장됨 |

```java
int num1;                   // 4byte 공간 형성 + num1 이름 부여 (변수 정의)
num1 = 10;                  // 리터럴상수 10 → 상수POOL 저장, num1에 대입
int num2 = 4;               // 상수POOL 저장 + num2 초기화
int num3 = num1 + num2;     // num1, num2 값 가져와 CPU 가산처리 → num3 초기화
System.out.println(num3);   // num3 값을 println으로 표준출력
```

### 5) CH01 — C05변수_자료형

**정수형 상세**

| 자료형 | 크기 | 특이사항 |
|--------|------|----------|
| `int` | 4byte | 기본 정수형 |
| `byte` | 1byte | 범위 초과 시 캐스팅 필요 — `(byte)129` → -127 |
| `short` | 2byte | signed, -32768 ~ 32767 |
| `char` | 2byte | unsigned, 0 ~ 65535 (문자 저장) |
| `long` | 8byte | 리터럴 접미사 `L` 필수 — `10000000000L` |

> `char`는 부호 없는 정수형이므로 `short`와 범위가 다르다

**실수형**

> 소수점 이하 값을 가지는 수 (유리수 + 무리수)  
> **고정소수점** : 빠른 저장, 메모리 낭비  
> **부동소수점** : 기본값, 효율적 저장

| 자료형 | 크기 | 정밀도 | 접미사 |
|--------|------|--------|--------|
| `float` | 4byte | 6~9자리 | `F` 또는 `f` |
| `double` | 8byte | 15~18자리 | 없음 (기본) |

```java
float n1  = 0.123456789123456789F;  // 정밀도 낮음
double n2 = 0.123456789123456789;   // 정밀도 높음
```

**문자형 char**

```java
char ch1 = 'a';              // 문자 저장
char ch2 = 98;               // 정수(아스키코드)로도 저장 가능
char ch3 = 'b' + 1;          // 연산 결과 저장
// \\u : 유니코드 이스케이프문자
System.out.printf("%c\n", '\uac03');
```

**논리형 boolean**

```java
boolean flag = (10 > 11);   // false

if (flag) {
    System.out.println("참인경우 실행");
} else {
    System.out.println("거짓인경우 실행");
}
```

**String (클래스 자료형)**

```java
String name = "홍길동";
String job  = "프로그래머";

// UTF-8 기준 한글 1글자 = 3byte
System.out.println(name.getBytes(StandardCharsets.UTF_8).length); // 9

char ch  = '홍';    // 16bit = 2byte (내부 저장)
String str = "홍";  // 24bit = 3byte (UTF-8 기준)
```

> `char`는 내부적으로 2byte, `String`의 한글은 UTF-8 기준 3byte 사용
