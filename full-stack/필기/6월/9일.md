# 6월 9일 필기 — Spring Boot / Transaction (JPA & MyBatis)

## 핵심요약

10_TX와 10_TX_MYBATIS에서는 Spring의 트랜잭션을 Java Config로 직접 구성하고 `@Transactional`로 롤백 동작을 실습했다. 트랜잭션을 수동으로 설정하려면 `DataSourceConfig`(HikariCP)→`ORM Config`(JPA는 `JPAConfig`, MyBatis는 `MybatisConfig`)→`TxConfig`(TransactionManager 빈) 순으로 세 개의 Config 클래스를 만든다. JPA는 `JpaTransactionManager`, MyBatis는 `DataSourceTransactionManager`를 사용하며 빈 이름을 명시해야 한다. `@Transactional`을 붙이지 않으면 메서드 안에서 예외가 발생해도 이미 실행된 INSERT는 그대로 커밋되어 DB에 남지만, `@Transactional(rollbackFor = SQLException.class, transactionManager = "빈이름")`을 붙이면 예외 발생 시 해당 트랜잭션 전체가 롤백된다. `RuntimeException`은 기본적으로 롤백되지만 `SQLException`처럼 `CheckedException`은 기본적으로 커밋되므로 `rollbackFor`로 명시해야 롤백된다. 10_TX_MYBATIS에서는 추가로 `ProductDAO`/`ProductMapper` 실습을 통해 어노테이션 CRUD(`@Insert`/`@Select`/`@Update`/`@Delete`)와 XML 매퍼를 함께 사용하는 패턴을 익혔다. MyBatis XML 동적 SQL은 `<if>`로 단일 조건 분기, `<where>`로 앞의 AND/OR 자동 제거, `<choose>/<when>/<otherwise>`로 switch 역할, `<foreach>`로 컬렉션 반복 검색을 처리한다.

### 2) JPAConfig.java — EntityManagerFactory 수동 구성

```java
@Configuration
@EntityScan(basePackages = {"com.example.demo.Domain.Common.Entity"})
@EnableJpaRepositories(basePackages = {"com.example.demo.Domain.Common.Repository"})
public class JPAConfig {
    @Autowired
    private DataSource dataSource;

    @Bean
    LocalContainerEntityManagerFactoryBean entityManagerFactory() {
        LocalContainerEntityManagerFactoryBean emf = new LocalContainerEntityManagerFactoryBean();
        emf.setDataSource(dataSource);
        emf.setJpaVendorAdapter(new HibernateJpaVendorAdapter());
        emf.setPackagesToScan("com.example.demo.Domain.Common.Entity");

        Map<String, Object> props = new HashMap<>();
        props.put("hibernate.hbm2ddl.auto", "update");  // 엔티티 변경 시 ALTER 반영
        props.put("hibernate.show_sql", true);
        emf.setJpaPropertyMap(props);
        return emf;
    }
}
```

| 설정 키 | 값 | 설명 |
|---|---|---|
| `hibernate.hbm2ddl.auto` | `update` | 엔티티 변경 시 ALTER TABLE 자동 반영 |
| `hibernate.show_sql` | `true` | 실행 SQL 콘솔 출력 |

### 3) TxConfig.java — JpaTransactionManager

```java
@Configuration
@EnableTransactionManagement
public class TxConfig {
    @Autowired
    private DataSource dataSource;

    @Bean(name = "jpaTransactionManager")
    public JpaTransactionManager jpaTransactionManager(EntityManagerFactory emf) {
        JpaTransactionManager tm = new JpaTransactionManager();
        tm.setEntityManagerFactory(emf);
        tm.setDataSource(dataSource);
        return tm;
    }
}
```

> `@EnableTransactionManagement` — `@Transactional` 어노테이션이 동작하도록 활성화

## Spring Boot — 10_TX 트랜잭션 실습

### 1) 트랜잭션 없음 vs @Transactional 롤백 비교

```java
@Service @Slf4j
public class TxTestService {
    @Autowired
    private MemoRepository memoRepository;

    // 트랜잭션 없음 → 3건 INSERT 후 예외 발생해도 이미 커밋된 3건은 DB에 남음
    public void addMemo() throws Exception {
        Memo memo = Memo.builder().text("addMemoTx...").writer("a@a.com")
                        .createAt(LocalDateTime.now()).build();
        memoRepository.save(memo);
        memo.setId(null); memoRepository.save(memo);
        memo.setId(null); memoRepository.save(memo);
        memo.setId(null);
        throw new SQLException();   // 예외 발생해도 위 3건은 이미 커밋
    }

    // @Transactional → 예외 발생 시 3건 모두 롤백
    @Transactional(rollbackFor = SQLException.class,
                   transactionManager = "jpaTransactionManager")
    public void addMemoTx() throws Exception {
        Memo memo = Memo.builder().text("addMemoTx...").writer("a@a.com")
                        .createAt(LocalDateTime.now()).build();
        memoRepository.save(memo);
        memo.setId(null); memoRepository.save(memo);
        memo.setId(null); memoRepository.save(memo);
        memo.setId(null);
        throw new SQLException();   // 트랜잭션 전체 롤백 → DB에 0건
    }
}
```

| 메서드 | `@Transactional` | 예외 발생 시 결과 |
|---|---|---|
| `addMemo()` | 없음 | 3건 DB에 남음 (부분 커밋) |
| `addMemoTx()` | `rollbackFor = SQLException.class` | 전체 롤백 (0건) |

| 예외 종류 | 기본 동작 | rollbackFor 필요 여부 |
|---|---|---|
| `RuntimeException` (unchecked) | 자동 롤백 | 불필요 |
| `SQLException` (checked) | 기본 커밋 | `rollbackFor = SQLException.class` 명시 필요 |

### 3) 핵심 정리

```
트랜잭션 수동 설정 흐름 (JPA)
```

## Spring Boot — 10_TX_MYBATIS Config 구성

### 1) MybatisConfig.java — SqlSessionFactory + SqlSessionTemplate

```java
@Configuration
public class MybatisConfig {
    @Autowired
    private DataSource dataSource;

    @Bean
    public SqlSessionFactory sqlSessionFactory() throws Exception {
        SqlSessionFactoryBean sf = new SqlSessionFactoryBean();
        sf.setDataSource(dataSource);
        Resource[] resources =
            new PathMatchingResourcePatternResolver()
                .getResources("classpath*:mapper/*.xml");   // XML 매퍼 위치 지정
        sf.setMapperLocations(resources);
        return sf.getObject();
    }

    @Bean
    public SqlSessionTemplate sqlSessionTemplate() throws Exception {
        return new SqlSessionTemplate(sqlSessionFactory());
    }
}
```

```java
public interface ProductMapper {
```

### 4) ProductDAO.java — Mapper 래핑 DAO

```java
@Repository @Slf4j
public class ProductDAO {
    @Autowired
    private ProductMapper productMapper;   // DAO가 Mapper를 감싼다

    public int insert(ProductDTO dto)   { return productMapper.insert(dto); }
    public List<ProductDTO> selectAll() { return productMapper.selectAllXML(); }
    public ProductDTO selectOne(Long id){ return productMapper.selectOne(id); }
    public int update(ProductDTO dto)   { return productMapper.update(dto); }
    public int delete(Long id)          { return productMapper.delete(id); }

    public List<Map<String, Object>> search(String keyword, Integer minStock) {
        Map<String, Object> param = new HashMap<>();
        param.put("keyword", keyword);
        param.put("minStock", minStock);
        return productMapper.searchXML(param);  // 동적 SQL 호출
    }
}
```

> 흐름: `ProductController → ProductDAO → ProductMapper(@Mapper) → XML`  
> DAO는 Mapper를 감싸는 계층으로 로깅, 파라미터 조합 등을 담당한다

## Spring Boot — 10_TX_MYBATIS 동적 SQL (MemoMapper.xml)

---

### 1) `<if>` — 단일 조건 분기

```xml
<select id="selectAllIfXML" resultType="java.util.Map" parameterType="java.util.Map">
    select * from tbl_memo
    <if test="type != null and type.equals('text')">
        where text like concat('%', #{keyword}, '%')
    </if>
</select>
```

> 조건이 없으면 `<if>` 블록이 통째로 생략되어 전체 조회가 된다

---

### 2) `<choose>/<when>/<otherwise>` — switch 문 역할

```xml
<select id="selectAllChooseXML" resultType="java.util.Map" parameterType="java.util.Map">
    select * from tbl_memo
    <where>
        <if test="field != null">
            <choose>
                <when test="type == 'id'">     id like concat('%', #{keyword}, '%') </when>
                <when test="type == 'text'">   text like concat('%', #{keyword}, '%') </when>
                <when test="type == 'writer'"> writer like concat('%', #{keyword}, '%') </when>
                <otherwise>                    createAt like concat('%', #{keyword}, '%') </otherwise>
            </choose>
        </if>
    </where>
</select>
```

| 태그 | 역할 |
|---|---|
| `<choose>` | switch 블록 시작 |
| `<when test="조건">` | case 분기 |
| `<otherwise>` | default (조건 미해당 시) |

---

### 3) `<foreach>` — 복수 필드 AND 검색

```xml
<select id="selectForEachAnd" parameterType="java.util.Map" resultType="java.util.Map">
    SELECT * FROM tbl_memo
    <where>
        <if test="fields != null and keyword != null">
            <foreach collection="fields" item="f" separator="AND">
                <choose>
                    <when test="f == 'id'">     id LIKE CONCAT('%', #{keyword}, '%') </when>
                    <when test="f == 'text'">   text LIKE CONCAT('%', #{keyword}, '%') </when>
                    <when test="f == 'writer'"> writer LIKE CONCAT('%', #{keyword}, '%') </when>
                </choose>
            </foreach>
        </if>
    </where>
</select>
```

| 속성 | 설명 |
|---|---|
| `collection` | 반복할 컬렉션 (Map key 또는 파라미터명) |
| `item` | 반복 변수명 |
| `separator` | 각 반복 사이에 삽입할 문자열 (`AND`, `,` 등) |

## Spring Boot — 10 핵심 정리

---

### 1) JPA vs MyBatis 트랜잭션 비교

| 항목 | 10_TX (JPA) | 10_TX_MYBATIS (MyBatis) |
|---|---|---|
| TransactionManager | `JpaTransactionManager` | `DataSourceTransactionManager` |
| 빈 이름 | `"jpaTransactionManager"` | `"dataSourceTransactionManager"` |
| ORM Config | `JPAConfig` (EntityManagerFactory) | `MybatisConfig` (SqlSessionFactory) |
| 데이터 접근 | `JpaRepository.save()` | `@Mapper` / XML |
| `@Transactional` 명시 | `transactionManager = "jpaTransactionManager"` | `transactionManager = "dataSourceTransactionManager"` |

---

### 2) 트랜잭션 공통 패턴

```
1. DataSourceConfig  → DataSource (HikariCP) 빈 등록
2. ORM Config        → EntityManagerFactory(JPA) 또는 SqlSessionFactory(MyBatis)
3. TxConfig          → TransactionManager 빈 (이름 지정 필수)
4. Service 메서드    → @Transactional(rollbackFor=..., transactionManager="빈이름")
```

---

### 3) 동적 SQL 태그 정리

| 태그 | 역할 | 비고 |
|---|---|---|
| `<if test="조건">` | 조건 충족 시 SQL 추가 | 단일 조건 분기 |
| `<where>` | 자식 조건이 있으면 WHERE 삽입, 앞의 AND/OR 자동 제거 | `<if>`와 조합 |
| `<choose>/<when>/<otherwise>` | switch/case/default | 여러 조건 중 하나만 |
| `<foreach collection item separator>` | 컬렉션 반복으로 SQL 생성 | AND/OR/IN 등 |

---

### 4) 핵심 포인트 요약

| 항목 | 설명 |
|---|---|
| `@EnableTransactionManagement` | `@Transactional`이 동작하도록 Config에 추가 |
| `rollbackFor = SQLException.class` | checked 예외도 롤백되도록 명시 |
| `transactionManager = "빈이름"` | TM이 여러 개일 때 어떤 TM을 쓸지 지정 |
| 트랜잭션 없음 | 각 save/insert가 즉시 커밋 → 예외 후에도 DB에 남음 |
| 트랜잭션 있음 | 예외 발생 시 메서드 내 모든 작업이 롤백 |
| `DAO → Mapper` 패턴 | DAO가 Mapper를 감싸 파라미터 조합·로깅 담당 |
| `<where>` | 조건이 없을 때 WHERE 절 자체를 생략해 전체 조회로 동작 |
