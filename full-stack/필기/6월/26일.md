# 6월 26일 필기 — Node.js 개요 · 번들러(Parcel) · SCSS

## 핵심요약

오늘은 **Node.js 전반**과 **번들러(Parcel v1/v2)**, **SCSS** 실습을 정리했다. Node.js는 Google V8 엔진 기반의 서버 사이드 JavaScript 런타임으로, 비동기/논 블로킹 I/O와 단일 스레드 이벤트 루프가 핵심 특징이다. 실습은 세 프로젝트로 진행됐다. FIRST는 Parcel v2 + CommonJS(`require`)로 lodash의 `_.shuffle()`을 다뤘고, TMP는 Parcel v1 + ES Module(`import`)로 `_.join()`을 사용해 CommonJS와 ESM의 차이를 비교했다. SCSS 실습에서는 선택자 중첩(Nesting), `&` 부모 참조, 속성 중첩(Property Nesting), 전역/지역 변수(`$variable`) 스코프 규칙을 다뤘다. 번들러는 소스 코드를 브라우저가 이해할 수 있는 단일 파일로 묶어주는 도구로, Parcel v1은 `.cache`를 Parcel v2는 `.parcel-cache`를 생성하는 차이가 있다.

## Node.js — 개요

---

### 1) Node.js란

Node.js는 웹 브라우저 외의 환경(주로 서버)에서 JavaScript를 실행할 수 있게 해주는 **오픈 소스 런타임 환경**이다. Google의 **V8 엔진**을 기반으로 하며, 데이터 집약적인 실시간 애플리케이션 개발에 특화되어 있다.

---

### 2) 주요 특징

| 특징 | 설명 |
|------|------|
| 자바스크립트 런타임 | 브라우저 밖(주로 서버)에서 JavaScript를 실행 |
| V8 엔진 사용 | Chrome과 동일한 고성능 V8 엔진 기반 → 빠른 실행 속도 |
| 비동기 I/O (Non-blocking I/O) | I/O 작업 중에도 다른 작업을 처리 → 대규모 동시 요청에 유리 |
| 단일 스레드 + 이벤트 루프 | 하나의 스레드 + Event Loop 메커니즘으로 비동기 작업을 효율적으로 처리 |
| NPM | 세계 최대 오픈 소스 라이브러리 생태계 — 방대한 모듈 활용 가능 |
| 내장 HTTP 서버 | Apache, Nginx 없이도 자체적으로 웹 서버 구축 가능 |

---

### 3) 장점 및 단점

**장점 (Pros)**

| 항목 | 내용 |
|------|------|
| 개발 생산성 | 프론트엔드 · 백엔드 모두 JavaScript 하나로 개발 가능 (코드 재사용성↑) |
| 고성능/효율성 | 비동기/논 블로킹 I/O → 네트워크·디스크 I/O 집약 환경에서 높은 처리량 |
| 경량 · 빠른 개발 | 서버 설치부터 애플리케이션 실행까지 빠르고 가벼움 |
| 방대한 생태계 | npm으로 다양한 모듈 · 라이브러리를 즉시 활용 |
| 낮은 러닝 커브 | JS 개발자는 새 언어 습득 없이 서버 개발 진입 가능 |

**단점 (Cons)**

| 항목 | 내용 |
|------|------|
| CPU 집약적 작업에 취약 | 단일 스레드 → CPU-intensive 작업 발생 시 전체 블로킹 (클러스터 모듈로 보완) |
| 콜백 지옥 | 비동기 중첩 콜백 → 가독성·유지보수 저하 (Promise, Async/Await으로 해결) |
| 프로세스 안정성 | 단일 스레드에서 예외 발생 시 전체 프로세스 종료 → 에러 관리 필수 |

## 번들러(Parcel) · 모듈 시스템 — 실습 프로젝트

---

### 1) FIRST — Parcel v2 + CommonJS + lodash

Parcel v2 번들러와 Node.js의 **CommonJS 모듈 시스템**(`require`)으로 lodash를 불러오는 방식을 다룬 첫 번째 실습.

```js
// index.js
const _ = require("lodash");

const arr = [1, 2, 3, 4, 5];
const shuffled = _.shuffle(arr);
console.log("shuffled:", shuffled);
```

| 항목 | 내용 |
|------|------|
| 번들러 | Parcel v2 (캐시 디렉토리: `.parcel-cache`) |
| 모듈 방식 | CommonJS (`require`) |
| 핵심 라이브러리 | lodash |
| 실습 포인트 | `_.shuffle()` — 배열 요소를 무작위로 섞기 |

---

### 2) TMP — Parcel v1 + ES Module + lodash

Parcel v1 번들러와 **ES Module 방식**(`import`)으로 lodash를 사용하며, 브라우저에서 실행되는 클라이언트 사이드 번들링을 다룬 실습.

```js
// index.js
import _ from 'lodash';

const result = _.join(['Hello', 'Parcel'], "|");
console.log(result); // Hello|Parcel
```

| 항목 | 내용 |
|------|------|
| 번들러 | Parcel v1 (캐시 디렉토리: `.cache`, 출력: `dist/TMP.e31bb0bc.js`) |
| 모듈 방식 | ES Module (`import`) |
| 핵심 라이브러리 | lodash |
| 실습 포인트 | `_.join()` — 배열 요소를 구분자로 연결 |

---

### 3) CommonJS vs ES Module 비교

| 항목 | CommonJS (FIRST) | ES Module (TMP) |
|------|-----------------|-----------------|
| 문법 | `require()` / `module.exports` | `import` / `export` |
| 실행 환경 | Node.js 서버 환경 기본 | 최신 브라우저 · 번들러 표준 |
| 로딩 방식 | 동기(Synchronous) | 비동기(Asynchronous) |
| 번들러 | Parcel v2 | Parcel v1 |
| 캐시 디렉토리 | `.parcel-cache` | `.cache` |

### 2) 01 — 중첩(Nesting) · `&` 부모 참조 · 속성 중첩(Property Nesting)

```scss
// 선택자 중첩 + & 부모 참조
.container {
  ul {
    li {
      &.name { color: green; }          // & = 부모(.li) 참조 → li.name
      &.age  { background-color: royalblue; }
    }
  }
}

// 속성 중첩 (Property Nesting)
.container3 .box {
  font: {              // font-weight, font-size, font-family를 한 블록으로 축약
    weight: bold;
    size: 10px;
    family: "sans-serif";
  };
  padding: {
    top: 10px; right: 20px; bottom: 30px; left: 40px;
  };
}
```

| 개념 | 설명 |
|------|------|
| 중첩(Nesting) | 선택자를 부모 안에 중첩하여 HTML 계층 구조를 그대로 표현 |
| `&` 선택자 | 현재 선택자(부모)를 참조. `&.active` → `.btn.active` 로 컴파일 |
| 속성 중첩 | `font:`, `margin:`, `padding:` 등 네임스페이스 접두사를 중첩하여 축약 |

---

### 3) 02 — 변수(`$variable`): 전역 vs 지역

```scss
$size: 200px;          // 전역 변수 — 파일 어디서든 사용 가능

.container {
  $localsize: 100px;   // 지역 변수 — 이 블록 내부에서만 유효

  .item {
    width: $size;       // 전역 변수 참조 가능
    height: $localsize; // 지역 변수 참조 가능
  }
}

// .other { height: $localsize; }  ← 에러: 스코프 밖에서 지역 변수 접근 불가
```

| 변수 종류 | 선언 위치 | 유효 범위 |
|-----------|-----------|-----------|
| 전역 변수 | 최상위 스코프 (`$size: 200px`) | 파일 전체 |
| 지역 변수 | 선택자 블록 내부 (`$localsize: 100px`) | 해당 블록 및 하위 블록 |

## 26일 핵심 정리

---

### 1) Node.js 이벤트 루프 동작 구조

```
[Node.js 단일 스레드]
        │
        ▼
  JavaScript 코드 실행 (Call Stack)
        │
        ├── 동기 작업 → 즉시 처리
        │
        └── 비동기 작업 (I/O, Timer 등)
              │
              ▼
        libuv (OS 수준 비동기 처리)
              │
              ▼ 완료 시 콜백 등록
        Event Queue (Callback Queue)
              │
              ▼ Call Stack이 비어 있으면
        Event Loop → Call Stack으로 콜백 이동
```

> 단일 스레드임에도 높은 동시성을 처리할 수 있는 이유 — I/O는 OS/libuv에 위임하고, 이벤트 루프가 완료된 콜백만 순차 처리

### 2) Parcel v1 vs Parcel v2 핵심 비교

| 항목 | Parcel v1 | Parcel v2 |
|------|-----------|-----------|
| 설치 패키지 | `parcel-bundler` | `parcel` |
| 캐시 디렉토리 | `.cache` | `.parcel-cache` |
| 모듈 방식 | ES Module (`import`) 권장 | CommonJS(`require`) · ESM 모두 지원 |
| 설정 파일 | 거의 불필요 (Zero-config) | Zero-config + `.parcelrc` 확장 |
| 출력 | `dist/파일명.해시.js` | `dist/` |

---

### 3) SCSS 핵심 문법 요약

| 기능 | 문법 | 컴파일 결과 |
|------|------|-------------|
| 선택자 중첩 | `.parent { .child { } }` | `.parent .child { }` |
| `&` 부모 참조 | `.btn { &.active { } }` | `.btn.active { }` |
| 속성 중첩 | `font: { size: 10px; }` | `font-size: 10px;` |
| 전역 변수 | `$var: 값;` (최상위) | CSS 값으로 인라인 치환 |
| 지역 변수 | `$var: 값;` (블록 내부) | 해당 블록 내부에서만 치환 |

---

### 4) 모듈 시스템 전환 흐름

```
브라우저
  └── 모듈 시스템 없음 (전역 스크립트만 지원)
        │
        ▼ Node.js 등장
  CommonJS (require / module.exports)
  — 서버 환경 표준, 동기 로딩
        │
        ▼ ES2015(ES6) 표준화
  ES Module (import / export)
  — 브라우저 + 번들러 표준, 비동기 로딩, 정적 분석 가능
        │
        ▼ 현재
  번들러(Webpack, Parcel, Vite)가
  CommonJS ↔ ESM 변환 처리 → 브라우저에서 실행 가능한 단일 파일 출력
```
