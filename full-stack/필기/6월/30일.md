# 6월 30일 필기 — React 심화 (조건부 렌더링 심화 · Router · 컴포넌트 통신 · 레이아웃 합성 · 스타일링)

## 핵심요약

오늘은 어제 배운 React 기초를 이어 **05_EX부터 09_EX**까지 심화 실습을 진행했다. 05_EX에서는 조건부 렌더링을 IIFE·객체 룩업·switch·옵셔널 체이닝(`?.`)·널 병합(`??`) 등 10가지 방식으로 확장했다. 06/06_EX에서는 `react-router-dom`을 도입해 `BrowserRouter·Routes·Route·Link·useParams·useLocation·useNavigate`를 실습하며 SPA 클라이언트 라우팅을 구현했다. 07/07_EX에서는 컴포넌트 간 데이터 흐름을 세 방식으로 비교했다 — props(부모→자식), callback(자식→부모), Context API(전역). Context API는 `createContext → Provider → useContext` 3단계로 prop drilling 없이 전역 값을 공유한다. 08/08_EX에서는 `children` prop과 컴포넌트 합성으로 Header/Aside/Section/Footer 레이아웃을 구성하고, 슬롯(slot) 패턴·Grid·Alert variant까지 다뤘다. 09/09_EX에서는 인라인 `style`, CSS/SCSS import, `react-bootstrap` 컴포넌트 세 가지 스타일링 방식을 비교하고, 동적 style·조건부 className·BS5 컴포넌트 조합과 레이아웃 매핑까지 완성했다.

### 05_EX — 10가지 방식 (`05조건부랜더링/05_EX.jsx`)

> 포인트: **return의 형태** — 같은 결과라도 여러 방식으로 표현할 수 있다.

| 단계 | 컴포넌트 | 방식 | 핵심 패턴 |
|------|----------|------|-----------|
| 1 | `LoginMessage` | `if/else` | 조건별 각각 `return` |
| 2 | `Greeting` | 삼항 직접 반환 | 래퍼 없이 `return 조건 ? <A/> : <B/>` |
| 3 | `CartList` | `&&` + 인라인 `map` | 빈 배열 안내 + 중괄호 없는 화살표 map |
| 4 | `Grade` | 중첩 삼항 + `&&` | `a ? x : b ? y : z` + 보너스 조건 |
| 5 | `StatusBadge` | IIFE | JSX 내 `if` 분기: `{(() => { ... })()}` |
| 6 | `UserName` | `\|\|` + `&&` | `name \|\| "익명"` · `isAdmin && " (관리자)"` |
| 7 | `SecretBox` | `return null` | 조건 미충족 시 아무것도 렌더하지 않음 |
| 8 | `StatusLabel` | 객체 룩업 | `labels[status] \|\| "알수없음"` |
| 9 | `TrafficLight` | `switch` | `case` 마다 다른 JSX `return` |
| 10 | `Profile` | 옵셔널 체이닝 + `??` | `user?.name ?? "이름 없음"` |

```jsx
// 단계 4 — 중첩 삼항 + && 보너스
export function Grade({ score }) {
  return (
    <p>
      점수 {score}점 - 등급{" "}
      {score >= 90 ? "A" : score >= 80 ? "B" : score >= 70 ? "C" : "F"}
      {score === 100 && <b>(만점!)</b>}
    </p>
  );
}

// 단계 5 — IIFE (JSX 안에서 if 분기)
export function StatusBadge({ status }) {
  return (
    <p>주문 상태 : {(() => {
      if (status === "ready")    return "준비중";
      if (status === "shipping") return "배송중";
      return "배송완료";
    })()}</p>
  );
}

// 단계 8 — 객체 룩업 (switch 대신)
export function StatusLabel({ status }) {
  const labels = { ready: "준비중", shipping: "배송중", done: "배송완료" };
  return <p>주문 상태 : {labels[status] || "알수없음"}</p>;
}

// 단계 10 — 옵셔널 체이닝 + ?? 기본값
export function Profile({ user }) {
  return <p>이름 : {user?.name ?? "이름 없음"}</p>;
}
```

### 1) 06 — 기본 라우팅 (`06Route/`)

**`App.js`**
```jsx
import { BrowserRouter as Router, Routes, Route } from "react-router-dom";

function App() {
  return (
    <Router>
      <Routes>
        <Route path="/"                     element={<Home />} />
        <Route path="/About"                element={<About />} />
        <Route path="/Contact/:name?/:age?" element={<Contact />} />
      </Routes>
    </Router>
  );
}
```

| 라우터 요소 | 역할 |
|-------------|------|
| `BrowserRouter` | HTML5 history API 기반 라우터 |
| `Routes` | 경로 목록 컨테이너 (첫 번째 매칭만 렌더) |
| `Route path element` | 경로와 컴포넌트 매핑 |
| `Link to` | `<a>` 대체, 페이지 새로고침 없이 이동 |
| `:param?` | 선택적 URL 파라미터 (없어도 오류 없음) |

```jsx
// Home.jsx — Link로 네비게이션
import { Link } from "react-router-dom";
<Link to="/About?name=timo&age=100">About</Link>
<Link to="/Contact/hong/40">Contact</Link>

// Contact.jsx — 선택적 파라미터 추출
const { name, age } = useParams();
```

### 2) 06_EX — 라우팅 응용 (`06ROUTE/06_EX.jsx`)

| 단계 | 컴포넌트 | Hook | 학습 내용 |
|------|----------|------|-----------|
| 1 | `Home` | `Link` | 기본 라우트 연결 확인 |
| 2 | `User` | `useParams` | `/user/:id` 에서 `id` 추출 |
| 3 | `Search` | `useLocation` | `location.search` → `URLSearchParams` → `query.get("keyword")` |
| 4 | `Move` | `useNavigate` | `navigate("/user/99")` · `navigate(-1)` 뒤로 가기 |
| 5 | `NotFound` | `path="*"` | 매칭 없는 경로의 404 처리 |

```jsx
// useParams — 경로 파라미터
function User() {
  const { id } = useParams();
  return <h2>USER - 파라미터 id : {id}</h2>;
}

// useLocation + URLSearchParams — 쿼리스트링
function Search() {
  const location = useLocation();
  const keyword = new URLSearchParams(location.search).get("keyword");
  return <h2>SEARCH - 검색어 : {keyword}</h2>;
}

// useNavigate — 프로그래매틱 이동
function Move() {
  const navigate = useNavigate();
  return (
    <>
      <button onClick={() => navigate("/user/99")}>USER 99 로 이동</button>
      <button onClick={() => navigate(-1)}>뒤로 가기</button>
    </>
  );
}

// Routes 구성
<Routes>
  <Route path="/"         element={<Home />} />
  <Route path="/user/:id" element={<User />} />
  <Route path="/search"   element={<Search />} />
  <Route path="/move"     element={<Move />} />
  <Route path="*"         element={<NotFound />} />   {/* 404 */}
</Routes>
```

### 1) 07 — 세 가지 방식 비교

> `07Component간통신/Props/` + `07Component간통신/ContextAPI/`

| 방식 | 방향 | 특징 |
|------|------|------|
| `props` | 부모 → 자식 | 직접 전달, 단계가 깊어지면 prop drilling 발생 |
| `callback` | 자식 → 부모 | 부모가 함수를 만들어 props로 내려주고 자식이 호출 |
| `Context API` | 전역 | prop drilling 없이 트리 어디서든 소비 |

```jsx
// Props + Callback 패턴
const Parent = () => {
  const [name, setName] = useState("홍길동");
  const handleChange = (name) => setName(name);  // 콜백 생성

  return <Son user={name} handleChangeParentName={handleChange} />;
};

// Context API — 3단계
// 1) 생성
const AppContext = React.createContext();

// 2) 제공 (Provider)
const CustomProvider = ({ children }) => {
  const [globalState, setGlobalState] = useState("KOREA_GLOBAL");
  return (
    <AppContext.Provider value={{ globalState, setGlobalState }}>
      {children}
    </AppContext.Provider>
  );
};

// 3) 소비 (useContext) — props 없이 어디서든
const { globalState } = useContext(AppContext);
```

**`App.js`**
```jsx
function App() {
  return (
    <CustomProvider>       {/* Provider로 앱 전체를 감싸 전역 값 제공 */}
      <Parent_01 />
    </CustomProvider>
  );
}
```

### 2) 07_EX — 통신 응용 (`07통신/07_EX.jsx`)

| 단계 | 컴포넌트 | 학습 내용 |
|------|----------|-----------|
| 1 | `Child1` | `props`로 `title` 받아 출력 |
| 2 | `Child2` | 제어 컴포넌트 + `onSend(value)` 콜백으로 부모에 전달 |
| 3 | `ThemeProvider` + `DeepChild` | `createContext → Provider → useContext` + `light/dark` 토글 |

```jsx
// 단계 3 — Context API 한 파일 완성 패턴
const ThemeContext = createContext();

function ThemeProvider({ children }) {
  const [theme, setTheme] = useState("light");
  return (
    <ThemeContext.Provider value={{ theme, setTheme }}>
      {children}
    </ThemeContext.Provider>
  );
}

function DeepChild() {
  const { theme, setTheme } = useContext(ThemeContext);
  return (
    <div>
      현재 테마: <b>{theme}</b>
      <button onClick={() => setTheme(theme === "light" ? "dark" : "light")}>
        테마 전환
      </button>
    </div>
  );
}
```

function Card({ title, footer, children }) {

    <div>
      <div>{title}</div>
      <div>{children}</div>
      <div>{footer}</div>
    </div>
  );
}
// 사용: <Card title="공지사항" footer="2026-06-30">본문</Card>

// 단계 6 — 객체 룩업 variant
function Alert({ type = "info", children }) {
  const colors = { success: "#2f9e44", danger: "#e03131", info: "#1971c2" };
  return (
    <div style={{ color: colors[type], borderLeft: `3px solid ${colors[type]}` }}>
      {children}
    </div>
  );
}

### 1) 조건부 렌더링 방식 선택 기준

```
분기 로직이 복잡하거나 블록이 길다
  └── if/else (단계 1·9)

JSX 내부에서 짧게 분기
  └── 삼항 연산자  isLogin ? <A/> : <B/>  (단계 2·4)

참일 때만 렌더 (false면 아무것도 없음)
  └── && 단축평가  isAdmin && <Badge/>  (단계 3·6)

JSX 안에서 if 문이 꼭 필요할 때
  └── IIFE  {(() => { if(...) return ...; })()}  (단계 5)

여러 상태값 → 레이블 매핑
  └── 객체 룩업  labels[status] || "기본값"  (단계 8)

중첩 객체 접근, null/undefined 방어
  └── 옵셔널 체이닝 + ??  user?.name ?? "이름 없음"  (단계 10)
```

### 2) React Router DOM Hook 비교

| Hook | 용도 | 예시 |
|------|------|------|
| `useParams` | URL 경로 파라미터 추출 | `/user/:id` → `{ id }` |

### 3) 컴포넌트 간 통신 선택 기준

```
부모 → 자식 (1~2단계)
  └── props

자식 → 부모
  └── callback (부모가 함수 생성 → props로 전달 → 자식이 호출)

깊은 트리 전체 공유 (테마·로그인·언어 등)
  └── Context API
      createContext() → Provider value={{...}} → useContext()
```
