# 6월 10일 필기 — Spring Boot / REST Controller + 비동기 클라이언트

## 핵심요약

11_RESTCONTROLLER에서는 `@RestController`로 JSON/XML을 반환하는 REST API를 만들고, 브라우저 비동기 클라이언트 4종(XHR, Fetch, Axios, jQuery)으로 호출하는 전체 흐름을 실습했다. `@Controller`는 뷰 이름을 반환하고, `@RestController`는 `@Controller + @ResponseBody` 합본으로 객체를 JSON/XML로 직렬화해 반환한다. `@Controller` 안에서 특정 메서드만 JSON을 반환하려면 `@ResponseBody`를 해당 메서드에 붙인다. 응답 상태코드를 직접 제어할 때는 `ResponseEntity<T>`를 사용하며 `ResponseEntity.ok(body)`, `ResponseEntity.status(HttpStatus.XXX).body(body)`, `ResponseEntity.noContent().build()` 형태로 생성한다. 다른 출처(Origin)에서 API를 호출하려면 CORS가 필요하며 컨트롤러에 `@CrossOrigin`을 붙여 허용 출처, 메서드, 헤더를 지정한다. REST 핸들러는 1.파라미터 수신(`@RequestBody @Valid DTO`) → 2.유효성 검증(`BindingResult`) → 3.서비스 실행 → 4.응답 반환의 4단계 패턴으로 작성한다. 비동기 클라이언트 비교에서는 XHR이 가장 원시적(콜백 + `JSON.parse` 수동), Fetch는 `async/await`으로 간결, Axios는 Promise 체이닝 + 자동 JSON 처리(단 DELETE body는 `{ data: ... }`에 넣어야 함), jQuery `$.ajax`는 `contentType`/`dataType`/`success`/`error` 옵션으로 처리한다는 차이가 있다.

## Spring Boot — @RestController 기초

---

### 1) @Controller vs @RestController

| | `@Controller` | `@RestController` |
|---|---|---|
| 반환 | 뷰 이름 (JSP/Thymeleaf) | 객체 → JSON/XML 직렬화 |
| `@ResponseBody` | 메서드마다 붙여야 함 | 클래스 전체에 자동 적용 |
| 주 용도 | 서버 사이드 렌더링(SSR) | REST API, 비동기 통신 |

> `@RestController` = `@Controller` + `@ResponseBody` 합본  
> `@Controller` 안에서 일부 메서드만 JSON 반환할 때는 해당 메서드에 `@ResponseBody`만 붙인다

---

### 2) produces / consumes

| 속성 | 방향 | 설명 |
|------|------|------|
| `produces` | 서버 → 클라이언트 | 서버가 반환하는 Content-Type |
| `consumes` | 클라이언트 → 서버 | 서버가 수신할 수 있는 Content-Type |

```java
@GetMapping(value="/getJson", produces = MediaType.APPLICATION_JSON_VALUE)
@PostMapping(value="/add", consumes = MediaType.APPLICATION_JSON_VALUE,
                           produces = MediaType.APPLICATION_JSON_VALUE)
```

### 1) HTTP 메서드 매핑 어노테이션

| 어노테이션 | HTTP 메서드 | 주 용도 |
|---|---|---|
| `@GetMapping` | GET | 조회 |
| `@PostMapping` | POST | 생성 |
| `@PutMapping` | PUT | 전체 수정 |
| `@PatchMapping` | PATCH | 부분 수정 |
| `@DeleteMapping` | DELETE | 삭제 |

### 2) RestTest2Controller — 메모리 기반 CRUD

```java
@RestController
@RequestMapping("/api/items")
public class RestTest2Controller {

    private final Map<Long, ItemResponse> store = new ConcurrentHashMap<>();
    private long seq = 1L;

    // GET /api/items?q=keyword  (목록 + 검색)
    @GetMapping
    public ResponseEntity<List<ItemResponse>> list(@RequestParam(required = false) String q) {
        List<ItemResponse> all = new ArrayList<>(store.values());
        if (q != null && !q.isBlank())
            all.removeIf(i -> !i.getName().toLowerCase().contains(q.toLowerCase()));
        return ResponseEntity.ok(all);
    }

    // GET /api/items/{id}
    @GetMapping("/{id}")
    public ResponseEntity<ItemResponse> get(@PathVariable Long id) {
        ItemResponse found = store.get(id);
        if (found == null) return ResponseEntity.notFound().build();
        return ResponseEntity.ok(found);
    }

    // POST /api/items
    @PostMapping
    public ResponseEntity<ItemResponse> create(@RequestBody ItemRequest req) {
        ItemResponse saved = ItemResponse.builder()
                .id(seq++).name(req.getName()).price(req.getPrice()).build();
        store.put(saved.getId(), saved);
        return ResponseEntity.ok(saved);
    }

    // PUT /api/items/{id}
    @PutMapping("/{id}")
    public ResponseEntity<ItemResponse> update(@PathVariable Long id, @RequestBody ItemRequest req) {
        if (store.get(id) == null) return ResponseEntity.notFound().build();
        ItemResponse updated = ItemResponse.builder()
                .id(id).name(req.getName()).price(req.getPrice()).build();
        store.put(id, updated);
        return ResponseEntity.ok(updated);
    }

    // PATCH /api/items/{id}  (부분 수정 — Map으로 변경 필드만 수신)
    @PatchMapping("/{id}")
    public ResponseEntity<ItemResponse> patch(@PathVariable Long id,
                                              @RequestBody Map<String, Object> patch) {
        ItemResponse found = store.get(id);
        if (found == null) return ResponseEntity.notFound().build();
        String name = patch.containsKey("name") ? String.valueOf(patch.get("name")) : found.getName();
        Integer price = patch.containsKey("price")
                ? ((Number) patch.get("price")).intValue() : found.getPrice();
        ItemResponse updated = ItemResponse.builder().id(id).name(name).price(price).build();
        store.put(id, updated);
        return ResponseEntity.ok(updated);
    }

    // DELETE /api/items/{id}
    @DeleteMapping("/{id}")
    public ResponseEntity<Void> delete(@PathVariable Long id) {
        if (store.remove(id) == null) return ResponseEntity.notFound().build();
        return ResponseEntity.noContent().build();  // 204
    }
}
```

> 실제 DB 없이 `ConcurrentHashMap`으로 메모리 저장. PUT은 전체 교체, PATCH는 전달된 필드만 변경.

## Spring Boot — Memo REST Controller + @CrossOrigin

### 1) @CrossOrigin — CORS 설정

```java
@CrossOrigin(
    originPatterns = {
        "http://localhost:[*]",
        "http://192.168.10.*:[*]"
    },
    allowCredentials = "false",   // TOKEN 기반: false / SESSION 기반: true
    allowedHeaders = {"Content-Type", "application/json"},
    methods = {RequestMethod.GET, RequestMethod.POST, RequestMethod.PUT, RequestMethod.DELETE}
)
```

| 속성 | 설명 |
|------|------|
| `origins` | 허용 출처 정확히 지정 (`http://localhost:5500`) |
| `originPatterns` | 와일드카드 패턴 허용 (`http://localhost:[*]`) |
| `allowCredentials` | 쿠키/세션 포함 여부. SESSION → `true`, TOKEN → `false` |
| `allowedHeaders` | 허용할 요청 헤더 |
| `methods` | 허용할 HTTP 메서드 |

> `origins={"*"}` + `allowCredentials="true"` 조합은 브라우저 보안 정책으로 불가  
> 와일드카드 허용 시 `originPatterns` 사용

---

### 2) REST 핸들러 패턴 (1→2→3→4)

```
1. 파라미터 받기  @RequestBody @Valid MemoDTO memoDTO, BindingResult bindingResult
2. 유효성 검증   bindingResult.hasErrors() → 400 BAD_REQUEST + 오류 Map 반환
3. 서비스 실행   memoService.xxx(memoDTO)
4. 응답 반환    ResponseEntity.status(HttpStatus.OK).body(responseMap)
```

### 3) Memo_addRest_Controller_cors

```java
@Controller
@RequestMapping("/memo")
@CrossOrigin(originPatterns={"http://localhost:[*]",...}, allowCredentials="false", ...)
public class Memo_addRest_Controller_cors {

    @Autowired private MemoService memoService;

    // 뷰 이동 핸들러
    @GetMapping("/rest/xhr")    public String restIndex_xhr()   { return "memo/rest/xhr"; }
    @GetMapping("/rest/fetch")  public String restIndex_fetch() { return "memo/rest/fetch"; }
    @GetMapping("/rest/jquery") public String restIndex_ajax()  { return "memo/rest/jquery"; }
    @GetMapping("/rest/axios")  public String restIndex_axios() { return "memo/rest/axios"; }

    // LIST
    @ResponseBody
    @GetMapping(value="/rest/list",
                consumes=MediaType.APPLICATION_JSON_VALUE, produces=MediaType.APPLICATION_JSON_VALUE)
    public ResponseEntity<Map<String,Object>> rest_list_get(PageDTO pageDTO) throws Exception {
        Map<String,Object> r = memoService.getMemoList(pageDTO);
        Map<String,Object> responseMap = new HashMap<>();
        responseMap.put("page",      r.get("page"));
        responseMap.put("list",      r.get("list"));
        responseMap.put("pageBlock", r.get("pageBlock"));
        return ResponseEntity.status(HttpStatus.OK).body(responseMap);
    }

    // ADD
    @ResponseBody
    @PostMapping(value="/rest/add",
                 consumes=MediaType.APPLICATION_JSON_VALUE, produces=MediaType.APPLICATION_JSON_VALUE)
    public ResponseEntity<Map<String,Object>> memoAddPost_rest(
            @RequestBody @Valid MemoDTO memoDTO, BindingResult bindingResult) throws Exception {
        Map<String,Object> responseMap = new HashMap<>();
        // 2. 유효성
        if (bindingResult.hasErrors()) {
            for (FieldError e : bindingResult.getFieldErrors())
                responseMap.put(e.getField(), e.getDefaultMessage());
            return ResponseEntity.status(HttpStatus.BAD_REQUEST).body(responseMap);
        }
        // 3. 서비스
        memoService.memoRegistration(memoDTO);
        // 4. 응답
        responseMap.put("message", "메모추가 성공!");
        return ResponseEntity.status(HttpStatus.OK).body(responseMap);
    }

    // UPDATE — PUT
    @ResponseBody
    @PutMapping(value="/rest/update", consumes=..., produces=...)
    public ResponseEntity<Map<String,Object>> memoUpdatePut_rest(
            @RequestBody @Valid MemoDTO memoDTO, BindingResult bindingResult) throws Exception { ... }

    // DELETE
    @ResponseBody
    @DeleteMapping(value="/rest/delete", consumes=..., produces=...)
    public ResponseEntity<Map<String,Object>> memoDelete_rest(@RequestBody MemoDTO dto) throws Exception {
        memoService.removeMemo(dto.getId());
        Map<String,Object> responseMap = new HashMap<>();
        responseMap.put("message", "메모삭제 성공!");
        return ResponseEntity.status(HttpStatus.OK).body(responseMap);
    }
}
```

### 1) XHR (XMLHttpRequest) — xhr.html

```javascript
// LIST
function getList() {
    var httpRequest = new XMLHttpRequest();
    httpRequest.onreadystatechange = function() {
        if (httpRequest.readyState == XMLHttpRequest.DONE && httpRequest.status == 200) {
            const data = JSON.parse(httpRequest.responseText);  // 수동 파싱
            console.log("list", data.list);
        }
    };
    httpRequest.open("GET", `/memo/rest/list?pageNo=${pageNo}&amount=${amount}`, true);
    httpRequest.setRequestHeader("Content-Type", "application/json");
    httpRequest.send();
}

// ADD / UPDATE / DELETE — body 전송
httpRequest.open("POST", `/memo/rest/add`, true);
httpRequest.setRequestHeader("Content-Type", "application/json");
httpRequest.send(JSON.stringify({title, text, writer}));
```

> `readyState == 4(XMLHttpRequest.DONE)` && `status == 200` 두 조건 모두 확인  
> `JSON.parse(httpRequest.responseText)` 로 수동 파싱

### 2) Fetch API — fetch.html

```javascript
// LIST
async function getList() {
    const opt = { method: "GET", headers: {"Content-Type": "application/json"} };
    const response = await fetch(`/memo/rest/list?pageNo=${pageNo}&amount=${amount}`, opt);
    const data = await response.json();   // 자동 JSON 파싱
    console.log(data);
}

// ADD
async function addMemo() {
    const opt = {
        method: "POST",
        headers: {"Content-Type": "application/json"},
        body: JSON.stringify({title, text, writer})
    };
    const response = await fetch(`/memo/rest/add`, opt);
    const data = await response.json();
}

// DELETE — body 포함 가능
const opt = {
    method: "DELETE",
    headers: {"Content-Type": "application/json"},
    body: JSON.stringify({id})
};
await fetch(`/memo/rest/delete`, opt);
```

> `async/await` 기반. `response.json()`으로 즉시 파싱.  
> XHR보다 코드가 간결하고 Promise 체이닝 가능.

```javascript
// LIST — axios.get(url, config)
axios.get(`/memo/rest/list?pageNo=${pageNo}&amount=${amount}`,

     .then(resp => console.log(resp))
     .catch(error => console.log(error));

axios.post(`/memo/rest/add`, JSON.stringify({title, text, writer}),
           { headers: {"Content-Type": "application/json"} })

axios.put(`/memo/rest/update`, JSON.stringify({id, title, text, writer}),
          { headers: {"Content-Type": "application/json"} })
     .then(resp => console.log(resp));

// DELETE — body는 config의 data에 넣어야 함
axios.delete(`/memo/rest/delete`,
             { headers: {"Content-Type": "application/json"}, data: JSON.stringify({id}) })
     .then(resp => console.log(resp));
```

> **Axios DELETE 주의**: `axios.delete(url, { data: body })` 형태로 body 전달  
> 다른 메서드처럼 두 번째 인자로 body를 넣으면 안 됨

## Spring Boot — 11 핵심 정리

---

### 1) 4종 비교

| | XHR | Fetch | Axios | jQuery $.ajax |
|---|---|---|---|---|
| 방식 | 콜백 | async/await | Promise/.then | 콜백 |
| JSON 파싱 | `JSON.parse` 수동 | `response.json()` 자동 | 자동 | `dataType:"json"` 자동 |
| DELETE body | `send(jsonBody)` | `body: jsonBody` | `{ data: jsonBody }` | `data: jsonBody` |
| 오류 처리 | `status` 직접 체크 | `response.ok` 체크 | `.catch()` | `error:` 콜백 |
| CDN 필요 | 없음 | 없음 | 필요 | 필요 |
| 코드량 | 많음 | 보통 | 적음 | 보통 |

---

### 2) 전체 흐름

```
브라우저 (xhr.html / fetch.html / axios.html / jquery.html)
    │  Content-Type: application/json + JSON body
    ▼
Memo_addRest_Controller_cors  (@Controller + @ResponseBody + @CrossOrigin)
    │  1. @RequestBody → MemoDTO 역직렬화
    │  2. @Valid + BindingResult → 유효성 검증 실패 시 400 반환
    │  3. memoService.xxx()
    │  4. ResponseEntity.status(200).body(responseMap)
    ▼
MemoServiceImpl  (@Transactional(transactionManager="jpaTransactionManager"))
    ▼
MemoRepository (JpaRepository)
    │  findAll(Pageable) / save() / findById() / deleteById()
    ▼
DB (memo 테이블)
```

---

### 3) 핵심 포인트 요약

| 항목 | 설명 |
|------|------|
| `@RestController` | `@Controller + @ResponseBody`. 모든 메서드 JSON 반환 |
| `@ResponseBody` | `@Controller` 안 특정 메서드만 JSON 반환할 때 |
| `ResponseEntity<T>` | HTTP 상태코드 + body 직접 제어 |
| `produces` | 서버가 반환하는 Content-Type 지정 |
| `consumes` | 서버가 수신할 수 있는 Content-Type 지정 |
| `@CrossOrigin` | CORS 허용. 컨트롤러 또는 메서드 레벨 |
| `allowCredentials` | SESSION 기반 `true`, TOKEN 기반 `false` |
| REST 핸들러 패턴 | 1.수신 → 2.유효성 → 3.서비스 → 4.응답 |
| Axios DELETE body | `axios.delete(url, { data: body })` |
| XHR 응답 처리 | `JSON.parse(httpRequest.responseText)` 수동 |

---
