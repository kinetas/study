# 6월 15일 필기 — Spring Boot / 네이버(Naver) 외부 API 연동

## 핵심요약

12_RESTFULAPI의 네이버 외부API 파트에서는 Naver의 2가지 서비스(책 검색, 로그인) 연동 패턴을 실습했다. **네이버 책 검색(NaverSearch)**은 `X-Naver-Client-Id` / `X-Naver-Client-Secret` 헤더 인증 방식으로 `openapi.naver.com/v1/search/book.json`을 GET 호출해 JSON 문자열을 그대로 응답한다. 카카오 검색 API가 `Authorization: KakaoAK {키}` 단일 헤더를 사용하는 것과 달리, 네이버는 두 개의 전용 헤더로 인증한다. **네이버 로그인(NaverLogin)**은 OAuth 2.0 인가코드 방식으로 ① `nid.naver.com/oauth2.0/authorize`로 인가코드 요청 → ② `nid.naver.com/oauth2.0/token`으로 액세스 토큰 교환 → ③ `openapi.naver.com/v1/nid/me`로 프로필 조회 3단계 흐름으로 진행된다. 프로필 응답의 `"response"` 키는 `@JsonProperty(value = "response")`로 `profile` 필드에 매핑한다. 로그아웃은 `nid.naver.com/nidlogin.logout`으로 redirect 하는 동기 방식과, 10×10 팝업창을 열어 네이버 세션을 종료한 뒤 `setTimeout`으로 팝업을 닫는 팝업 방식 두 가지를 제공한다.

## Spring Boot — NaverSearch 책 검색

---

### 1) 컨트롤러

```java
@Slf4j
@Controller
@RequestMapping("/Naver/search")
public class NaverSearchController {

    private String CLIENT_ID = "";
    private String CLEINT_SECRET = "";

    @GetMapping("/book/{keyword}")
    @ResponseBody
    public String search(@PathVariable String keyword) {
        log.info("GET /Naver/search...{}", keyword);
        String url = "https://openapi.naver.com/v1/search/book.json?query=" + keyword;

        HttpHeaders header = new HttpHeaders();
        header.add("X-Naver-Client-Id", CLIENT_ID);
        header.add("X-Naver-Client-Secret", CLEINT_SECRET);

        HttpEntity entity = new HttpEntity<>(header);

        RestTemplate restTemplate = new RestTemplate();
        ResponseEntity<String> response =
                restTemplate.exchange(url, HttpMethod.GET, entity, String.class);

        return response.getBody();
    }
}
```

## Spring Boot — NaverLogin OAuth 2.0 로그인

---

### 1) 전체 엔드포인트

| 메서드 | 경로 | 역할 |
|--------|------|------|
| GET | `/Naver/login` | 네이버 로그인 페이지로 redirect |
| GET | `/Naver/callback` | 인가코드 수신 → access_token 교환 |
| GET | `/Naver` | 프로필 조회 → main.html 렌더링 |
| GET | `/Naver/logout` | 네이버 세션 로그아웃 (동기 redirect) |

---

### 2) 로그인 흐름

```
1. GET /Naver/login
   └─ redirect: nid.naver.com/oauth2.0/authorize
                ?response_type=code&client_id=...&state=STATE_STRING&redirect_uri=...

2. GET /Naver/callback?code=인가코드&state=xxx
   └─ POST nid.naver.com/oauth2.0/token
         params: grant_type=authorization_code, client_id, client_secret, code, state
   └─ NaverTokenResponse(access_token) 저장
   └─ redirect: /Naver

3. GET /Naver
   └─ POST openapi.naver.com/v1/nid/me
         header: Authorization: Bearer {access_token}
   └─ NaverProfileResponse 저장 → Model에 담아 Naver/main.html 렌더링
```

---

### 3) 컨트롤러 전체

```java
@Controller @Slf4j @RequestMapping("/Naver")
public class NaverLoginController {

    private String CLIENT_ID = "";
    private String CLEINT_SECRET = "";
    private String CALLBACK_URL = "http://127.0.0.1:8080/Naver/callback";

    private NaverTokenResponse naverTokenResponse;
    private NaverProfileResponse naverProfileResponse;

    // 1단계: 네이버 로그인 페이지로 리다이렉트
    @GetMapping("/login")
    public String login() {
        return "redirect:https://nid.naver.com/oauth2.0/authorize"
                + "?response_type=code"
                + "&client_id=" + CLIENT_ID
                + "&state=STATE_STRING"
                + "&redirect_uri=" + CALLBACK_URL;
    }

    // 2단계: 인증 코드 수신 → access_token 교환
    @GetMapping("/callback")
    public String callback(String code, String state, String error, String error_description) {
        String url = "https://nid.naver.com/oauth2.0/token";

        MultiValueMap<String, String> params = new LinkedMultiValueMap<>();
        params.add("grant_type",   "authorization_code");
        params.add("client_id",    CLIENT_ID);
        params.add("client_secret", CLEINT_SECRET);
        params.add("code",         code);
        params.add("state",        state);

        HttpHeaders header = new HttpHeaders();
        HttpEntity<MultiValueMap<String, String>> entity = new HttpEntity<>(params, header);

        RestTemplate restTemplate = new RestTemplate();
        ResponseEntity<NaverTokenResponse> response =
                restTemplate.exchange(url, HttpMethod.POST, entity, NaverTokenResponse.class);

        this.naverTokenResponse = response.getBody();
        return "redirect:/Naver";
    }

    // 3단계: access_token으로 프로필 조회
    @GetMapping
    public String main(Model model) {
        String url = "https://openapi.naver.com/v1/nid/me";

        HttpHeaders header = new HttpHeaders();
        header.add("Authorization", "Bearer " + naverTokenResponse.getAccess_token());
        HttpEntity entity = new HttpEntity(header);

        RestTemplate restTemplate = new RestTemplate();
        ResponseEntity<NaverProfileResponse> response =
                restTemplate.exchange(url, HttpMethod.POST, entity, NaverProfileResponse.class);

        this.naverProfileResponse = response.getBody();
        model.addAttribute("naverTokenResponse",  naverTokenResponse);
        model.addAttribute("naverProfileResponse", naverProfileResponse);
        return "Naver/main";
    }

    // 4단계: 동기 로그아웃
    @GetMapping("logout")
    public String logout() {
        return "redirect:https://nid.naver.com/nidlogin.logout"
                + "?returl=http://127.0.0.1:8080/Naver/login";
    }
}
```

### 4) 내부 DTO 클래스 (static inner class)

```java
@Data
private static class NaverTokenResponse {
    public String access_token;   // API 호출용 토큰
    public String refresh_token;  // 갱신 토큰
    public String token_type;     // "bearer"
    public String expires_in;     // 유효 시간(초)
}

@Data
private static class Profile {
    public String id;
    public String nickname;
    public String email;
    public String name;
}

@Data
private static class NaverProfileResponse {
    public String resultcode;     // "00" 성공
    public String message;        // "success"
    @JsonProperty(value = "response")
    public Profile profile;       // 네이버 API 응답의 "response" 키 → profile 매핑
}
```

> `@JsonProperty(value = "response")` — 네이버 프로필 API 응답의 최상위 키가 `"response"`이므로 필드명 `profile`과 매핑이 필요

### 6) 로그아웃 2가지 비교

| 방식 | 구현 | 동작 |
|------|------|------|
| 동기 redirect | `GET /Naver/logout` → redirect `nidlogin.logout?returl=...` | 네이버 로그아웃 페이지 이동 후 `returl`로 복귀 |
| 팝업창 JS | `window.open()` + `setTimeout` | 현재 페이지 유지하며 팝업으로 세션 종료, 2.5초 후 팝업 닫고 앱 로그인 이동 |

```javascript
// 팝업 방식 로그아웃
async function requestNaverLogout() {
    const popup = window.open(
        "https://nid.naver.com/nidlogin.logout",
        "naverLogout",
        "width=10,height=10,left=0,top=0"
    );
    if (popup) { window.focus(); }

    setTimeout(function () {
        if (popup) popup.close();
        alert("로그아웃 완료");
        location.href = "Naver/login";
    }, 2500);
}
```

## Spring Boot — 15 핵심 정리

---

### 1) 네이버 API 2종 비교

| 컨트롤러 | 경로 | 인증 방식 | HTTP 메서드 |
|----------|------|-----------|-------------|
| NaverSearch | `/Naver/search/book/{keyword}` | `X-Naver-Client-Id` + `X-Naver-Client-Secret` 헤더 | GET |
| NaverLogin | `/Naver/login ~` | OAuth 2.0 (access_token) | POST |

---

### 2) RestTemplate 공통 호출 패턴

```
네이버 검색 (헤더 인증 + GET)
  1. HttpHeaders: X-Naver-Client-Id, X-Naver-Client-Secret
  2. HttpEntity<>(header)  — body 없음
  3. restTemplate.exchange(url, HttpMethod.GET, entity, String.class)

네이버 토큰 교환 (form-data + POST)
  1. MultiValueMap<String, String> params: grant_type, client_id, client_secret, code, state
  2. HttpHeaders: 별도 헤더 없음 (기본값)
  3. HttpEntity<MultiValueMap<String, String>> entity = new HttpEntity<>(params, header)
  4. restTemplate.exchange(url, HttpMethod.POST, entity, NaverTokenResponse.class)

네이버 프로필 조회 (Bearer + POST)
  1. HttpHeaders: Authorization: Bearer {access_token}
  2. HttpEntity<>(header)  — body 없음
  3. restTemplate.exchange(url, HttpMethod.POST, entity, NaverProfileResponse.class)
```

---

### 3) 네이버 vs 카카오 API 비교

| 항목 | 네이버 | 카카오 |
|------|--------|--------|
| 검색 인증 | `X-Naver-Client-Id` / `X-Naver-Client-Secret` 헤더 | `Authorization: KakaoAK {REST_API_KEY}` |
| 로그인 OAuth | `nid.naver.com/oauth2.0/authorize` | `kauth.kakao.com/oauth/authorize` |
| 토큰 교환 | `nid.naver.com/oauth2.0/token` | `kauth.kakao.com/oauth/token` |
| 프로필 조회 | `openapi.naver.com/v1/nid/me` | `kapi.kakao.com/v2/user/me` |
| 프로필 인증 헤더 | `Authorization: Bearer {access_token}` | `Authorization: Bearer {access_token}` |
| 로그아웃 URL | `nid.naver.com/nidlogin.logout` | `kauth.kakao.com/oauth/logout` |

```

    │  redirect → nid.naver.com/oauth2.0/authorize?...&response_type=code

    │  POST nid.naver.com/oauth2.0/token → access_token 수령

    │  POST openapi.naver.com/v1/nid/me (Bearer access_token)
    │  naverProfileResponse 저장 → Model에 담기
    ▼

```

### 5) 핵심 포인트 요약

| 항목 | 설명 |
|------|------|
| 검색 API 인증 | `X-Naver-*` 헤더 2개 — `Authorization` 헤더 사용 안 함 |
| `@JsonProperty("response")` | 네이버 프로필 응답의 `"response"` 키를 `profile` 필드에 매핑 |
| 프로필 조회 메서드 | `HttpMethod.POST` — GET이 아님에 주의 |
| `NaverTokenResponse` | 컨트롤러 필드에 저장 → 이후 프로필 조회 시 `access_token` 재사용 |
| 로그아웃 팝업 | `window.open()` 10×10 크기 + `setTimeout(2500)` 패턴 |
| `returl` 파라미터 | 동기 로그아웃 후 돌아올 URL 지정 — 네이버 로그아웃 URL에 쿼리스트링으로 전달 |
