# 6월 12일 필기 — Spring Boot / 카카오(Kakao) 외부 API 연동

## 핵심요약

12_RESTFULAPI의 카카오 외부API 파트에서는 Kakao의 4가지 서비스(지도, 로그인, 채널, 페이) 연동 패턴을 실습했다. **카카오 지도(C01)**는 JS SDK(`dapi.kakao.com`)를 HTML에서 직접 로드하여 키워드 장소 검색, 마커, 인포윈도우, 페이지네이션을 프론트엔드 JavaScript로 구현한다. **카카오 로그인(C02)**는 OAuth 2.0 인가코드 방식으로 ① `kauth.kakao.com/oauth/authorize`로 인가코드 요청 → ② `kauth.kakao.com/oauth/token`으로 액세스 토큰 교환 → ③ `kapi.kakao.com/v2/user/me`로 프로필 조회 3단계 흐름으로 진행되며, 로그아웃은 서비스 토큰만 만료(`/v1/user/logout`), 앱 연결 해제(`/v1/user/unlink`), 카카오 계정까지 함께 로그아웃(redirect) 3가지로 나뉜다. 권한 추가 인가(`scope=talk_message,friends`)를 별도로 받아 나에게/친구에게 카카오톡 메시지 전송과 친구 목록 조회도 실습했다. **카카오 채널(C03)**은 Kakao JS SDK의 `Kakao.Channel.createAddChannelButton()`으로 채널 추가 버튼을 동적 생성한다. **카카오페이(C04)**는 `Content-Type: application/json` + `Authorization: SECRET_KEY {키}` 헤더로 결제 준비 API를 호출하며, 결제 성공/취소/실패 콜백 URL을 파라미터로 전달한다. 카카오 로그인·메시지는 `Bearer {access_token}`, 카카오페이는 `SECRET_KEY {secret_key}` 형식으로 인증 헤더가 다름에 주의한다.

```html
<!-- SDK 로드 — appkey 발급 필요 -->
<script src="//dapi.kakao.com/v2/maps/sdk.js?appkey=&libraries=services"></script>
<script>
    var map = new kakao.maps.Map(mapContainer, {
        center: new kakao.maps.LatLng(37.566826, 126.9786567),  // 서울 중심
        level: 3
    });

    ps.keywordSearch(keyword, placesSearchCB);

        if (status === kakao.maps.services.Status.OK) {
            displayPlaces(data);        // 마커 + 목록 표출
            displayPagination(pagination);
        }
    }
</script>
```

## Spring Boot — C02 카카오 로그인 (OAuth 2.0)

---

### 1) 전체 엔드포인트

| 메서드 | 경로 | 역할 |
|--------|------|------|
| GET | `/Kakao/login` | 카카오 인가 코드 요청 (redirect → kakao) |
| GET | `/Kakao/callback` | 인가 코드 수신 → 액세스 토큰 교환 |
| GET | `/Kakao` | 사용자 프로필 조회 → main.html 렌더링 |
| GET | `/Kakao/logout1` | 서비스에서만 로그아웃 (`/v1/user/logout`) |
| GET | `/Kakao/logout2` | 카카오 연결 해제 (`/v1/user/unlink`) |
| GET | `/Kakao/logout3` | 카카오 계정과 함께 로그아웃 (redirect → kakao) |
| GET | `/Kakao/getMessageCode` | 메시지·친구 권한 추가 인가 (`scope=talk_message,friends`) |
| GET | `/Kakao/message/me/{message}` | 나에게 카카오톡 메시지 전송 |
| GET | `/Kakao/message/friends/{message}` | 친구 전체에게 카카오톡 메시지 전송 |
| GET | `/Kakao/friends` | 카카오 친구 목록 조회 |

---

### 2) 로그인 흐름

```
1. GET /Kakao/login
   └─ redirect: kauth.kakao.com/oauth/authorize?client_id=...&response_type=code

2. GET /Kakao/callback?code=인가코드
   └─ POST kauth.kakao.com/oauth/token
         params: grant_type=authorization_code, client_id, redirect_uri, code
   └─ KakaoTokenResponse(access_token) 저장
   └─ redirect: /Kakao

3. GET /Kakao
   └─ GET kapi.kakao.com/v2/user/me
         header: Authorization: Bearer {access_token}
   └─ KakaoProfileResponse 저장 → Model에 담아 Kakao/main.html 렌더링
```

---

### 3) 토큰 교환 코드 패턴

```java
// callback — 인가코드 → 액세스 토큰
String url = "https://kauth.kakao.com/oauth/token";

MultiValueMap<String, String> params = new LinkedMultiValueMap<>();
params.add("grant_type",   "authorization_code");
params.add("client_id",    CLIENT_ID);
params.add("redirect_uri", REDIRECT_URI);
params.add("code",         code);

HttpHeaders header = new HttpHeaders();
header.add("Content-Type", "application/x-www-form-urlencoded;charset=utf-8");

HttpEntity<MultiValueMap<String, String>> entity = new HttpEntity<>(params, header);
RestTemplate restTemplate = new RestTemplate();
ResponseEntity<KakaoTokenResponse> response =
        restTemplate.exchange(url, HttpMethod.POST, entity, KakaoTokenResponse.class);

this.kakaoTokenResponse = response.getBody();
```

---

### 4) 내부 DTO 클래스 (static inner class)

```java
@Data
private static class KakaoTokenResponse {
    public String access_token, token_type, refresh_token, scope;
    public int expires_in, refresh_token_expires_in;
}

@Data
private static class KakaoProfileResponse {
    public long id;
    public Date connected_at;
    public Properties properties;      // nickname, profile_image, thumbnail_image
    public KakaoAccount kakao_account; // email, profile{nickname, ...}
}

@Data
private static class KakaoFriendsResponse {
    public ArrayList<Element> elements;  // uuid, profile_nickname, allowed_msg ...
    public int total_count, favorite_count;
    public Object after_url;
}
```

### 6) 메시지 전송

```java
// 나에게 메시지 전송
// POST kapi.kakao.com/v2/api/talk/memo/default/send
JSONObject template_Object = new JSONObject();
template_Object.put("object_type", "text");
template_Object.put("text",        message);
template_Object.put("link",        new JSONObject());
template_Object.put("button_title", ".");

params.add("template_object", template_Object.toJSONString());
header.add("Authorization", "Bearer " + kakaoTokenResponse.getAccess_token());
```

```java
// 친구에게 메시지 전송
// POST kapi.kakao.com/v1/api/talk/friends/message/default/send
String[] uuids = kakaoFriendsResponse.getElements()
        .stream().map(el -> el.getUuid())
        .collect(Collectors.toList()).toArray(String[]::new);

JSONArray receiver_uuids = new JSONArray();
for (String uuid : uuids) receiver_uuids.add(uuid);

params.add("receiver_uuids",    receiver_uuids);
params.add("template_object",   template_Object.toJSONString());
```

> 친구 메시지를 보내려면 `/Kakao/getMessageCode`로 `scope=talk_message,friends` 권한을 별도 인가받아야 함

## Spring Boot — C04 카카오페이 결제 준비

---

### 1) 컨트롤러

```java
@Controller @Slf4j @RequestMapping("/Kakao/pay")
public class C04KakaoPayController {

    private String SECRET_KEY = "";

    @GetMapping
    @ResponseBody
    public void req() {
        String url = "https://open-api.kakaopay.com/online/v1/payment/ready";

        HttpHeaders header = new HttpHeaders();
        header.add("Authorization", "SECRET_KEY " + SECRET_KEY);  // Bearer 아님!
        header.add("Content-Type",  "application/json");

        JSONObject params = new JSONObject();
        params.put("cid",              "TC0ONETIME");  // 테스트용 가맹점 코드
        params.put("partner_order_id", "shop_1111");
        params.put("partner_user_id",  "shop_p_1111");
        params.put("item_name",        "초코파이");
        params.put("quantity",         "1");
        params.put("total_amount",     "2200");
        params.put("tax_free_amount",  "200");
        params.put("approval_url",     "http://127.0.0.1:8080/Kakao/pay/success");
        params.put("cancel_url",       "http://127.0.0.1:8080/Kakao/pay/cancel");
        params.put("fail_url",         "http://127.0.0.1:8080/Kakao/pay/fail");

        HttpEntity<JSONObject> entity = new HttpEntity<>(params, header);
        RestTemplate restTemplate = new RestTemplate();
        ResponseEntity<String> response =
                restTemplate.exchange(url, HttpMethod.POST, entity, String.class);

        System.out.println(response.getBody());
    }

    @GetMapping("/success") @ResponseBody
    public void success() { log.info("결제 성공 후 이동되는 위치"); }

    @GetMapping("/cancel")  @ResponseBody
    public void cancel()  { log.info("결제 취소 후 이동되는 위치"); }

    @GetMapping("/fail")    @ResponseBody
    public void fail()    { log.info("결제 실패 후 이동되는 위치"); }
}
```

---

### 2) 결제 준비 파라미터

| 파라미터 | 예시 값 | 설명 |
|----------|---------|------|
| `cid` | `TC0ONETIME` | 테스트용 가맹점 코드 (단건 결제) |
| `partner_order_id` | `shop_1111` | 가맹점 주문 번호 |
| `partner_user_id` | `shop_p_1111` | 가맹점 회원 ID |
| `item_name` | `초코파이` | 상품명 |
| `quantity` | `1` | 수량 |
| `total_amount` | `2200` | 총 결제 금액 (원) |
| `tax_free_amount` | `200` | 비과세 금액 |
| `approval_url` | `/Kakao/pay/success` | 결제 성공 후 리디렉션 URL |
| `cancel_url` | `/Kakao/pay/cancel` | 결제 취소 후 리디렉션 URL |
| `fail_url` | `/Kakao/pay/fail` | 결제 실패 후 리디렉션 URL |

> `Content-Type: application/json` → body를 `JSONObject`로 전달 (form-data 아님)  
> Authorization 헤더: `SECRET_KEY {시크릿 키}` — 카카오페이 전용 형식 (Bearer와 다름)

### 2) RestTemplate 공통 호출 패턴

```
카카오 로그인/메시지/친구 (form-data)
  1. MultiValueMap<String, String> params 구성
  2. HttpHeaders: Authorization: Bearer {access_token}
                  Content-Type: application/x-www-form-urlencoded;charset=utf-8
  3. HttpEntity<MultiValueMap<String, String>> entity = new HttpEntity<>(params, header)
  4. restTemplate.exchange(url, HttpMethod.POST, entity, 응답타입.class)

카카오페이 (JSON)
  1. JSONObject params 구성
  2. HttpHeaders: Authorization: SECRET_KEY {secret_key}
                  Content-Type: application/json
  3. HttpEntity<JSONObject> entity = new HttpEntity<>(params, header)
  4. restTemplate.exchange(url, HttpMethod.POST, entity, String.class)
```

### 5) 핵심 포인트 요약

| 항목 | 설명 |
|------|------|
| `MultiValueMap` vs `JSONObject` | form-data 전송은 `MultiValueMap`, JSON 전송은 `JSONObject` |
| `KakaoTokenResponse` | 컨트롤러 필드에 저장 → 이후 API 호출 시 재사용 |
| 메시지 권한 | 기본 로그인 토큰으로는 불가 — `getMessageCode`로 `scope` 추가 인가 필요 |
| 친구 uuid | `KakaoFriendsResponse.elements`에서 stream으로 추출 → `receiver_uuids`로 전달 |
| 카카오페이 CID | `TC0ONETIME` = 테스트용 단건 결제 코드 |
| JS SDK vs REST API | 지도·채널은 JS SDK(프론트), 로그인·페이는 백엔드 RestTemplate |
