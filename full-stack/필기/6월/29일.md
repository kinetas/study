# 6월 29일 필기 — React 기초 (JSX · Hooks · 이벤트 핸들링 · 조건부 렌더링)

## 핵심요약

오늘은 **React 기초 전반**을 실습했다. React는 Facebook이 만든 컴포넌트 기반 UI 라이브러리로, Virtual DOM과 단방향 데이터 흐름이 핵심이다. 실습은 `start/` 폴더의 단계별 프로젝트로 진행됐다. 01은 CRA 기본 구조를 확인했고, 02/02_EX에서는 JSX 문법(props·map·이벤트 핸들러)을 다뤘다. 03/03_EX에서는 `useState`와 `useEffect` 두 Hook을 학습했고, `useEffect`의 의존성 배열(`[]` / `[state]` / 없음)에 따른 실행 시점 차이를 이해했다. 04/04_EX에서는 `onClick·onMouse·onChange·onKey·onSubmit·onScroll` 6종 이벤트를 실습했으며, `e.preventDefault()`와 스프레드 연산자를 통한 불변성 유지 패턴을 익혔다. 05에서는 `if/else`, 삼항 연산자 `? :`, `&&` 단축평가 세 가지 조건부 렌더링 방식을 비교했다.

## React — 개요

---

### 1) React란

React는 웹 브라우저의 **사용자 인터페이스(UI)**를 구축하기 위한 오픈 소스 JavaScript 라이브러리다. Facebook에서 개발·유지 관리하며, 주로 **단일 페이지 애플리케이션(SPA)**에서 빠르고 효율적인 UI 업데이트를 위해 활용된다.

---

### 2) 주요 특징

| # | 특징 | 설명 |
|---|------|------|
| 1 | 컴포넌트 기반 | UI를 독립적·재사용 가능한 단위로 분할 |
| 2 | Virtual DOM | 최소한의 실제 DOM 변경으로 성능 향상 |
| 3 | 단방향 데이터 흐름 | 부모 → 자식 방향으로만 데이터 흐름, 예측 가능한 상태 관리 |
| 4 | JSX | JavaScript + HTML 혼합 문법, 가독성·유지보수성 향상 |
| 5 | React Hooks | 함수형 컴포넌트에서 상태관리 및 생명주기 활용 |

### 2) 02 — JSX 핵심 패턴 (`02JSX/01_Basic.jsx`)

| 요소 | 내용 |
|------|------|
| `Element1` | 화살표 함수로 JSX 반환 |
| `Element2` | `function` 키워드로 JSX 반환 |
| `Element3` | `props` 객체로 값 전달, 조건 분기 (`ROLE_ADMIN` / `ROLE_USER`) |
| `Element4` | props **구조분해할당**으로 `{auth, name}` 직접 받기 |
| `Element5` | `props.list` 배열을 `map()`으로 `<li>` 목록 렌더링 |

```jsx
// props 구조분해할당
export function Element4({ auth, name }) {
  if (auth === "ROLE_ADMIN") return <h2>관리자님 환영합니다, NAME={name}</h2>;
  else if (auth === "ROLE_USER") return <h2>유저님 환영합니다, NAME={name}</h2>;
}

// 배열 map() 렌더링
export const Element5 = (props) => (
  <ul>
    {props.list.map((item, idx) => <li key={idx}>{item}</li>)}
  </ul>
);
```

### 3) 02_EX — JSX 응용 연습 (쇼핑몰 테마)

| 단계 | 컴포넌트 | 학습 내용 |
|------|----------|-----------|
| 1 | `ShopBanner`, `ShopNotice` | 화살표 함수 / `function` 두 방식으로 컴포넌트 생성 |
| 2 | `ProductLine` | `props.name`, `props.price` 출력 |
| 3 | `DiscountLine` | 구조분해할당 + 할인가 계산 (`price * (100 - rate) / 100`) |

```jsx
// 단계 5 — map + reduce 패턴
export function Cart({ items }) {
  const total = items.reduce((sum, item) => sum + item.price * item.qty, 0);
  return (
    <>
      <ul>{items.map(item => <li key={item.id}>{item.name} x {item.qty} = {item.price * item.qty}</li>)}</ul>
      <p>합계 : {total}</p>
    </>
  );
}
```

## Hooks 실습

---

### 1) 03 — `useState` / `useEffect`

```jsx
import { useState, useEffect } from "react";

const MyComponent = () => {
  const [count, setCount] = useState(1);

  useEffect(() => {
    console.log("init setting..");
  }, []);           // [] → 마운트 시 1회만 실행

  useEffect(() => {
    console.log("count changed:", count);
  }, [count]);      // [count] → count 변경 시마다 실행

  return <button onClick={() => setCount(count + 1)}>BTN Count : {count}</button>;
};
```

| Hook | 의존성 배열 | 실행 시점 |
|------|------------|-----------|
| `useEffect(() => {}, [])` | `[]` | 마운트 시 **1회** |
| `useEffect(() => {}, [count])` | `[count]` | `count` 변경 **시마다** |
| `useEffect(() => {})` | 없음 | **매 렌더링** 후 |

## 조건부 렌더링

---

### 05 — 3가지 방식 비교 (`05ConditionRandering/`)

| 파일 | 방식 | 특징 |
|------|------|------|
| `Component01.jsx` | `if / else` | 복잡한 분기 처리에 적합 |
| `Component02.jsx` | 삼항 연산자 `? :` | JSX 내부 인라인 분기, 가장 간결 |
| `Component03.jsx` | `&&` 단축평가 | 조건이 참일 때만 렌더링 (null 방어) |

```jsx
// if/else
const Component01 = ({ isAuth }) => {
  if (isAuth) return <div><h1>인증 완료 상태</h1></div>;
  else         return <div><h1>로그인이 필요합니다</h1></div>;
};

// 삼항 연산자 (가장 간결)
const Component02 = ({ isAuth }) =>
  isAuth ? <div>인증 완료 상태</div> : <div>로그인이 필요합니다</div>;

// && 단축평가 (undefined/null 방어)
const Component03 = ({ items }) => (
  <ul>
    {items && items.map((item, idx) => <li key={idx}>{item}</li>)}
  </ul>
);
```

## 29일 핵심 정리

---

### 1) React 컴포넌트 3대 원칙

| 원칙 | 설명 |
|------|------|
| 단일 책임 | 하나의 컴포넌트는 하나의 역할만 담당 |
| props는 읽기 전용 | 자식이 props를 직접 수정하지 않음 (단방향 흐름) |
| 상태(state)는 useState로 | 화면이 바뀌어야 하는 값은 반드시 `useState`로 관리 |

---

### 2) useState vs useEffect

```
useState
  └── 화면에 반영되어야 하는 값 관리
  └── setter 호출 → 리렌더링 트리거

useEffect
  ├── 의존성 []        → 마운트 1회 (초기화 작업)
  ├── 의존성 [state]   → state 변경마다 (사이드 이펙트)
  └── 의존성 없음      → 매 렌더링마다 (거의 사용 안 함)
```

---

### 3) 이벤트 핸들링 핵심 패턴

| 이벤트 | 핵심 포인트 |
|--------|------------|
| `onClick` | `() => setState(...)` 인라인 또는 핸들러 함수 |
| `onChange` | `e.target.value`로 값 읽기, `value` prop으로 제어 컴포넌트 |
| `onKeyUp` | `e.key === "Enter"` 로 특정 키 감지 |
| `onSubmit` | `e.preventDefault()` 필수 → 페이지 새로고침 방지 |
| `onMouseEnter/Leave` | `boolean` 상태로 hover 효과 구현 |
| `onScroll` | `e.target.scrollTop` 으로 스크롤 위치 읽기 |

---

### 4) 조건부 렌더링 선택 기준

```
분기가 복잡하거나 JSX 블록이 길다
  └── if / else

JSX 내부에서 짧게 분기
  └── 삼항 연산자 isAuth ? <A /> : <B />

조건이 참일 때만 렌더링 (false면 아무것도 없음)
  └── && 단축평가  items && items.map(...)
```
