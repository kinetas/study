# 2월 26일 필기 — 화면설계 (screen-design)

## 핵심요약

프로토타입 작업 시 기준 폰트로 프리텐다드를 사용하며, 초기에 서체 기준을 먼저 정하는 것이 효율적이다. 폰트가 바뀌면 간격·줄바꿈·컴포넌트 크기까지 함께 수정해야 하므로 서체를 고정하는 습관이 중요하다. 프리텐다드는 한글 가독성이 안정적이고 굵기 구분이 명확해 제목과 본문을 나누기 쉽다. 프로토타입은 완성본이 아닌 화면 흐름과 방향을 빠르게 맞추는 의사소통 도구임을 기억해야 한다.

## 프로토타입 폰트 기준

- 폰트 참고: **프리텐다드**
- 제목, 본문, 버튼, 메뉴에 같은 계열 서체를 쓰면 시안의 통일감이 높아짐
- 폰트를 자주 바꾸면 간격, 줄바꿈, 컴포넌트 크기까지 함께 수정
- 그래서 초기 화면설계에서는 서체 기준을 먼저 정하고 나머지를 맞추는 편이 효율적

## 프리텐다드 장점

- 한글 가독성이 안정적이라 본문과 UI 라벨에 모두 무난하게 적용
- 굵기 구분이 비교적 잘 되어 제목과 설명 문장을 나누기 쉬움
- 웹과 앱 시안에서 모두 무난하게 보여 참고용 서체로 쓰기 좋음

## 프로토타입에서 주의점

- 글자 크기: 제목과 본문의 위계를 분명히 보여 주는지 확인
- 줄 간격: 설명 문장이 길어져도 답답하지 않은지 확인
- 버튼 문구: 너무 길어 버튼 폭이 깨지지 않는지 확인
- 네비게이션: 메뉴명이 길어졌을 때도 정렬이 유지되는지 점검

## 실무적 연결

- 디자인 참고 링크를 받아도 폰트가 다르면 같은 화면처럼 보이지 않을 수 있음
- 그래서 "참고 시안의 분위기"와 "우리 프로젝트의 실제 서체"를 구분해서 보는 습관이 필요
- 이 단계의 목적은 완성본 제작보다도 화면 방향을 빠르게 맞추는 것
- 프로토타입은 구현 전 의사소통 도구
