# 5월 13일 필기 — JAVA

## 핵심요약

Swing에서 `ActionListener`, `KeyListener`, `MouseListener`를 한 클래스에 `implements`로 다중 구현하면 버튼 클릭·Enter 키·마우스 클릭을 단일 클래스에서 처리할 수 있다. `JFileChooser`는 `showSaveDialog()`/`showOpenDialog()`로 OS 파일 다이얼로그를 띄우며, `getSelectedFile()`로 경로를 받아 `FileWriter`/`FileReader`와 연결해 텍스트를 저장·불러온다. 보조 스트림(Wrapper Stream)은 기본 스트림을 감싸 기능을 추가하는 데코레이터 패턴으로, `InputStreamReader`/`OutputStreamWriter`는 바이트↔문자 변환을, `BufferedReader`/`BufferedWriter`는 버퍼를, `PrintWriter`는 `println()`을 제공한다. `BufferedReader.readLine()`은 줄 단위로 읽고 EOF에서 `null`을 반환한다. `DataOutputStream`/`DataInputStream`은 기본 타입을 그대로 저장·복원하며, 반드시 **쓴 순서대로** 읽어야 한다. 객체 직렬화는 `implements Serializable`과 `serialVersionUID`를 선언하고 `ObjectOutputStream.writeObject()`로 저장, `ObjectInputStream.readObject()`로 복원하며, 더 읽을 객체가 없으면 `EOFException`이 발생한다. JDBC는 `Class.forName()`으로 드라이버를 로드하고, `DriverManager.getConnection()`으로 `Connection`을 얻어 `PreparedStatement`로 SQL을 실행한다. DML(INSERT·UPDATE·DELETE)은 `executeUpdate()`로, SELECT는 `executeQuery()`로 실행하며 결과는 `ResultSet`에 담긴다. SELECT 결과를 DTO에 매핑해 `List`로 관리하는 패턴이 실무 기본이다. 트랜잭션은 `setAutoCommit(false)` → 작업 → `commit()`/`rollback()` 순서로 제어하며, `setSavepoint()`로 부분 롤백 지점을 설정할 수 있다.

## JAVA03 — Ch07 보조 스트림 (Wrapper Stream)

> 보조 스트림은 단독 사용 불가, 기본 스트림을 감싸서 기능을 추가하는 데코레이터 패턴.

**보조 스트림 체인 구조**

```
기본 스트림 → 변환 스트림 → 버퍼 스트림
FileOutputStream → OutputStreamWriter → BufferedWriter
FileInputStream  → InputStreamReader  → BufferedReader
```

---

### 1) 변환 스트림 (InputStreamReader / OutputStreamWriter)

바이트 스트림 ↔ 문자 스트림 변환 브릿지.

**쓰기**

```java
OutputStream out   = new FileOutputStream("경로/test3.txt");
OutputStreamWriter wout = new OutputStreamWriter(out);    // 바이트 → 문자 변환
BufferedWriter bout     = new BufferedWriter(wout);       // 버퍼 추가
bout.write("문자 변환 스트림테스트!");
bout.flush();
bout.close(); wout.close(); out.close();
```

**읽기**

```java
InputStream in       = new FileInputStream("경로/test3.txt");
InputStreamReader rin = new InputStreamReader(in);    // 바이트 → 문자 변환
BufferedReader bin    = new BufferedReader(rin);       // 버퍼 추가
int data;
while ((data = bin.read()) != -1) System.out.println((char) data);
```

| 클래스 | 역할 |
|---|---|
| `InputStreamReader` | `InputStream` → `Reader` 변환 |
| `OutputStreamWriter` | `OutputStream` → `Writer` 변환 |
| `BufferedReader` | 문자 스트림에 버퍼 추가 |
| `BufferedWriter` | 문자 스트림에 버퍼 추가 |

---

### 2) 줄 단위 읽기 (BufferedReader.readLine / PrintWriter)

**쓰기 (PrintWriter)**

```java
Writer out        = new FileWriter("경로/test3.txt");
BufferedWriter bout = new BufferedWriter(out);
PrintWriter pout    = new PrintWriter(bout);   // print/println 사용 가능
pout.print("HELLO WORLD");
pout.flush(); pout.close(); bout.close(); out.close();
```

**읽기 (readLine — 줄 단위)**

```java
Reader in         = new FileReader("경로/test3.txt");
BufferedReader bin = new BufferedReader(in);
while (true) {
    String msg = bin.readLine();    // 줄 끝 개행문자 제거 후 반환, EOF면 null
    if (msg == null) break;
    System.out.println(msg);
}
in.close();
```

| 메서드 | 설명 |
|---|---|
| `readLine()` | 한 줄 읽기, EOF 도달 시 `null` 반환 |
| `PrintWriter.print()` | 개행 없이 출력 |
| `PrintWriter.println()` | 개행 포함 출력 |

---

### 3) 타입별 읽기/쓰기 (DataInputStream / DataOutputStream)

기본 타입을 그대로 저장·복원. **쓴 순서와 동일한 순서로 읽어야 한다.**

**쓰기**

```java
FileOutputStream out   = new FileOutputStream("경로/test3.txt");
DataOutputStream dout  = new DataOutputStream(out);
dout.writeUTF("홍길동");   // String (UTF-8)
dout.writeDouble(95.5);   // double (8 byte)
dout.writeInt(100);       // int (4 byte)
dout.flush(); dout.close();
```

**읽기 (쓴 순서 그대로)**

```java
FileInputStream in    = new FileInputStream("경로/test3.txt");
DataInputStream din   = new DataInputStream(in);
String name   = din.readUTF();      // "홍길동"
double weight = din.readDouble();   // 95.5
int data      = din.readInt();      // 100
```

| 메서드 쌍 | 타입 |
|---|---|
| `writeUTF` / `readUTF` | `String` |
| `writeDouble` / `readDouble` | `double` |
| `writeInt` / `readInt` | `int` |
| `writeBoolean` / `readBoolean` | `boolean` |

### 4) 객체 직렬화 (ObjectInputStream / ObjectOutputStream)

객체를 바이트 스트림으로 변환(직렬화)해 파일에 저장하고 복원(역직렬화).

**Serializable 구현**

```java
class Board implements Serializable {
    private static final long serialVersionUID = 1L;  // 버전 관리용
    private int bno;
    private String title;
    private String content;
    private String writer;
    private Date date;
    // 생성자, getter/setter, toString 생략
}
```

**직렬화 — 객체 → 파일**

```java
FileOutputStream out       = new FileOutputStream("경로/board.db");
ObjectOutputStream oout    = new ObjectOutputStream(out);
oout.writeObject(board1);
oout.writeObject(board2);
oout.writeObject(board3);
oout.flush(); oout.close(); out.close();
```

**역직렬화 — 파일 → 객체**

```java
FileInputStream in        = new FileInputStream("경로/board.db");
ObjectInputStream oin     = new ObjectInputStream(in);
Board b1 = (Board) oin.readObject();   // Object → 타입 캐스팅 필수
Board b2 = (Board) oin.readObject();
Board b3 = (Board) oin.readObject();
// 더 읽을 객체가 없을 때 readObject() 호출 → EOFException
oin.close(); in.close();
```

| 항목 | 내용 |
|---|---|
| `implements Serializable` | 직렬화 대상 클래스 필수 마커 인터페이스 |
| `serialVersionUID` | 버전 식별자, 불일치 시 역직렬화 실패 |
| `writeObject()` | 객체 → 바이트 스트림 |
| `readObject()` | 바이트 스트림 → `Object` (캐스팅 필요) |
| `EOFException` | 더 읽을 객체가 없을 때 발생 |
| `.db` 확장자 | 관례적 사용, 실제로는 바이트 파일 |

## JAVA03 — Ch08 JDBC (Java Database Connectivity)

---

### 1) JDBC 기본 구조

JDBC는 Java에서 DBMS에 접속하여 SQL을 실행하는 표준 API.

**참조변수 3종**

| 변수 | 타입 | 역할 |
|---|---|---|
| `conn` | `Connection` | DBMS의 특정 DB에 연결되는 객체 |
| `pstmt` | `PreparedStatement` | SQL 쿼리 전송용 객체 |
| `rs` | `ResultSet` | SELECT 결과를 담는 객체 |

**연결 순서**

```
1. Class.forName("com.mysql.cj.jdbc.Driver")    → 드라이버 로드
2. DriverManager.getConnection(url, id, pw)     → DB 연결
3. conn.prepareStatement(sql)                   → SQL 준비
4. executeUpdate() / executeQuery()             → SQL 실행
5. finally: rs → pstmt → conn 순서로 close()
```

```java
String id  = "root";
String pw  = "1234";
String url = "jdbc:mysql://localhost:3306/opendatadb";

Connection conn         = null;
PreparedStatement pstmt = null;
ResultSet rs            = null;

try {
    Class.forName("com.mysql.cj.jdbc.Driver");
    conn = DriverManager.getConnection(url, id, pw);

} catch (Exception e) {
    e.printStackTrace();
} finally {
    try { pstmt.close(); } catch (Exception e) { e.printStackTrace(); }
    try { conn.close();  } catch (Exception e) { e.printStackTrace(); }
}
```

---

### 2) INSERT / UPDATE / DELETE

`executeUpdate()` → 영향받은 행 수(int) 반환. 0이면 실패.  
`?` 는 1-based 인덱스로 `setXxx(순서, 값)` 으로 바인딩.

```java
// INSERT
pstmt = conn.prepareStatement("insert into tbl_a values(?,?)");
pstmt.setInt(1, Integer.parseInt(args[0]));
pstmt.setString(2, args[1]);

// UPDATE
pstmt = conn.prepareStatement("update tbl_a set name = ? where no = ?");
pstmt.setString(1, args[0]);
pstmt.setInt(2, Integer.parseInt(args[1]));

// DELETE
pstmt = conn.prepareStatement("delete from tbl_a where no = ?");
pstmt.setInt(1, Integer.parseInt(args[0]));

int result = pstmt.executeUpdate();
if (result > 0) System.out.println("성공");
else            System.out.println("실패");
```

---

### 3) SELECT

`executeQuery()` → `ResultSet` 반환. `rs.next()`로 한 행씩 이동.

```java
pstmt = conn.prepareStatement("select * from tbl_a");
rs = pstmt.executeQuery();

while (rs.next()) {
    System.out.print(rs.getInt("no") + ".");
    System.out.println(rs.getString("name"));
}
```

**finally close 순서** — rs → pstmt → conn (열린 순서의 역순)

```java
finally {
    try { rs.close();    } catch (Exception e) { e.printStackTrace(); }
    try { pstmt.close(); } catch (Exception e) { e.printStackTrace(); }
    try { conn.close();  } catch (Exception e) { e.printStackTrace(); }
}
```

| 메서드 | 용도 |
|---|---|
| `executeUpdate()` | INSERT / UPDATE / DELETE — 영향 행 수 반환 |
| `executeQuery()` | SELECT — `ResultSet` 반환 |
| `rs.next()` | 다음 행으로 이동 (없으면 `false`) |
| `rs.getInt("col")` | int 컬럼 값 추출 |
| `rs.getString("col")` | String 컬럼 값 추출 |

### 4) DTO 패턴 + 오픈데이터 조회

SELECT 결과를 DTO 객체에 담아 `List`로 관리하는 실무 기본 패턴.

**DTO 구조 (도시가스 충전소)**

```java
class C06ChargeDto {
    private int    순번;
    private String 행정구역, 지사, 시설명, 주소;
    private int    우편번호;
    // 기본 생성자 + 모든인자 생성자 + getter/setter + toString 재정의
}
```

**SELECT → DTO → List**

```java
pstmt = conn.prepareStatement("select * from tbl_charge");
rs = pstmt.executeQuery();

List<C06ChargeDto> list = new ArrayList();
while (rs.next()) {
    C06ChargeDto dto = new C06ChargeDto();
    dto.set순번(rs.getInt("순번"));
    dto.set행정구역(rs.getString("행정구역"));
    dto.set지사(rs.getString("지사"));
    dto.set시설명(rs.getString("시설명"));
    dto.set우편번호(rs.getInt("우편번호"));
    dto.set주소(rs.getString("주소"));

}
list.forEach(System.out::println);
```

> DTO(Data Transfer Object): DB 컬럼을 필드로 매핑해 데이터를 이동시키는 객체.  
> `toString()` 재정의로 출력 형식을 고정하고 `System.out::println`으로 간결하게 출력.

### 5) 트랜잭션 & Savepoint

기본값은 `autoCommit=true`(SQL마다 자동 커밋). 수동 제어 시 `setAutoCommit(false)` 먼저 호출.

```java
conn.setAutoCommit(false);  // TX 시작

pstmt = conn.prepareStatement("insert into tbl_a values(1,'a')");
pstmt.executeUpdate();

Savepoint sp1 = conn.setSavepoint("sp1");  // 저장점 설정

pstmt = conn.prepareStatement("insert into tbl_a values(2,'b')");
pstmt.executeUpdate();
// 중복 PK 등 예외 발생 시 catch 진입

conn.commit();  // 전체 성공 시 커밋
```

**예외 발생 시 롤백**

```java
catch (Exception e) {
    try {
        if (sp1 != null) conn.rollback(sp1);  // Savepoint까지 부분 롤백
        else             conn.rollback();      // 전체 롤백
        conn.commit();                         // 롤백 후 반드시 커밋
    } catch (Exception rollback) { rollback.printStackTrace(); }
}
finally {
    try { conn.setAutoCommit(true); } catch (SQLException e) { e.printStackTrace(); }
    try { pstmt.close(); } catch (Exception e) { e.printStackTrace(); }
    try { conn.close();  } catch (Exception e) { e.printStackTrace(); }
}
```

| 메서드 | 설명 |
|---|---|
| `setAutoCommit(false)` | 수동 트랜잭션 모드 전환 |
| `setSavepoint("이름")` | 현재 시점에 저장점 설정 |
| `commit()` | 트랜잭션 확정 |
| `rollback()` | 트랜잭션 전체 취소 |
| `rollback(savepoint)` | 지정 저장점까지만 취소 |

> **주의**: `rollback()` 후 반드시 `commit()`을 호출해야 DB에 반영됨

### Ch08 핵심 정리

**JDBC CRUD 흐름**

```
드라이버 로드 → DB 연결 → SQL 준비 → 바인딩 → 실행 → (ResultSet) → close
```

**executeUpdate vs executeQuery**

| 메서드 | 대상 SQL | 반환 |
|---|---|---|
| `executeUpdate()` | INSERT / UPDATE / DELETE | 영향 행 수 (`int`) |
| `executeQuery()` | SELECT | `ResultSet` |

**트랜잭션 제어 흐름**

```
setAutoCommit(false)
    → 작업들...
    → setSavepoint()   ← 부분 롤백 기준점
    → 추가 작업...
성공 → commit()
실패 → rollback(sp) or rollback() → commit()
finally → setAutoCommit(true) → close()
```
