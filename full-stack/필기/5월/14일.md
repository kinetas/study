# 5월 14일 필기 — JAVA

## 핵심요약

소켓 통신은 `ServerSocket.accept()`로 클라이언트 연결을 대기하고, `DataOutputStream.writeUTF()` / `DataInputStream.readUTF()`로 문자열을 송수신한다. 1:1 순차 방식은 송수신이 번갈아 이루어져 동시 채팅이 불가능하므로, 스레드로 수신(`RecvThread`)과 송신(`SendThread`)을 분리해 해결한다. 스레드 생성은 `Runnable` 인터페이스 구현 또는 `Thread` 익명 클래스 두 가지 방법이 있으며, `th.start()`로 실행, `th.interrupt()`로 `InterruptedException`을 발생시켜 종료, `th.join()`으로 완료 대기한다. 여러 스레드가 공유 변수를 동시에 수정하면 Race Condition이 발생하고, `synchronized` 블록과 `wait()` / `notifyAll()`로 순서를 제어해 해결한다. Swing GUI 채팅은 수신 스레드가 `area.append()`로 GUI를 갱신하고, 송신은 엔터키 `KeyListener` 이벤트에서 처리한다. N:N 다중 채팅은 서버가 `Map<닉네임, DataOutputStream>`으로 모든 클라이언트를 관리하고 `broadCast()`로 전체 전파하며, 클라이언트마다 `ServerRecvThread` 스레드 1개를 생성한다. 리플렉션은 `Class.forName()`으로 런타임에 클래스를 로딩하고 `Constructor.newInstance()`, `Method.invoke()`, `Field.set()`으로 동적 조작하며, 성능 오버헤드가 있어 프레임워크 내부에서 주로 사용된다. 디자인 패턴은 생성(싱글톤·빌더·프로토타입·팩토리메서드·추상팩토리), 구조(어댑터·데코레이터·프록시), 행위(옵저버·전략·커맨드) 세 범주로 나뉜다.

### 2) C01 — 단방향 통신 (서버 → 클라이언트)

서버가 접속한 클라이언트에 환영 메시지 1회 전송. `while(true)`로 반복 접속 처리.

**Server**

```java
ServerSocket server = new ServerSocket(7000);
while(true) {
    Socket client = server.accept();
    DataOutputStream dout = new DataOutputStream(client.getOutputStream());
    dout.writeUTF("[SERVER] WELCOME TO SERVER + " + new Date());
    dout.flush();
    dout.close(); client.close();
}
```

**Client**

```java
Socket server = new Socket("192.168.5.15", 7000);
DataInputStream din = new DataInputStream(server.getInputStream());
String recv = din.readUTF();
System.out.println("메시지 : " + recv);
din.close(); server.close();
```

### 3) C02 — 양방향 채팅 (1:1 순차 방식)

서버가 먼저 전송 → 클라이언트 수신 후 응답. `"q"` 입력 시 종료.  
**단점**: 순차 방식이라 동시 송수신 불가 → Ch11에서 스레드로 해결.

**Server**

```java
Socket client = server.accept();
DataInputStream din   = new DataInputStream(client.getInputStream());
DataOutputStream dout = new DataOutputStream(client.getOutputStream());

while(true) {
    send = sc.nextLine();
    dout.writeUTF(send); dout.flush();    // 송신
    if(send.equals("q")) break;
    recv = din.readUTF();                 // 수신
    if(recv.equals("q")) break;
    System.out.println("[CLIENT] : " + recv);
}
```

**Client**

```java
while(true) {
    recv = din.readUTF();                 // 수신 (서버가 먼저 보냄)
    if(recv.equals("q")) break;
    System.out.println("[SERVER] : " + recv);
    send = sc.nextLine();
    dout.writeUTF(send); dout.flush();    // 송신
    if(send.equals("q")) break;
}
```

### 4) C04/C05 — 동기화 (Race Condition → synchronized)

**문제**: 여러 스레드가 `counter++`을 동시 접근하면 결과 불일치 (Race Condition)

```java
// 문제 코드 — 4개 스레드가 counter++을 각각 100000번 수행
// 예상: 400000, 실제: 불규칙
private static int counter = 0;
```

**해결**: `synchronized` + `wait()` / `notifyAll()`

```java
private static final Object Lock = new Object();

public void run() {
    for(int i = 0; i < 100000; i++) {
        synchronized(Lock) {       // Lock 획득 (다른 스레드 대기)
            counter++;
            Lock.notifyAll();      // 대기 중인 스레드 깨움
            Lock.wait(5);          // Lock 해제 후 5ms 대기
        }
    }
}
```

**`join()`으로 모든 스레드 완료 대기**

```java
thread1.join(); thread2.join();
thread3.join(); thread4.join();
System.out.println("Final value: " + incrementThread1.getCounter());
```

| 메서드 | 설명 |
|---|---|
| `synchronized(obj)` | 블록 진입 시 obj 락 획득, 나올 때 해제 |
| `obj.wait(ms)` | 락 해제 후 ms 동안 대기. 0이면 무한 대기 |
| `obj.notifyAll()` | obj를 기다리는 모든 스레드 깨움 |
| `th.join()` | th 스레드 종료까지 현재 스레드 블로킹 |

## JAVA03 — Ch11 멀티스레드 소켓 통신

---

### 1) 개념

Ch09 C02의 순차 방식 한계를 스레드로 해결. **수신**과 **송신**을 독립 스레드로 분리해 동시 처리.

```
ServerMain
├── ServerRecvThread  (클라이언트로부터 수신)
└── ServerSendThread  (클라이언트로 송신)

ClientMain
├── ClientRecvThread  (서버로부터 수신)
└── ClientSendThread  (서버로 송신)
```

---

### 2) 스레드 분리 구조

```java
// ServerMain
ServerRecvThread recvThread = new ServerRecvThread(din, server);
ServerSendThread sendThread = new ServerSendThread(dout, server);
Thread th1 = new Thread(recvThread);
Thread th2 = new Thread(sendThread);
th1.start(); th2.start();
th1.join();  th2.join();
```

**RecvThread — 수신 전담**

```java
public void run() {
    while(true) {
        try {
            recv = din.readUTF();
        } catch(EOFException e) { break; }
         catch(Exception e)    { break; }
        if(recv.equals("q")) { server.close(); break; }
        System.out.println("[CLIENT] : " + recv);
    }
}
```

**SendThread — 송신 전담**

```java
public void run() {
    while(true) {
        send = sc.nextLine();
        if(send.equals("q")) {
            dout.writeUTF(send); dout.flush();
            server.close(); break;
        }
        dout.writeUTF(send); dout.flush();
    }
}
```

|---|---|---|
| 송수신 방식 | 번갈아가며 | 동시 (독립 스레드) |
| 먼저 보내는 쪽 | 서버 고정 | 누구든 가능 |
| 구현 복잡도 | 단순 | Recv/Send 스레드 분리 |

## JAVA03 — Ch13 다중 클라이언트 채팅 (N:N)

---

### 1) 구조

1:1 → N:N 확장. 서버가 `Map<닉네임, DataOutputStream>`으로 클라이언트 스트림 관리 후 `broadCast()` 전파.

```
ServerUI
└── ServerBackground
    ├── ServerSocket (포트 5555)
    ├── ClientList : Map<String, DataOutputStream>
    ├── addClient()    — 접속 등록 + 전체 알림
    ├── removeClient() — 퇴장 처리
    └── broadCast()    — 전체 or 자신 제외 전송

클라이언트 접속마다 ServerRecvThread 1개 생성
```

---

### 2) ServerBackground — 핵심 로직

```java
// 동기화된 클라이언트 목록
Map<String, DataOutputStream> ClientList = new HashMap<>();
Collections.synchronizedMap(ClientList);

// 접속 루프 (클라이언트마다 스레드 생성)
while(true) {
    client = server.accept();
    new Thread(new ServerRecvThread(client, this, gui)).start();
}

// 전체 브로드캐스트
public void broadCast(String msg) {
    for(String key : ClientList.keySet()) {
        ClientList.get(key).writeUTF(msg);
        ClientList.get(key).flush();
    }
}

// 자신 제외 브로드캐스트
public void broadCast(String nick, String msg) {
    for(String tmp : ClientList.keySet()) {
        if(!nick.equals(tmp)) {
            ClientList.get(tmp).writeUTF(msg);
        }
    }
}
```

---

### 3) ServerRecvThread — 닉네임 등록 + 수신

```java
// 생성자: 첫 메시지로 닉네임 수신 후 등록
nick = Din.readUTF();
background.addClient(nick, Dout);

// run(): 메시지 수신 → broadCast
public void run() {
    while(true) {
        recv = Din.readUTF();
        background.broadCast(nick, recv);       // 자신 제외 전파
        gui.area.append(recv + "\n");
        gui.area.setCaretPosition(gui.area.getDocument().getLength());
    }
    // 예외 시 퇴장 처리
    background.removeClient(nick);
}
```

---

### 4) ClientUI — 닉네임 + 접속 + 송신

## JAVA03 — Ch14 리플렉션 (Reflection)

---

### 1) 개념

런타임에 클래스 정보를 동적으로 조회·조작하는 API (`java.lang.reflect`).  
프레임워크·라이브러리 내부에서 주로 활용. 일반 코드에서는 성능 이유로 지양.

| 클래스 | 역할 |
|---|---|
| `Class` | 클래스 메타데이터 (이름, 필드, 메서드, 생성자) |
| `Field` | 필드 정보 조회 및 값 읽기/수정 |
| `Method` | 메서드 정보 조회 및 호출 |
| `Constructor` | 생성자 정보 조회 및 객체 생성 |

---

### 2) C01 — 클래스 정보 조회

```java
// 문자열로 클래스 런타임 로딩
Class<?> clazz = Class.forName("java.lang.String");

Field[]       fields       = clazz.getDeclaredFields();
Constructor[] constructors = clazz.getDeclaredConstructors();
Method[]      methods      = clazz.getDeclaredMethods();
for(Method m : methods) System.out.println(m);
```

---

### 3) C02 — 생성자·메서드·필드 동적 호출

```java
Class<?> clazz = Class.forName("Ch14.Simple");

// 생성자로 객체 생성
Constructor[] cons = clazz.getConstructors();
Object ob2 = cons[1].newInstance("홍길동", 30, "대구");

// 메서드 호출
Method setName = clazz.getDeclaredMethod("setName", String.class);
Method getName = clazz.getDeclaredMethod("getName", null);
setName.invoke(ob1, "남길동");
getName.invoke(ob1, null);

// 필드 직접 접근 (private 아닌 필드)
Field addr = clazz.getDeclaredField("addr");
addr.set(ob2, "창원");

// 범용 객체 생성 유틸
public static Object genObject(Class<?> clazz) throws Exception {
    return clazz.getDeclaredConstructor(null).newInstance(null);
}
genObject(Class.forName("Ch14.Simple"));
```

| 메서드 | 설명 |
|---|---|
| `Class.forName("패키지.클래스")` | 런타임에 클래스 로딩 |
| `clazz.getDeclaredMethod(이름, 타입...)` | 메서드 탐색 |
| `method.invoke(obj, args...)` | 메서드 호출 |
| `field.set(obj, value)` | 필드 값 설정 |
| `constructor.newInstance(args...)` | 객체 생성 |

> **주의**: 리플렉션은 컴파일 타임 타입 안전성 미보장 + 성능 오버헤드. 가능하면 정적 방식 사용.

### 1) 생성 패턴

**싱글톤 (Singleton)** — 비유: 회사 사장 (단 한 명)

```java
class Singleton {
    private static Singleton instance;
    private Singleton() {}                         // 외부 생성 차단

    public static Singleton getInstance() {
        if(instance == null) instance = new Singleton();
        return instance;
    }
}

Singleton s1 = Singleton.getInstance();
Singleton s2 = Singleton.getInstance();
System.out.println(s1 == s2);   // true
```

용도: 설정(Config), 로거(Logger), DB 커넥션 풀, 캐시

---

**빌더 (Builder)** — 비유: 햄버거 주문 (옵션 골라서 조립)

```java
Car car = new Car.Builder("V6", 4)   // 필수
        .color("red")                 // 선택
        .sunroof(true)
        .build();

// Builder 내부: 체이닝 구조
public Builder color(String c) { this.color = c; return this; }
public Car build() { return new Car(this); }
```

용도: 선택 옵션이 많은 객체, 생성자 파라미터가 너무 많을 때

---

**프로토타입 (Prototype)** — 비유: 문서 복사 (원본 두고 사본)

```java
class Computer implements Cloneable {
    String cpu;
    protected Computer clone() throws CloneNotSupportedException {
        return (Computer) super.clone();   // 얕은 복사
    }
}

Computer copy = original.clone();
copy.cpu = "AMD Ryzen";   // 복사본만 변경, 원본 유지
```

---

**팩토리 메서드 (Factory Method)** — 비유: 동물 공장 (개공장→개)

```java
interface AnimalFactory { Animal createAnimal(); }

class DogFactory implements AnimalFactory {
    public Animal createAnimal() { return new Dog(); }
}

AnimalFactory factory = new DogFactory();
factory.createAnimal().sound();  // 멍멍
```

---

**추상 팩토리 (Abstract Factory)** — 비유: OS별 GUI 세트

```java
interface GUIFactory {
    Button   createButton();
    CheckBox createCheckBox();
}
class WinFactory implements GUIFactory { ... }  // Windows 세트
class MacFactory implements GUIFactory { ... }  // Mac 세트

// 공장 한 줄만 바꾸면 모든 UI가 일관되게 변경
GUIFactory factory = new WinFactory();
factory.createButton().click();
factory.createCheckBox().check();
```

| 비교 | 팩토리 메서드 | 추상 팩토리 |
|---|---|---|
| 생성 단위 | 객체 1개 | 관련 객체 세트 |
| 확장 방향 | 서브클래스(공장) 추가 | 공장 인터페이스 확장 |

### 2) 구조 패턴

**어댑터 (Adapter)** — 비유: 220V→110V 변환 어댑터

```java
class VoltageAdapter implements Socket220V {
    private Device110V device;
    public void plug220() {
        device.plug110();   // 220V 요청 → 110V 호출로 변환
    }
}

Socket220V socket = new VoltageAdapter(new Device110V());
socket.plug220();
```

용도: 수정 불가한 외부 클래스를 기존 인터페이스에 맞춰 사용

---

**데코레이터 (Decorator)** — 비유: 기본 커피 → 우유 → 시럽 (감싸기)

```java
class MilkDecorator implements Coffee {
    private Coffee coffee;
    public String desc() { return coffee.desc() + " + 우유"; }
    public int cost()    { return coffee.cost() + 500; }
}

// 중첩 감싸기
Coffee c = new SyrupDecorator(new MilkDecorator(new BasicCoffee()));
// → "커피 + 우유 + 시럽 : 4200원"
```

용도: 상속 없이 기능 확장, 옵션 조합이 다양할 때 (BufferedReader 등)

---

**프록시 (Proxy)** — 비유: 비서 (실제 객체 앞에서 접근 제어)

```java
class ProxyImage implements Image {
    private RealImage realImage;   // 처음엔 null

    public void show() {
        if(realImage == null) realImage = new RealImage(fileName);  // 첫 호출 시만 로딩
        realImage.show();
    }
}

Image img = new ProxyImage("photo.jpg");  // 아직 로딩 X
img.show();  // 이때 로딩 발생
img.show();  // 재로딩 없음 (캐시)
```

용도: 지연 로딩, 접근 제어(권한 체크), 로깅, 캐싱

### 3) 행위 패턴

**옵저버 (Observer)** — 비유: 유튜브 구독 알림

```java
class Subject {
    private List<Observer> observers = new ArrayList<>();

    public void setState(int state) {
        this.state = state;
        for(Observer o : observers) o.update();  // 전체 알림
    }
    public void attach(Observer o) { observers.add(o); }
}

new BinaryObserver(subject);  // 구독 등록
new OctalObserver(subject);
subject.setState(10);         // → 모든 옵저버 update() 호출
```

---

**전략 (Strategy)** — 비유: 결제 방식 선택 (카드/현금/카카오페이)

```java
interface Strategy { int execute(int a, int b); }
class AddStrategy      implements Strategy { public int execute(int a, int b){ return a+b; } }
class MultiplyStrategy implements Strategy { public int execute(int a, int b){ return a*b; } }

class Context {
    private Strategy strategy;
    public int executeStrategy(int a, int b) { return strategy.execute(a, b); }
}

Context ctx = new Context(new AddStrategy());
ctx.executeStrategy(10, 5);   // 15 → new MultiplyStrategy()로 바꾸면 50
```

용도: if-else 분기로 알고리즘 선택하던 코드 정리, 정렬·인증 방식 동적 교체

---

**커맨드 (Command)** — 비유: TV 리모컨 버튼

```java
interface Command { void execute(); }
class TV { void turnOn(){...} void turnOff(){...} }    // Receiver

class TurnOnCommand implements Command {
    private TV tv;
    public void execute() { tv.turnOn(); }
}

class RemoteControl {                                   // Invoker
    private Command command;
    public void setCommand(Command c) { this.command = c; }
    public void pressButton() { command.execute(); }
}

RemoteControl remote = new RemoteControl();
remote.setCommand(new TurnOnCommand(tv));
remote.pressButton();   // TV가 켜졌습니다
```

| 역할 | 클래스 | 설명 |
|---|---|---|
| Command | `Command` 인터페이스 | 실행 계약 |
| Receiver | `TV` | 실제 동작 수행 |
| ConcreteCommand | `TurnOnCommand` | 명령 캡슐화 |
| Invoker | `RemoteControl` | 명령 실행 요청 |

용도: 작업 큐(Job Queue), Undo/Redo, 호출자와 실행자 분리

### Ch15 핵심 정리

**생성 패턴 선택 기준**

```
인스턴스 1개 공유              → 싱글톤
옵션 많은 객체 생성            → 빌더
비슷한 객체 다량 복제          → 프로토타입
조건따라 다른 객체 생성        → 팩토리 메서드
관련 객체 세트 생성            → 추상 팩토리
```

**구조 패턴 선택 기준**

```
호환 안 되는 인터페이스 연결   → 어댑터
런타임 기능 추가 (감싸기)      → 데코레이터
접근 제어 / 지연 로딩          → 프록시
```

**행위 패턴 선택 기준**

```
상태 변화 → 여러 객체 자동 알림 → 옵저버
알고리즘 동적 교체              → 전략
요청 캡슐화 / Undo 필요        → 커맨드
```
