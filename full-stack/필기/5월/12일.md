# 5월 12일 필기 — JAVA

## 핵심요약

Java I/O는 문자 스트림(`Reader`/`Writer`)과 바이트 스트림(`InputStream`/`OutputStream`)으로 나뉜다. 문자 스트림은 텍스트 파일에, 바이트 스트림은 이미지·PDF 등 바이너리 파일에 적합하며, `FileWriter`의 두 번째 인자를 `true`로 지정하면 append(이어쓰기) 모드가 된다. 스트림 사용 후에는 반드시 `flush()` → `close()` 순서로 호출해야 버퍼에 남은 데이터가 유실되지 않는다. 성능 개선을 위해 `char[]` 또는 `byte[]` 배열을 버퍼로 사용하면 한 번에 여러 바이트를 읽어 처리 속도를 크게 높일 수 있다. 네트워크 URL에서 스트림을 열 때는 기본 바이트 스트림 위에 `BufferedInputStream`(버퍼)과 `InputStreamReader`(byte→char 변환) 같은 보조 스트림을 데코레이터 방식으로 씌운다. Jsoup은 외부 라이브러리로 HTML을 DOM 파싱하여 `getElementsByTag()` 등으로 요소를 추출하고, `URL.openStream()`으로 이미지를 바이너리 다운로드할 수 있다. Selenium은 실제 ChromeDriver를 제어해 `findElement(By.id/cssSelector)`로 요소를 찾고 `sendKeys`·`click`으로 조작하며, `JavascriptExecutor`로 JS를 직접 실행할 수 있다. Java 11+ 내장 `HttpClient`는 빌더 패턴으로 `HttpRequest`를 구성하고 `send()` 메서드로 응답을 받으며, Jackson의 `ObjectMapper.readTree()`로 JSON을 `JsonNode` 트리로 파싱한다.

### 1) 스트림 종류 비교

Java I/O 클래스는 처리 단위에 따라 두 계열로 나뉜다.

| 구분 | 문자 스트림 | 바이트 스트림 |
|---|---|---|
| 입력 | `Reader` / `FileReader` | `InputStream` / `FileInputStream` |
| 출력 | `Writer` / `FileWriter` | `OutputStream` / `FileOutputStream` |
| 단위 | `char` (2byte) | `byte` (1byte) |
| 적합 | 텍스트 파일 | 이미지·PDF 등 바이너리 |

### 2) 문자 스트림 — FileWriter / FileReader (C01~C03)

텍스트 파일 쓰기, 읽기, 복사를 모두 `Reader`/`Writer` 계열로 처리.

**FileWriter — 파일 쓰기**

```java
// false: 덮어쓰기(기본값)  /  true: 이어쓰기(append)
Writer fout = new FileWriter("경로/test1.txt", true);
fout.write("HELLOWORLD\n");
fout.flush();   // 버퍼 강제 출력 — 쓰기 후 반드시 호출
fout.close();
```

**FileReader — 파일 읽기**

```java
Reader fin = new FileReader("경로/test1.txt");
int data;
while ((data = fin.read()) != -1) {     // -1 이면 EOF
    System.out.print((char) data);      // int → char 변환
}
fin.close();
```

**텍스트 파일 복사 (C03)**

```java
Reader fin  = new FileReader(PATH + args[0]);   // 소스
Writer fout = new FileWriter(PATH + args[1]);   // 대상
int data;
while ((data = fin.read()) != -1) {
    fout.write((char) data);
    fout.flush();
}
fin.close();
fout.close();
// 실행 시 VM arguments에 소스·대상 파일명 전달
```

### 3) 버퍼를 이용한 성능 개선 (C04)

한 글자씩 읽으면 I/O 호출이 많아 느림 → `char[]` 배열로 한 번에 여러 글자 읽기.

```java
Reader fin = new FileReader("경로/origin.txt");
StringBuffer strBuffer = new StringBuffer();
char[] buff = new char[1024];   // 버퍼 크기 1024 char 권장
int data;

long startTime = System.currentTimeMillis();
while ((data = fin.read(buff)) != -1) {
    strBuffer.append(buff);     // 버퍼 내용을 통째로 추가
}
fin.close();
long endTime = System.currentTimeMillis();

System.out.println("총문자 길이 : " + strBuffer.length());
System.out.println("소요시간 : " + (endTime - startTime) + " ms");
```

| 방식 | 반환값 | 설명 |
|---|---|---|
| `fin.read()` | `int` | 1글자씩 읽음 |
| `fin.read(buff)` | `int` (읽은 글자 수) | 버퍼 크기만큼 한 번에 읽음 |

### 4) 바이트 스트림 — FileInputStream / FileOutputStream (C05~C07)

이미지·PDF 등 바이너리 파일은 반드시 바이트 스트림 사용.

**FileInputStream — 바이너리 읽기**

```java
InputStream fin = new FileInputStream("경로/파일.pdf");
byte[] buffer = new byte[1024];
int data;
while ((data = fin.read(buffer)) != -1) {
    System.out.print(data);   // 읽은 바이트 수 출력
}
fin.close();
```

**FileOutputStream — 바이너리 쓰기**

```java
OutputStream fout = new FileOutputStream("경로/test2.txt");
fout.write("가나다".getBytes(StandardCharsets.UTF_8));  // 한글: UTF-8 명시
fout.write('a');
fout.write('b');
fout.flush();
fout.close();
```

**바이너리 파일 복사 (C07)**

```java
InputStream  fin  = new FileInputStream(PATH + args[0]);
OutputStream fout = new FileOutputStream(PATH + args[1]);
byte[] buff = new byte[4096];   // 바이너리 버퍼 크기 4096 byte 권장
int data;
while ((data = fin.read(buff)) != -1) {
    fout.write(buff, 0, data);  // 실제 읽은 길이(data)만큼만 씀 — 마지막 블록 잘림 방지
    fout.flush();
}
fin.close();
fout.close();
```

### 5) URL 스트림 + 보조 스트림 (C08)

URL에서 스트림을 열고 보조 스트림을 데코레이터 방식으로 씌워 기능을 추가.

```java
URL url = (new URI("https://...")).toURL();

InputStream in = url.openStream();                  // 기본 스트림 (byte)
BufferedInputStream buffIn = new BufferedInputStream(in);   // 보조: 버퍼 추가
Reader rin = new InputStreamReader(in);             // 보조: byte → char 변환

Writer fout = new FileWriter("경로/index.html");    // 출력 스트림

int data;
while ((data = rin.read()) != -1) {
    fout.write((char) data);
    fout.flush();
}
fout.close();
rin.close();
buffIn.close();
in.close();
```

| 보조 스트림 | 역할 |
|---|---|
| `BufferedInputStream` | 버퍼를 추가해 읽기 성능 향상 |
| `InputStreamReader` | 바이트 스트림 → 문자 스트림 변환 |

### 6) Jsoup — HTML 파싱 & 이미지 크롤링 (C09)

외부 라이브러리 `org.jsoup`으로 URL에 접속해 DOM을 파싱하고 이미지를 다운로드.

```java
// 연결 설정
Connection conn = Jsoup.connect("https://...")
    .userAgent("Mozilla/5.0 ...")   // 봇 차단 우회
    .timeout(0)
    .ignoreHttpErrors(true)
    .followRedirects(true);

Document document = conn.get();     // GET 요청 → DOM 파싱

// img 태그 전체 추출
Elements elements = document.getElementsByTag("img");
System.out.println("총 img size : " + elements.size());

elements.forEach(el -> {
    String imgUrl = el.getElementsByAttribute("src").attr("src");

    // 상대경로 → 절대경로 변환
    if (!imgUrl.startsWith("http")) imgUrl = "https://도메인" + imgUrl;

    // URL 스트림으로 이미지 바이너리 다운로드
    InputStream in = (new URI(imgUrl)).toURL().openStream();
    BufferedInputStream bin = new BufferedInputStream(in);
    OutputStream out = new FileOutputStream("저장경로/" + UUID.randomUUID() + ".png");

    int data;
    while ((data = bin.read()) != -1) {
        out.write((byte) data);
        out.flush();
    }
    out.close(); bin.close(); in.close();
});
```

| 항목 | 설명 |
|---|---|
| `conn.get()` | GET 요청 → `Document` (DOM) 반환 |
| `getElementsByTag("img")` | 태그명으로 요소 선택 |
| `el.attr("src")` | 속성값 추출 |
| `UUID.randomUUID()` | 중복 없는 파일명 생성 |

### 7) Selenium — 브라우저 자동화 (C10)

외부 라이브러리 `org.openqa.selenium`으로 Chrome을 제어해 요소 탐색·조작·HTML 저장.

**드라이버 설정**

```java
// Selenium 3.x: chromedriver.exe 경로 직접 지정
System.setProperty("webdriver.chrome.driver", "src/Drivers/chromedriver.exe");
// Selenium 4.x: Windows PATH에 등록하면 위 설정 불필요

ChromeOptions options = new ChromeOptions();
// options.addArguments("--headless");   // 창 없이 백그라운드 실행

WebDriver driver = new ChromeDriver(options);
```

**요소 탐색 & 조작**

```java
driver.get("https://www.naver.com");

WebElement searchEl = driver.findElement(By.id("query"));   // id로 탐색
searchEl.sendKeys("건조기");        // 텍스트 입력
searchEl.sendKeys(Keys.RETURN);     // 엔터 전송

Thread.sleep(500);  // 동적 페이지 로딩 대기

WebElement btn = driver.findElement(By.cssSelector(".wrap .tab:nth-child(1)"));

// JS 실행 — target="_blank" 속성 제거 (새 창 방지)
JavascriptExecutor js = (JavascriptExecutor) driver;
js.executeScript("arguments[0].removeAttribute('target')", btn);
btn.click();
```

**복수 요소 추출 → HTML 파일 저장**

```java
List<WebElement> items = driver.findElements(By.cssSelector(".list_item"));
StringBuilder html = new StringBuilder();

// CSS 링크 추출
for (WebElement link : driver.findElements(By.tagName("link"))) {
    if ("stylesheet".equals(link.getAttribute("rel"))) {
        html.append("<link rel=\"stylesheet\" href=\"")
            .append(link.getAttribute("href")).append("\">\n");
    }
}

// 본문 HTML 추출 (outerHTML: 태그 포함 전체 HTML 문자열)
for (WebElement el : items) html.append(el.getAttribute("outerHTML")).append("\n");

Writer out = new FileWriter("C:\\IOTEST\\" + UUID.randomUUID() + ".html");
out.write(html.toString());
out.flush();
out.close();
```

| 항목 | 설명 |
|---|---|
| `By.id("id")` | id 속성으로 요소 탐색 |
| `By.cssSelector(".cls")` | CSS 선택자로 요소 탐색 |
| `sendKeys(Keys.RETURN)` | 엔터키 전달 |
| `JavascriptExecutor` | JS 직접 실행 (속성 제거·스크롤 등) |
| `outerHTML` | 태그 포함 전체 HTML 문자열 |

### 8) REST API — HttpClient & Jackson (C11)

Java 11+ 내장 `java.net.http`로 HTTP 요청, Jackson으로 JSON 파싱.

```java
// 01 URL + 쿼리 파라미터 구성
String url = "https://api.example.com/data?mode=json&addr=중구";

// 02 요청 객체 빌드 (빌더 패턴)
HttpRequest request = HttpRequest.newBuilder()
    .uri(URI.create(url))
    .GET()
    .build();

// 03 요청 전송 → 응답 수신
HttpClient httpClient = HttpClient.newHttpClient();
HttpResponse<String> response =
    httpClient.send(request, HttpResponse.BodyHandlers.ofString());

// 04 JSON 문자열 → JsonNode 트리로 파싱
ObjectMapper mapper = new ObjectMapper();
JsonNode root = mapper.readTree(response.body());

System.out.println("status : " + root.get("status"));
System.out.println("total : "  + root.get("total"));

// 배열 순회
JsonNode data = root.get("data");
for (int i = 0; i < data.size(); i++) {
    System.out.println(data.get(i).get("BZ_NM"));
}
```

| 항목 | 설명 |
|---|---|
| `HttpRequest.newBuilder()` | 빌더 패턴으로 요청 구성 |
| `BodyHandlers.ofString()` | 응답 바디를 `String`으로 수신 |
| `response.body()` | 응답 본문 (JSON 문자열) |
| `mapper.readTree()` | JSON → `JsonNode` 트리 파싱 |
| `jsonNode.get("key")` | 키로 값 접근 |
| `jsonNode.size()` | 배열 크기 |

### Ch06 핵심 정리

**스트림 선택 기준**

```
텍스트 파일     →  Reader / Writer (FileReader, FileWriter)
바이너리 파일   →  InputStream / OutputStream (FileInputStream, FileOutputStream)
네트워크 URL   →  url.openStream() + 보조 스트림 추가
```

**스트림 사용 원칙**

| 항목 | 내용 |
|---|---|
| `flush()` | 버퍼에 남은 데이터 강제 출력 — 쓰기 후 반드시 호출 |
| `close()` | 스트림 자원 해제 — `flush()` 후 호출 |
| append 모드 | `new FileWriter(path, true)` |
| 한글 출력 | `getBytes(StandardCharsets.UTF_8)` 명시 |
| 읽기 종료 조건 | `read()` 반환값이 `-1`이면 EOF |
| 버퍼 크기 | 텍스트 `1024` char / 바이너리 `4096` byte 권장 |
| 바이너리 복사 | `write(buff, 0, data)` — 마지막 블록 잘림 방지를 위해 `data`(실제 읽은 수) 전달 |

**외부 라이브러리 비교**

| 라이브러리 | 용도 | 핵심 클래스 |
|---|---|---|
| `Jsoup` | HTML 파싱, 정적 크롤링 | `Jsoup`, `Document`, `Elements` |
| `Selenium` | 브라우저 자동화, 동적 크롤링 | `WebDriver`, `By`, `JavascriptExecutor` |
| `Jackson` | JSON 파싱 | `ObjectMapper`, `JsonNode` |
