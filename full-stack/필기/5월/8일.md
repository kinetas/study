# 5월 8일 필기 — JAVA

## 핵심요약

컬렉션 프레임워크는 배열의 고정 크기 한계를 보완하기 위해 만들어진 `java.util` 패키지의 자료구조 모음으로, `List`(순서O·중복O), `Set`(순서X·중복X), `Map`(Key-Value) 세 계열로 나뉜다. `ArrayList`는 배열 기반으로 조회에 유리하고, `LinkedList`는 노드 연결 구조로 삽입·삭제에 유리하며, `Vector`는 멀티스레드 환경에서 임계영역을 설정해 동시 접근을 잠금(Lock)으로 처리한다. `HashSet`은 중복을 자동 제거하며, 커스텀 객체를 넣을 때는 `hashCode()` 먼저 비교 후 `equals()`를 비교하므로 두 메서드를 반드시 함께 오버라이드해야 한다. `HashMap`은 Key 중복 시 나중 값으로 덮어쓰며, `Map<String, Object>`로 여러 타입을 한 Map에 담을 수 있고 꺼낼 때는 `instanceof`로 타입 확인 후 다운캐스팅한다. Java Swing은 `JFrame → JPanel → Component` 계층으로 GUI를 구성하며, `setLayout(null)` 후 `setBounds()`로 절대 좌표 배치를 한다. 이벤트 처리는 리스너 인터페이스를 `implements`하고 `addXxxListener(this)`로 등록한 뒤 콜백 메서드를 오버라이드하는 3단계로 이루어지며, `ActionListener`(버튼), `KeyListener`(키보드), `MouseListener`(마우스) 세 종류를 다중 구현할 수 있다.

### 1) ArrayList — List 기본

```java
List<String> list = new ArrayList();

list.forEach(System.out::println);           // 메서드 참조 순회

list.remove(0);                 // 인덱스로 삭제
list.remove("REACT");           // 값으로 삭제
list.clear();                   // 전체 삭제
```

### 2) List 구현체 비교 — ArrayList / LinkedList / Vector

세 클래스 모두 `List` 인터페이스를 구현하므로 사용법은 동일, 내부 동작 방식만 다르다.

| 구현체 | 특징 | 권장 상황 |
|---|---|---|
| `ArrayList` | 배열 기반 — 삽입/삭제 시 뒤 요소를 밀어야 함 | 조회 중심 |
| `LinkedList` | 노드 연결 구조 — 삽입/삭제 더 빠름 | 삽입/삭제 중심 |
| `Vector` | ArrayList와 동일 + **멀티스레드 임계영역 설정** — 동시 접근 시 잠금(Lock) | 멀티스레드 환경 |

```java
// 구현체만 바꾸면 나머지 코드 동일
List<String> list = new ArrayList();
// List<String> list = new LinkedList();
// List<String> list = new Vector();  // 멀티스레드 안전
```

### 3) HashSet — Set 기본

순서 없음, **중복 불허** — 같은 값을 add해도 한 번만 저장됨.

```java
Set<String> set = new HashSet();

set.add("JAVA");
set.add("JAVA");       // 중복 → 무시됨
set.size();            // 중복 제거된 개수
set.remove("REACT");

// 탐색 — Iterator (구형)
Iterator<String> iter = set.iterator();
while (iter.hasNext()) System.out.println(iter.next());

// 탐색 — 향상된 for (최신)
for (String el : set) System.out.println(el);

set.clear();
```

### 4) 로또 번호 생성 + 정렬 (실습)

Set의 중복 불허 특성으로 고유 번호 6개를 뽑고, Stream으로 정렬.

```java
Random rand = new Random();
Set<Integer> set = new HashSet();

// 오름차순 정렬 — 스트림 + Collectors
List<Integer> list = set.stream().sorted().collect(Collectors.toList());
list.forEach(el -> System.out.print(el + " "));

// 내림차순: sorted(Comparator.reverseOrder())
// 또는 List 변환 후: Collections.sort(list)
```

### 5) HashSet + equals() & hashCode() 오버라이드

HashSet 중복 판별: `hashCode()` 먼저 비교 → 같으면 `equals()` 비교.
커스텀 객체를 Set에 넣을 때 **두 메서드를 반드시 함께 오버라이드**해야 한다.

```java
class Person {
    String name;  int age;

    @Override
    public boolean equals(Object obj) {
        if (obj instanceof Person) {
            Person down = (Person) obj;
            return this.name.equals(down.name) && this.age == down.age;
        }
        return false;
    }

    @Override
    public int hashCode() {
        return Objects.hash(this.name, this.age);  // 필드값 기반 해시
    }
}

Set<Person> set = new HashSet();
Person ob1 = new Person("홍길동", 55);
Person ob3 = new Person("홍길동", 55); // ob1과 내용 동일

set.add(ob1);
set.add(ob3); // 중복 → 추가되지 않음
System.out.println(set.size()); // 1 (ob1만 저장)
```

### 6) HashMap — Map 기본

Key-Value 쌍으로 저장, **Key 중복 불허** — 같은 Key로 put하면 나중 값으로 덮어씀.

```java
Map<String, Integer> map = new HashMap();

    System.out.println("KEY : " + key + " VALUE : " + map.get(key));
}

map.get("aaa");    // 단건 조회
map.remove("aaa"); // 삭제
map.size();        // 개수
```

### 7) Map\<String, Object\> — 다양한 타입 저장

`Object` 타입으로 선언하면 String, int, 배열, List 등 모든 타입을 한 Map에 저장 가능.
꺼낼 때는 `instanceof`로 타입 확인 후 다운캐스팅.

```java
Map<String, Object> params = new HashMap();

// 배열, List, 기본 타입 혼합 저장
params.put("자격증", new String[]{"정보처리기사", "SQLD"});
params.put("취미", Arrays.asList("등산", "게임"));
params.put("이름", "홍길동");
params.put("나이", 20);

// 꺼낼 때 instanceof로 구분
for (String key : params.keySet()) {
    Object value = params.get(key);

    if (value instanceof String[]) {
        Arrays.stream((String[]) value).forEach(el -> System.out.println("배열 : " + el));
    } else if (value instanceof List) {
        ((List) value).forEach(el -> System.out.println("리스트 : " + el));
    } else {
        System.out.println("KEY : " + key + " VALUE : " + value);
    }
}
```

### 3) JPanel + 컴포넌트 절대 배치

`setLayout(null)` 로 레이아웃 매니저를 해제하면 `setBounds()`로 절대 위치 배치.
`JTextArea`는 반드시 `JScrollPane`으로 감싸서 추가.

```java
JPanel panel = new JPanel();
panel.setLayout(null); // 절대 배치 모드

JButton btn1 = new JButton("BTN01");
btn1.setBounds(10, 10, 100, 30);  // x, y, width, height

JTextField txt1 = new JTextField();
txt1.setBounds(10, 50, 210, 30);

JTextArea area1 = new JTextArea();
JScrollPane scroll = new JScrollPane(area1); // JTextArea를 JScrollPane으로 감쌈
scroll.setBounds(10, 90, 210, 300);

panel.add(btn1);
panel.add(txt1);
panel.add(scroll); // area1이 아닌 scroll을 추가

add(panel);
setVisible(true); // 컴포넌트 추가 후 마지막에 호출
```

| 컴포넌트 | 클래스 | 용도 |
|---|---|---|
| 버튼 | `JButton` | 클릭 이벤트 |
| 한 줄 입력 | `JTextField` | 짧은 텍스트 입력 |
| 여러 줄 입력 | `JTextArea` | 채팅 로그 등 |
| 스크롤 | `JScrollPane` | JTextArea 감싸 스크롤 제공 |

### 4) ActionListener — 버튼 클릭 이벤트

이벤트 처리 3단계: **① implements → ② addListener → ③ 메서드 오버라이드**

```java
// ① 클래스에 implements
class MyGUI extends JFrame implements ActionListener {
    JButton btn1, btn2; // 필드로 선언해야 actionPerformed에서 참조 가능

    MyGUI(String title) {
        // ...
        // ② 리스너 등록
        btn1.addActionListener(this);
        btn2.addActionListener(this);
    }

    // ③ 콜백 메서드 구현
    @Override
    public void actionPerformed(ActionEvent e) {
        if (e.getSource() == btn1) {
            System.out.println("btn1 클릭");
        } else if (e.getSource() == btn2) {
            System.out.println("btn2 클릭");
        }
    }
}
```

> `e.getSource()` — 이벤트 발생 컴포넌트 반환 → `==` 로 어느 버튼인지 비교

### 5) KeyListener — 키보드 이벤트

Enter키 감지로 메시지 전송 구현.

```java
class MyGUI extends JFrame implements ActionListener, KeyListener {
    JTextField txt1;
    JTextArea area1;

    // txt1에 키 리스너 등록
    // txt1.addKeyListener(this);

    @Override
    public void keyTyped(KeyEvent e) { }    // 문자키 입력 시

    @Override
    public void keyPressed(KeyEvent e) { }  // 키를 누르는 동안

    @Override
    public void keyReleased(KeyEvent e) {   // 키를 뗐을 때
        if (e.getSource() == txt1) {
            if (e.getKeyChar() == 10) {     // Enter키 = keyChar 10
                String message = txt1.getText();  // 입력값 가져오기
                area1.append(message + "\n");     // TextArea에 추가
                txt1.setText("");                 // 입력창 초기화
            }
        }
    }
}
```

| 메서드 | 설명 |
|---|---|
| `e.getKeyChar()` | 입력된 문자 (char) — Enter는 `10` |
| `e.getKeyCode()` | 입력된 키코드 (int) |
| `txt1.getText()` | 텍스트 가져오기 |
| `area1.append(str)` | TextArea 끝에 텍스트 추가 |
| `txt1.setText("")` | 텍스트 초기화 |

### 6) MouseListener — 마우스 이벤트 + 줄 텍스트 추출

JTextArea 클릭 시 해당 줄의 텍스트를 추출하는 패턴.

```java
class MyGUI extends JFrame implements ActionListener, KeyListener, MouseListener {

    // area1.addMouseListener(this);

    @Override
    public void mouseClicked(MouseEvent e) {
        try {
            Point point = e.getPoint();
            int offset      = area1.viewToModel(point);           // 좌표 → 문자 오프셋
            int row         = area1.getLineOfOffset(offset);      // 오프셋 → 줄 번호
            int startOffset = area1.getLineStartOffset(row);      // 줄 시작
            int endOffset   = area1.getLineEndOffset(row);        // 줄 끝
            String str = area1.getText(startOffset, endOffset - startOffset);
            System.out.println(str);
        } catch (Exception e1) { e1.printStackTrace(); }
    }

    // 나머지 5개 메서드는 빈 구현 (인터페이스 강제)
    @Override public void mousePressed(MouseEvent e)  { }
    @Override public void mouseReleased(MouseEvent e) { }
    @Override public void mouseEntered(MouseEvent e)  { }
    @Override public void mouseExited(MouseEvent e)   { }
}
```

| JTextArea 줄 관련 메서드 | 설명 |
|---|---|
| `viewToModel(Point)` | 화면 좌표 → 텍스트 오프셋 |
| `getLineOfOffset(offset)` | 오프셋 → 줄 번호 |
| `getLineStartOffset(row)` | 줄 시작 오프셋 |
| `getLineEndOffset(row)` | 줄 끝 오프셋 |
| `getText(start, len)` | 오프셋 기준 텍스트 추출 |

### Ch05 핵심 정리

**Swing 구성 계층**

```
JFrame (창)
 └── JPanel (setLayout(null) → 절대 배치)
      ├── JButton
      ├── JTextField
      └── JScrollPane → JTextArea
```

**이벤트 처리 3단계**

```
① 클래스에 implements XxxListener 추가
② 컴포넌트에 addXxxListener(this) 등록
③ 인터페이스 콜백 메서드 오버라이드
```

**리스너 종류 비교**

| 리스너 | 인터페이스 | 등록 메서드 | 핵심 콜백 |
|---|---|---|---|
| 버튼 클릭 | `ActionListener` | `addActionListener(this)` | `actionPerformed(e)` |
| 키보드 | `KeyListener` | `addKeyListener(this)` | `keyReleased(e)` 등 3개 |
| 마우스 | `MouseListener` | `addMouseListener(this)` | `mouseClicked(e)` 등 5개 |

> **주의**: 이벤트 처리할 컴포넌트(`JButton` 등)는 **필드로 선언**해야 `actionPerformed()` 안에서 `e.getSource() ==` 비교 가능
