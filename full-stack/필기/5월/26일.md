# 5월 26일 필기 — JSP / Servlet

## 핵심요약

Ch05는 Session을 이용한 인증 처리를 학습했다. `request.setAttribute()`는 현재 요청 1회만 유지되어 Redirect 후 소멸되지만, `session.setAttribute()`는 세션 만료 전까지 유지되므로 로그인 상태 관리에 사용한다. `session.setMaxInactiveInterval(초)`로 유지 시간을 변경할 수 있고, `session.invalidate()`로 세션 전체를 삭제한다. 로그인 성공 시 `isAuth`·`role`을 세션에 저장하고 페이지 진입 시 `isAuth`가 없으면 로그인 페이지로 강제 이동시키는 인가(Authorization) 처리 패턴을 구현했다. 또한 메시지를 세션에 저장했다가 표시 후 즉시 `removeAttribute()`로 제거하는 **Flash Message 패턴**도 적용했다. `ROLE` enum으로 `ROLE_USER` / `ROLE_ADMIN`을 구분해 역할 기반 인가를 처리했다. Ch06은 Cookie를 학습했다. 쿠키는 클라이언트(브라우저)에 문자열만 저장하며 1개당 4KB·도메인당 20개로 제한된다. `setMaxAge(-1)`은 브라우저 종료 시 소멸, 양수는 파일로 저장, `0`은 즉시 만료(삭제)에 사용한다. 쿠키 삭제는 같은 이름·Path로 `MaxAge(0)` 쿠키를 재전송해 덮어씌우는 방식이며, `setPath()`로 쿠키가 전송될 URI 범위를 제한할 수 있다. Ch07은 내장 객체 심화와 파일 다운로드를 학습했다. `pageContext`는 모든 내장 객체의 최상위 컨텍스트이며, EL에서 컨텍스트 경로는 `${pageContext.request.contextPath}`로 접근한다. 파일 다운로드는 `out.clear()` + `pageContext.pushBody()`로 HTML 버퍼를 비운 뒤 `Content-Disposition: attachment` 헤더를 설정하고 `ServletOutputStream`으로 전송한다. ZIP 다운로드는 `ZipOutputStream`에 파일마다 `ZipEntry`를 추가하는 방식이다. Servlet 파트에서는 생명주기(`init` → `service` → `destroy`)와 등록 방식 두 가지(`@WebServlet` 어노테이션 vs `web.xml`)를 학습했으며, `web.xml`의 `url-pattern`은 반드시 `/`로 시작해야 하고 미작성 시 Tomcat 배포 자체가 실패해 프로젝트 전체가 동작 불가 상태가 된다.

## JSP01 — Ch05 Session (세션)

---

### 1) 핵심 개념

| 구분 | `request` | `session` |
|---|---|---|
| **저장 위치** | 서버 (요청 단위) | 서버 (세션 단위) |
| **유지 범위** | 현재 요청 1회 (Forward 유지) | 세션이 만료될 때까지 |
| **Redirect 후** | 소멸 | 유지 |
| **주 용도** | Forward 데이터 전달 | 로그인 상태, 사용자 정보 |

| 주요 메서드 | 설명 |
|---|---|
| `session.setAttribute(key, value)` | 세션에 값 저장 |
| `session.getAttribute(key)` | 세션 값 조회 |
| `session.removeAttribute(key)` | 특정 세션 속성 제거 |
| `session.invalidate()` | 세션 전체 삭제 (로그아웃) |
| `session.setMaxInactiveInterval(초)` | 세션 유지 시간 변경 (기본 1800초) |

### 2) Session 기본 — request vs session 범위 비교

```jsp
REQUEST t1 : ${t1 }   <!-- 같은 요청 내에서 접근 가능 -->

```

```jsp
<!-- getSession.jsp — 링크 클릭 = 새 요청 -->

SESSION t2 : ${t2 }   <!-- "v2" 그대로 유지 -->
```

### 5) auth/Login.jsp — 세션 기반 로그인 처리

```jsp
<jsp:useBean id="authDTO" class="Ch05.AuthDTO" scope="request" />
<jsp:setProperty name="authDTO" property="*" />
<jsp:useBean id="dao" class="Ch05.UserDAO" scope="request" />
<%@ page import="Ch05.UserDTO, Ch05.ROLE" %>
<%
    UserDTO userDTO = dao.select(authDTO.getUserid());

    if (userDTO == null) {                                // ① userid 없음
        session.setAttribute("isAuth", false);
        session.setAttribute("message", "⚠ 동일한 ID가 없습니다.");
        response.sendRedirect(request.getContextPath() + "/Ch05/02/login_form.jsp");
        return;
    }
    if (!authDTO.getPassword().equals(userDTO.getPassword())) {  // ② PW 불일치
        session.setAttribute("isAuth", false);
        session.setAttribute("message", "⚠ PW가 일치하지 않습니다.");
        response.sendRedirect(request.getContextPath() + "/Ch05/02/login_form.jsp");
        return;
    }

    // ③ 로그인 성공
    session.setMaxInactiveInterval(60);   // 세션 유지 시간 60초
    session.setAttribute("isAuth", true);
    session.setAttribute("role", authDTO.getUserid().equals("admin") ? ROLE.ROLE_ADMIN : ROLE.ROLE_USER);
    session.setAttribute("message", "✔ 로그인 완료!");
    response.sendRedirect(request.getContextPath() + "/Ch05/02/main.jsp");
%>
```

| 처리 순서 | 설명 |
|---|---|
| ① userid 확인 | `dao.select(userid)` → `null`이면 실패 |
| ② PW 검증 | DB 저장값과 입력값 비교 |
| ③ 세션 저장 | `isAuth`, `role`, `message` 세션에 저장 |
| ④ 이동 | 성공 → `main.jsp` / 실패 → `login_form.jsp` |

## JSP01 — Ch06 Cookie (쿠키)

---

### 1) 핵심 개념

| 특징 | 내용 |
|---|---|
| **저장 위치** | 클라이언트(브라우저) |
| **저장 데이터** | 문자열만 가능 |
| **크기 제한** | 쿠키 1개당 4KB 이하 |
| **개수 제한** | 브라우저당 최대 300개, 도메인당 20개 |
| **초과 시** | 최근에 사용되지 않은 쿠키부터 자동 삭제 |

| `setMaxAge()` 값 | 동작 |
|---|---|
| `-1` (기본값) | 파일 미생성, 브라우저 닫으면 소멸 |
| `0` | 즉시 만료 (쿠키 삭제에 사용) |
| 양수 (초) | 지정한 초 후 만료, 파일로 저장 |

---

### 2) Cookie CRUD

```jsp
<!-- setCookie.jsp : 쿠키 생성 -->
<%
    Cookie cookie1 = new Cookie("cookie1", "value1");
    cookie1.setPath("/");                   // MaxAge 미설정(-1) → 브라우저 종료 시 소멸

    Cookie cookie2 = new Cookie("cookie2", "value2");
    cookie2.setMaxAge(60 * 5);             // 5분 후 만료, 파일로 저장
    cookie2.setPath("/");

    response.addCookie(cookie1);
    response.addCookie(cookie2);
%>
```

```jsp
<!-- getCookie.jsp : 쿠키 전체 조회 -->
<%
    Cookie[] cookies = request.getCookies();   // 쿠키 없으면 null
    for (Cookie cookie : cookies) {
%>
        <%= cookie.getName() %> : <%= cookie.getValue() %> <br/>
        <a href="./deleteCookie.jsp?name=<%= cookie.getName() %>">삭제</a>
<%  } %>
```

```jsp
<!-- deleteCookie.jsp : 쿠키 삭제 -->
<%
    String cookieName = request.getParameter("name");
    if (cookieName != null) {
        Cookie cookie = new Cookie(cookieName, null);
        cookie.setMaxAge(0);      // 즉시 만료
        cookie.setPath("/");      // 생성 시와 동일한 Path 지정 필수!
        response.addCookie(cookie);
    }
%>
<script>
    alert("쿠키가 삭제되었습니다.");
    location.href = './getCookie.jsp';
</script>
```

> 쿠키 삭제 = 동일한 이름·Path 로 `MaxAge(0)` 쿠키를 재전송해 덮어씌우는 것  
> `setPath()` 가 다르면 삭제되지 않으므로 주의

---

### 3) Cookie Path 설정

```jsp
<%
    // "/" : 도메인 전체 (http://localhost:8080 내 모든 페이지)
    Cookie cookie1 = new Cookie("cookie1", "value1");
    cookie1.setMaxAge(60 * 10);
    cookie1.setPath("/");

    // "./" : 현재 디렉토리 이하 (/01_JSP/Ch06/02/*)
    Cookie cookie2 = new Cookie("cookie2", "value2");
    cookie2.setMaxAge(60 * 10);
    cookie2.setPath("./");

    // 특정 경로 : /01_JSP/Ch01/* 에서만 전송
    Cookie cookie3 = new Cookie("cookie3", "value3");
    cookie3.setMaxAge(60 * 10);
    cookie3.setPath("/01_JSP/Ch01");
%>
```

| Path | 쿠키 전송 범위 |
|---|---|
| `"/"` | 도메인 전체 |
| `"./"` | 현재 디렉토리 이하 |
| `"/01_JSP/Ch01"` | 해당 경로 이하만 |

## JSP01 — Ch07 내장 객체 심화 & 파일 다운로드

---

### 1) pageContext — 컨텍스트 경로 접근

| 방식 | 코드 | 비고 |
|---|---|---|
| 표현식 | `<%=request.getContextPath()%>` | `/01_JSP` |
| EL | `${pageContext.request.contextPath}` | 정석 |
| EL (잘못된 예) | `${request.contextPath}` | 미동작 |

> `action`, `href` 절대경로에는 `${pageContext.request.contextPath}` 를 항상 앞에 붙여야 배포 환경에서 경로가 깨지지 않음

---

### 2) request 주요 메서드 심화

```jsp
<%
    String protocol   = request.getProtocol();           // HTTP/1.1
    String serverName = request.getServerName();         // localhost
    int    serverPort = request.getServerPort();         // 8080
    String remoteAddr = request.getRemoteAddr();         // 클라이언트 IP
    String method     = request.getMethod();             // GET / POST
    StringBuffer url  = request.getRequestURL();         // http://localhost:8080/...
    String uri        = request.getRequestURI();         // /01_JSP/Ch07/...
    String browser    = request.getHeader("User-Agent");
    String fileType   = request.getHeader("Accept");
%>
```

| 메서드 | 반환값 예시 |
|---|---|
| `getRequestURL()` | 전체 URL (`StringBuffer`) |
| `getRequestURI()` | 도메인 제외 경로 (`String`) |
| `getHeader("User-Agent")` | 브라우저 정보 |
| `getHeader("Accept")` | 클라이언트 수용 MIME 타입 |

### 3) response 주요 기능


### 4) 단일 파일 다운로드

```jsp
<%@page import="java.io.*"%>
<%
    String dirPath = pageContext.getServletContext().getRealPath("/") + "Ch07\\files\\";
    InputStream fin = new FileInputStream(dirPath + "file1.pptx");

    out.clear();                    // HTML 버퍼 초기화
    out = pageContext.pushBody();   // 바이너리 전송 모드 전환

    response.setHeader("Content-Type",        "application/octet-stream;charset=utf-8");
    response.setHeader("Content-Disposition", "attachment;filename=file1.pptx");

    ServletOutputStream bout = response.getOutputStream();
    byte[] buff = new byte[4096];
    while (true) {
        int data = fin.read(buff);
        if (data == -1) break;
        bout.write(buff, 0, data);
        bout.flush();
    }
    bout.close();
    fin.close();
%>
```

| 핵심 포인트 | 설명 |
|---|---|
| `getRealPath("/")` | 웹 앱 루트의 실제 파일 시스템 경로 반환 |
| `out.clear()` + `pushBody()` | HTML 출력 버퍼를 비우고 바이너리 전송 모드로 전환 |
| `Content-Type: application/octet-stream` | 브라우저가 파일을 다운로드로 처리하게 함 |
| `Content-Disposition: attachment;filename=...` | 저장 파일명 지정 |
| 4KB 버퍼 루프 | 대용량 파일을 메모리 부담 없이 스트리밍 전송 |

---

### 5) ZIP 묶음 다운로드

```jsp
<%@page import="java.io.*, java.util.zip.*"%>
<%
    File[] fileList = new File(dirPath).listFiles();

    out.clear();
    out = pageContext.pushBody();

    ServletOutputStream bout = response.getOutputStream();
    ZipOutputStream zout = new ZipOutputStream(bout);

    response.setHeader("Content-Type",        "application/octet-stream;charset=utf-8");
    response.setHeader("Content-Disposition", "attachment; filename=ALLFILES.zip");

    for (File file : fileList) {
        FileInputStream in = new FileInputStream(file);
        zout.putNextEntry(new ZipEntry(file.getName()));  // 항목 시작
        int data;
        while ((data = in.read()) != -1)
            zout.write((byte) data);
        zout.closeEntry();   // 항목 종료
        in.close();
    }
    zout.flush();
    zout.close();
    bout.close();
%>
```

| 클래스 / 메서드 | 역할 |
|---|---|
| `ZipOutputStream` | ZIP 형식으로 데이터를 쓰는 출력 스트림 |
| `ZipEntry(name)` | ZIP 파일 내 개별 파일 항목 정의 |
| `putNextEntry(entry)` | 새 파일 항목 시작 |
| `closeEntry()` | 현재 파일 항목 종료 |

## JSP02 — Servlet 기초 & 등록 방식

---

### 1) 핵심 개념

| 개념 | 내용 |
|---|---|
| **Servlet** | `HttpServlet`을 상속받아 구현하는 서버 사이드 Java 클래스 |
| **JSP와의 관계** | JSP는 내부적으로 Servlet으로 변환·실행됨 |
| **등록 방식** | `@WebServlet` 어노테이션 또는 `web.xml` 중 하나 사용 |

---

### 2) 서블릿 생명주기 (Lifecycle)

```java
@WebServlet("/TEST_01")
public class C01Servlet_Test extends HttpServlet {

    @Override
    public void init() throws ServletException {
        // 최초 로딩 시 1회만 실행
        System.out.println("init invoke...");
    }

    @Override
    public void service(ServletRequest req, ServletResponse res)
            throws ServletException, IOException {
        // 클라이언트 요청마다 실행
        System.out.println("service invoke...");
    }

    @Override
    public void destroy() {
        // 클래스 변경 감지 / 서버 종료 시 실행
        System.out.println("destroy invoke...");
    }
}
```

| 메서드 | 호출 시점 | 호출 횟수 |
|---|---|---|
| `init()` | 서블릿 최초 로딩 시 | 1회 |
| `service()` | 클라이언트 요청마다 | 매 요청 |
| `destroy()` | 클래스 변경 감지 / 서버 종료 시 | 1회 |

---

### 3) 서블릿 등록 방식 2가지

#### 방식 1 — `@WebServlet` 어노테이션

```java
@WebServlet("/TEST_01")
public class C01Servlet_Test extends HttpServlet { ... }
```

#### 방식 2 — `web.xml` 등록

```xml
<servlet>
    <servlet-name>C02Servlet</servlet-name>
    <servlet-class>Servlet.C02Servlet_Test</servlet-class>
</servlet>
<servlet-mapping>
    <servlet-name>C02Servlet</servlet-name>
    <url-pattern>/TEST_02</url-pattern>   <!-- 반드시 / 로 시작 -->
</servlet-mapping>
```

| 구분 | `@WebServlet` | `web.xml` |
|---|---|---|
| **설정 위치** | 클래스 파일 내 | `WEB-INF/web.xml` |
| **URL 변경 시** | 코드 재컴파일 필요 | 설정 파일만 수정 |
| **적합한 상황** | 소규모, URL 고정 | 대규모, URL 중앙 관리 |

---

### 4) ⚠ url-pattern 주의사항

```xml
<!-- 잘못됨 -->
<url-pattern>TEST_02</url-pattern>

<!-- 올바름 -->
<url-pattern>/TEST_02</url-pattern>
```

> `/` 없이 작성하면 Tomcat이 web.xml 파싱 시 `IllegalArgumentException`을 던지며 **앱 전체가 배포 실패**  
> `@WebServlet` 어노테이션 방식으로 등록된 서블릿까지 모두 응답 불가 상태가 됨

---
