# 5월 7일 필기 — JAVA

## 핵심요약

`java.lang.Object`는 자바의 모든 클래스가 자동으로 상속받는 최상위 부모 클래스로, `toString()`은 `println()` 호출 시 자동으로 실행되며 오버라이드하지 않으면 주소값 형태로 출력된다. `equals()`는 기본적으로 참조 비교를 하므로 값 비교가 필요하면 오버라이드해야 하고, 이때 `hashCode()`도 반드시 함께 오버라이드해야 컬렉션에서 올바르게 동작한다. Wrapper 클래스는 기본 자료형을 객체로 다루기 위한 클래스이며 Java 5부터 AutoBoxing/AutoUnBoxing으로 대부분 자동 변환된다. 날짜 API는 `Date` → `Calendar` → `LocalDateTime` 순으로 발전했으며 최신 코드는 `LocalDateTime`과 `DateTimeFormatter`를 사용한다. 예외 처리는 `try-catch-finally` 구조로 프로그램 종료를 방지하며, `throw`로 직접 예외를 발생시키고 `throws`로 호출자에게 위임할 수 있다. 제네릭은 타입을 파라미터로 받아 컴파일 타임에 타입 안전성을 보장하며, `<T extends 상위클래스>`로 허용 타입을 제한하고 `?` 와일드카드로 유연한 수용이 가능하다. 단 제네릭은 불공변이라 `Box<Object>`와 `Box<String>`은 상속 관계가 없다.

## JAVA03 — Ch01 Object 클래스 & Wrapper & 날짜

---

### 1) Object 클래스 — toString()

`java.lang.Object`는 모든 클래스의 최상위 부모 — `toString()`, `equals()`, `hashCode()` 등 공통 메서드 제공.

```java
class A {
    int x, y;
    @Override
    public String toString() { return "A [x=" + x + ", y=" + y + "]"; }
}

A ob1 = new A();
System.out.println(ob1.toString()); // "A [x=0, y=0]"
System.out.println(ob1);            // println이 toString() 자동 호출 — 결과 동일

Object ob2 = new Object();
System.out.println(ob2);            // 기본 toString() → "패키지.클래스명@해시코드"
```

> 오버라이드 안 하면 주소값처럼 보이는 문자열 출력

---

### 2) Object 클래스 — equals()

기본 `equals()`는 **참조(주소) 비교** — 값 비교가 필요하면 오버라이드.

```java
@Override
public boolean equals(Object obj) {
    if (obj instanceof C02Simple) {
        C02Simple down = (C02Simple) obj;  // instanceof 확인 후 다운캐스팅
        return down.n == this.n;           // 필드값 비교
    }
    return false;
}

C02Simple ob1 = new C02Simple(10);
C02Simple ob2 = new C02Simple(20);
C02Simple ob3 = new C02Simple(10);
ob1.equals(ob2); // false
ob1.equals(ob3); // true — 값이 같음
```

---

### 3) Object 클래스 — equals() + hashCode()

`equals()`를 오버라이드하면 **`hashCode()`도 반드시 함께 오버라이드** — equals가 true이면 hashCode도 같아야 한다는 규약.

```java
@Override
public boolean equals(Object obj) {
    if (obj instanceof C03Simple)
        return ((C03Simple) obj).n == this.n;
    return false;
}

@Override
public int hashCode() {
    return Objects.hash(this.n);  // 필드값 기반 해시 생성
}

// ob1(n=10)과 ob3(n=10)은 해시코드 동일
// ob2(n=20)은 다른 해시코드
System.out.printf("%x\n", ob1.hashCode()); // == ob3.hashCode()
```

| 메서드 | 기본 동작 | 오버라이드 이유 |
|---|---|---|
| `toString()` | 클래스명@해시코드 | 객체 내용을 읽기 좋게 출력 |
| `equals()` | 참조 비교 | 필드값 기준 동등성 판단 |
| `hashCode()` | 주소 기반 해시 | equals 규약 유지, 컬렉션 필수 |

### 4) Wrapper 클래스

기본 자료형을 객체로 다룰 때 사용 — `int → Integer`, `double → Double` 등.

```java
// Boxing
Integer ob1 = new Integer(100);       // deprecated
Integer ob3 = Integer.valueOf("300"); // 권장

// UnBoxing
int n1 = ob1.intValue();

// AutoBoxing / AutoUnBoxing (Java 5+)
Integer ob4 = 100;               // AutoBoxing: int → Integer
int n4 = ob1 + ob3 + ob4;        // AutoUnBoxing: Integer → int
```

| 구분 | 방향 | 예시 |
|---|---|---|
| Boxing | `int` → `Integer` | `Integer.valueOf(100)` |
| UnBoxing | `Integer` → `int` | `ob.intValue()` |
| AutoBoxing | 자동 | `Integer ob = 100;` |
| AutoUnBoxing | 자동 | `int n = ob1 + ob2;` |

### 5) 날짜·시간 API 비교

| API | 월 보정 | 요일 범위 | 권장 |
|---|---|---|---|
| `Date` | +1 필요 | 0~6 (일~토) | X (deprecated) |
| `Calendar` | +1 필요 | 1~7 (일~토) | △ |
| `LocalDateTime` | 불필요 (1~12) | DayOfWeek 열거형 | O |

```java
// Date (구버전)
Date d1 = new Date();
d1.getYear() + 1900;  d1.getMonth() + 1;  d1.getDay();

// Calendar (구버전)
Calendar cal = Calendar.getInstance();
cal.get(Calendar.YEAR);  cal.get(Calendar.MONTH) + 1;  cal.get(Calendar.DAY_OF_WEEK);

// LocalDateTime (권장)
LocalDateTime now = LocalDateTime.now();
now.getYear();  now.getMonthValue();  now.getDayOfWeek();  // 보정 불필요
```

### 6) SimpleDateFormat & DateTimeFormatter

```java
// SimpleDateFormat — Date 용 (구버전)
SimpleDateFormat fmtin  = new SimpleDateFormat("yyyy/MM/dd");
Date date = fmtin.parse("2025/05/07");          // 문자열 → Date
SimpleDateFormat fmtout = new SimpleDateFormat("yyyy~MM~dd");
fmtout.format(date);                            // "2025~05~07"
// parse()는 ParseException → throws 필요

// DateTimeFormatter — LocalDateTime 용 (권장)
DateTimeFormatter fmtin  = DateTimeFormatter.ofPattern("yyyy/MM/dd HH:mm:ss");
LocalDateTime date = LocalDateTime.parse("2025/05/07 14:30:00", fmtin);
DateTimeFormatter fmtout = DateTimeFormatter.ofPattern("yyyy-MM-dd HH-mm-ss");
fmtout.format(date);                            // "2025-05-07 14-30-00"
```

| 구분 | `SimpleDateFormat` | `DateTimeFormatter` |
|---|---|---|
| 대상 | `Date` | `LocalDate` / `LocalDateTime` |
| Thread-safe | X | O |
| 파싱 | `fmtin.parse(str)` | `LocalDate.parse(str, fmtin)` |

## JAVA03 — Ch02 예외 처리 (Exception Handling)

---

### 1) try-catch 기본 & 예외 객체 정보

예외가 발생해도 프로그램이 종료되지 않고 이후 코드가 계속 실행된다.

```java
try {
    String str = null;
    str.toString();           // NullPointerException 발생
} catch (NullPointerException e) {
    e.printStackTrace();      // 발생 위치·원인 콘솔 출력 (가장 많이 사용)
    e.getMessage();           // 예외 메시지
    e.getCause();             // 원인 객체
    e.getStackTrace();        // StackTraceElement 배열
}
System.out.println("이후 코드 정상 실행");
```

---

### 2) finally

`finally`는 예외 발생 여부와 **무관하게 항상 실행** — 자원 해제에 사용.

```java
try {
    int[] arr = {10, 20, 30};
    System.out.println(arr[3]); // ArrayIndexOutOfBoundsException
} catch (ArrayIndexOutOfBoundsException e) {
    e.printStackTrace();
} finally {
    System.out.println("항상 실행");  // 예외 발생해도 실행
}
```

| 블록 | 실행 조건 |
|---|---|
| `try` | 예외 없으면 끝까지 |
| `catch` | 해당 예외 발생 시 |
| `finally` | 항상 |

---

### 3) 중첩 try-catch & 다중 catch

```java
try {
    int n1 = sc.nextInt();  int n2 = sc.nextInt();
    try {
        System.out.println(n1 / n2);       // n2==0 → ArithmeticException
    } catch (ArithmeticException e) {
        if (n2 == 0) System.out.println("n2가 0");
    }
} catch (InputMismatchException e) {       // 정수 아닌 값 입력
    System.out.println("잘못된 값");
} finally {
    sc.close();
}
```

> 안쪽 try에서 처리 못한 예외 → 바깥 catch로 전달
> catch 순서: **구체적인 예외 먼저 → 상위 예외 나중**

### 4) ClassCastException & Exception 일괄 처리

```java
// ClassCastException — UpCasting 없이 강제 다운캐스팅 시 발생
Animal tori = new Cat();
Dog down = (Dog) tori;   // ClassCastException
// 안전한 방법: if (tori instanceof Dog) { ... }

// Exception으로 일괄 처리
try {
    Animal ob = new Dog();
    Cat down2 = (Cat) ob;  // ClassCastException
} catch (Exception e) {    // 모든 예외를 한 번에
    e.printStackTrace();
}
```

### 5) throw & throws

```java
class A {
    void func() throws NullPointerException {  // throws: 호출자에게 예외 위임
        String str = null;
        str.toString();                        // 예외 발생 지점
    }
}
class B {
    void func() throws Exception {
        throw new ArithmeticException();       // throw: 예외 직접 발생
    }
}

try {
    new B().func();
} catch (NullPointerException e1) { e1.printStackTrace(); }  // 구체적 먼저
  catch (Exception e2)            { e2.printStackTrace(); }  // 상위 나중
```

| 키워드 | 위치 | 역할 |
|---|---|---|
| `throw` | 메서드 본문 | 예외 객체 직접 발생 |
| `throws` | 메서드 선언부 | 예외 처리를 호출자에게 위임 |

## JAVA03 — Ch03 제네릭 (Generics)

---

### 1) 제네릭 클래스 — 상한 제한 bounded type

`<T extends 상위클래스>` — T에 들어올 수 있는 타입을 상위 클래스의 하위 타입으로 제한.

```java
class 호빵<T extends 호빵재료> {  // 팥·야채·피자 OK, 민트초코 X
    private T 재료;
    호빵(T 재료) { this.재료 = 재료; }
}

호빵<팥>  ob1 = new 호빵<>(new 팥());   // OK
// 호빵<민트초코> ob = ...;             // 컴파일 에러 — 호빵재료 하위 타입 아님
```

> `<T>` 만 쓰면 모든 타입 허용, `<T extends X>` 로 범위 제한

---

### 2) 다중 타입 파라미터

```java
class Couple<M extends Person, W extends Person> {
    private M role_man;
    private W role_woman;
    Couple(M m, W w) { this.role_man = m; this.role_woman = w; }
}

Couple<Man, Woman> couple1 = new Couple<>(new Man("철수", 20), new Woman("영희", 20));
Couple<Man, Man>   couple2 = new Couple<>(new Man("A", 55),   new Man("B", 40));
```

---

### 3) 제네릭 메서드

반환 타입 앞에 `<T>` 를 선언 — 메서드 단위로 타입 파라미터 적용.

```java
public static <T extends Animal> void PrintGeneric(T[] arr) {
    for (T el : arr) System.out.println(el);
}
public static void PrintByObject(Object[] arr) {
    for (Object el : arr) System.out.println(el);
}

PrintGeneric(new Tiger[]{ ... });   // OK
PrintGeneric(new Panda[]{ ... });   // OK
// PrintGeneric(new Object[]{ ... }); // 컴파일 에러 — Object[]는 Animal[] 아님
PrintByObject(new Object[]{ ... }); // OK — 타입 안전성 없음
```

| 방식 | 타입 안전성 | 사용 제한 |
|---|---|---|
| `<T extends Animal> void f(T[])` | O | Animal 하위 타입 배열만 |
| `void f(Object[])` | X | 모든 배열 |

---

### 4) 와일드카드 & 불공변성

제네릭은 **불공변(invariant)** — `Box<Object>`와 `Box<String>`은 상하 관계 없음.

```java
Box<?> b1 = new Box<Object>();           // OK — ? 는 모든 타입 수용
Box<?> b2 = new Box<String>();           // OK
// Box<Object> b = new Box<Fruit>();     // 컴파일 에러 — 불공변
Box<? extends Fruit> b3 = new Box<Apple>(); // OK — 상한 와일드카드
// new Box<? extends Fruit>();           // 컴파일 에러 — new에 와일드카드 불가
```

**와일드카드 vs 제네릭 메서드 (merge 예시)**

```java
// ? extends — 읽기만 가능, addAll 불가 (PECS 규칙)
public static ArrayList<? extends Product> merge(...) {
    // newList.addAll(list2); // 컴파일 에러
}

// T로 통일 — 읽기·쓰기 모두 가능
public static <T extends Product> ArrayList<T> merge(ArrayList<T> l1, ArrayList<T> l2) {
    ArrayList<T> result = new ArrayList<>(l1);
    result.addAll(l2); // OK
    return result;
}
```

| 문장 | 결과 |
|---|---|
| `Box<Object> b = new Box<Fruit>()` | 컴파일 에러 (불공변) |
| `Box<?> b = new Box<Object>()` | OK |
| `Box<? extends Fruit> b = new Box<Apple>()` | OK |
| `new Box<? extends Fruit>()` | 컴파일 에러 (new에 와일드카드 불가) |

### 5) 실용 예제 — Generic + Reflection + JDBC

제네릭, Reflection, JDBC를 조합해 JPA Repository 패턴을 직접 구현.
Entity 클래스를 받으면 생성자에서 자동으로 `CREATE TABLE`을 실행한다.

```
Entity 클래스 전달 → Reflection으로 필드 목록 추출
→ 필드 타입에 따라 SQL 자료형 결정 → JDBC로 CREATE TABLE 실행
```

```java
// Reflection — 런타임에 클래스 구조 읽기
Class<?> clazz  = Class.forName("Ch03.A");
Field[]  fields = clazz.getDeclaredFields();
for (Field field : fields) {
    field.getType().toString(); // 타입명
    field.getName();            // 필드명
}
```

```java
// Generic Repository — E: Entity 타입, D: PK 자료형
public class C05JPARepository<E, D> {
    public C05JPARepository(E entity, Class<?> dataType) {
        Class<?>  clazz  = Class.forName(entity.getClass().getName());
        Field[]   fields = clazz.getDeclaredFields();

        String sql = "CREATE TABLE IF NOT EXISTS tmpdb." + clazz.getSimpleName() + "(";
        for (Field field : fields) {
            if (field.getType().toString().contains("String"))
                sql += field.getName() + " varchar(45) not null,";
            else if (field.getType().toString().contains("int"))
                sql += field.getName() + " int not null,";
        }
        sql = sql.substring(0, sql.lastIndexOf(",")) + ")";
        conn.prepareStatement(sql).executeUpdate();
    }
}

// 각 Entity별 Repository
class MemberRepository  extends C05JPARepository<Member,  String>  { ... }
class ProductRepository extends C05JPARepository<Product, Integer> { ... }

// 인스턴스 생성 시 생성자에서 테이블 자동 생성
public static MemberRepository memberRepository = new MemberRepository(new Member(), String.class);
```

> 실제 JPA가 `@Entity` 클래스를 보고 테이블을 자동 생성하는 방식과 동일한 원리
