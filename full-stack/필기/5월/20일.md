# 5월 20일 필기 — JAVA

## 핵심요약

Ch20은 스타크래프트 유닛·건물 시스템을 OOP로 모델링한 종합 실습이다. `Unit`·`Gun`·`Building`을 추상 클래스로 선언해 공통 상태와 강제 구현 메서드를 정의하고, `Marine/Medic`·`Pistol/Rifle`이 이를 상속한다. `ICanFly`·`UnitGenerator` 인터페이스로 행동 규약을 분리하며, `Barrack`은 `Building`을 상속하면서 동시에 `ICanFly`를 구현한다. 유닛 생성은 `Map<String, UnitGenerator>`에 구현체를 등록하는 팩토리 패턴으로 처리하고, `genUnit(Class<?>, int)` 메서드는 리플렉션(`Constructor.newInstance()`)으로 클래스명 문자열만으로 N개의 인스턴스를 동적 생성한다. `Main_01`은 `Thread` 익명 클래스 3개를 생성해 medic→marine1 힐링(1.5초), marine1↔marine2 상호 공격(1초)을 동시에 실행하는 멀티스레드 전투 시뮬레이션이다. 클래스 다이어그램에서는 UML 관계 3종을 학습했다. **연관**은 클래스를 멤버 변수로 보유하는 지속적 관계(Person→Car), **집합/집약**은 포함 관계로 합성(Car→Radio, filled diamond — 생성자에서 직접 new)과 집약(Person←Car, empty diamond — 외부에서 주입)을 구분하며, **의존**은 메서드 파라미터로 잠깐 사용할 뿐 멤버로 두지 않는 관계(Person.move(Car car), dashed arrow)다.

### 4) unit/Marine.java — Unit 상속

> `총기사용` / `기본공격` 상수로 공격 방식을 선택, `attack(Unit)` 위임

```java
package Ch20.unit;

import Ch20.weapon.Gun;

public class Marine extends Unit {

    int base_damage;
    Gun myGun;
    int attackMethod;

    public final static int 총기사용 = 1;
    public final static int 기본공격 = 2;

    public Marine() {
        this.hp = 100;
        this.amor = 100;
        this.type = "해병";
        this.base_damage = 10;
    }

    public void attackMethod(int 공격방식) { this.attackMethod = 공격방식; }

    public void attack(Unit unit) {
        if (!unit.isDead) {
            if (this.attackMethod == this.총기사용)
                myGun.fire(unit);
            else
                unit.underAttack(this.base_damage);
        }
    }

    @Override
    public void move() { System.out.println(this.type + " 이 이동합니다"); }

    @Override
    public void underAttack(int damage) {
        // amor → hp → dead 순서로 피해 적용
        if (amor - damage > 0)
            amor -= damage;
        else if (hp - (amor - damage) > 0) {
            hp = hp - (damage - amor);
            amor = 0;
        } else {
            hp = 0; amor = 0;
            isDead = true;
            System.out.println("[경고] " + this.type + "전사했습니다");
        }
    }

    public Gun getMyGun()           { return myGun; }
    public void setMyGun(Gun myGun) { this.myGun = myGun; }
}
```

> 피해 흐름: **amor 먼저 감소 → amor 소진 후 hp 감소 → hp 0 이면 isDead = true**

### 6) weapon/Gun.java & 구현체

```java
// 추상 클래스
public abstract class Gun {
    int maxBuillitCnt;
    int curBuillitCnt;
    int power;

    abstract public void fire(Unit unit);
    abstract public void reload(int bullit);
}

// Pistol — power 10, 탄창 10
public class Pistol extends Gun {
    public Pistol() { this.power = 10; this.curBuillitCnt = 0; this.maxBuillitCnt = 10; }

    @Override
    public void fire(Unit unit) {
        if (curBuillitCnt == 0) { System.out.println("[경고] 재장전이 필요합니다"); return; }
        unit.underAttack(this.power);
        curBuillitCnt--;
        System.out.println("[정보] " + unit.getType() + " 에게 " + this.power
            + " 만큼 피해를 입혔습니다. hp : " + unit.getHp() + " amor : " + unit.getAmor());
    }

    @Override
    public void reload(int bullit) {
        if (this.maxBuillitCnt > (bullit + this.curBuillitCnt))
            this.curBuillitCnt = maxBuillitCnt;
        else
            this.curBuillitCnt += bullit;
        System.out.println("[정보] 재장전 완료");
    }
}

// Rifle — power 50, 탄창 100 (구조 동일, 수치만 다름)
```

### 8) building/Barrack.java — HashMap 팩토리 + 리플렉션

```java
package Ch20.building;

import java.lang.reflect.Constructor;
import java.util.*;
import Ch20.unit.*;

public class Barrack extends Building implements ICanFly {

    // Key: 유닛 이름,  Value: 생성 인터페이스 구현체
    public Map<String, UnitGenerator> generator;

    public Barrack() {
        generator = new HashMap();
        generator.put("marine", new MarineGenerator());
        generator.put("medic",  new MedicGenerator());
    }

    public Marine genMarine() {
        return (Marine) ((MarineGenerator) generator.get("marine")).Gen();
    }
    public Medic genMedic() {
        return (Medic) ((MedicGenerator) generator.get("medic")).Gen();
    }

    // 리플렉션: Class<?> 와 개수(n)를 받아 동적으로 유닛 N개 생성
    public <T extends Unit> List<Unit> genUnit(Class<?> clazz, int n) {
        List<Unit> list = new ArrayList<>();
        try {
            Constructor con = clazz.getConstructor(null); // 기본 생성자 획득
            for (int i = 0; i < n; i++) {
                Object obj = con.newInstance(null);        // 인스턴스 생성
                list.add((Unit) obj);
            }
        } catch (Exception e) { e.printStackTrace(); }
        return list;
    }

    @Override public void BuildStructure() { System.err.println("Barrack 건물 건설"); }
    @Override public void UnderAttack(int damage) { System.err.println("Barrack 공격 받음"); }
    @Override public void Move() {}
    @Override public void Land() {}
    @Override public void Fly() {}
}
```

| 핵심 | 설명 |
|---|---|
| `generator.put("marine", new MarineGenerator())` | 이름 → 생성기 등록 |
| `generator.get("marine").Gen()` | 이름으로 생성기 조회 → 유닛 생성 |
| `clazz.getConstructor(null)` | 리플렉션으로 기본 생성자 획득 |
| `con.newInstance(null)` | 생성자 실행 → 인스턴스 반환 |
| `Class.forName("Ch20.unit.Marine")` | 문자열 클래스명 → Class 객체 |

## UML 클래스 다이어그램 — 관계 3종

---

### 1) 핵심 개념

| 관계 | 표기 | 설명 | 코드 특징 |
|---|---|---|---|
| **연관 (Association)** | ──▷ | 클래스를 멤버로 보유, 지속적 관계 | 필드로 선언 |
| **집합 (Composition)** | ◆── | 강한 포함 — 부모가 자식 생성·소유 | 생성자 내부에서 `new` |
| **집약 (Aggregation)** | ◇── | 약한 포함 — 외부에서 주입, 독립 존재 | 메서드 파라미터로 주입 |
| **의존 (Dependency)** | - - ▷ | 잠깐 사용, 멤버로 두지 않음 | 메서드 파라미터로만 사용 |

---

### 2) 연관 (Association) — 멤버로 보유

> 연관된 클래스를 **멤버 변수**로 두어 지속적 관계 형성

```
[예시] 사람이 자동차를 소유하다
Person ──▷ Car
```

```java
class Person {
    Car myCar;              // 연관 — Car를 멤버로 보유
    void setMyCar(Car car) { this.myCar = car; }
}
```

| 다중성 표기 | 의미 |
|---|---|
| `0..1` | 0 또는 1개 |
| `1` | 정확히 1개 |
| `0..*` | 0개 이상 |
| `1..*` | 1개 이상 |

**연관 예시 3가지**

| 예시 | 관계명 | 다중성 |
|---|---|---|
| Person → Car | 소유 | Person 0..* — Car 0..* |
| Person → Computer | 구매 | Person 0..* — Computer 0..1 |
| Student → Lecture | 수강신청 | Student 0..* — Lecture 0..* |

---

### 3) 집합 (Composition, 합성) vs 집약 (Aggregation)

> 둘 다 포함 관계지만 **생명주기 공유 여부**로 구분

| 구분 | 집합 (Composition) | 집약 (Aggregation) |
|---|---|---|
| **다이아몬드** | ◆ (채워짐) | ◇ (비워짐) |
| **생명주기** | 부모가 자식 생성·파괴 | 자식이 독립적으로 존재 |
| **코드** | 생성자 내부에서 `new` | 메서드 파라미터로 주입 |
| **예시** | Car ◆── Radio | ChargingPort ◇── Phone |

```java
// 집합 (Composition) — Car가 Radio를 직접 생성·소유
class Car {
    Radio radio;
    ChargingPort port1;

    Car() {
        radio = new Radio();          // 내부에서 직접 생성
        port1 = new ChargingPort();
    }
}

// 집약 (Aggregation) — ChargingPort는 Phone을 외부에서 받아 사용
class ChargingPort {
    Phone phone;

    void connect(Phone phone) {       // 외부에서 주입 (Phone은 독립 존재)
        this.phone = phone;
    }
}

// 집약 (Aggregation) — Person은 Car를 외부에서 받음
class Person {
    Car myCar;
    void setMyCar(Car car) { this.myCar = car; }  // 외부에서 주입
}
```

### 4) 의존 (Dependency) — 잠깐 사용

> 클래스를 **멤버로 두지 않고** 메서드 파라미터로만 사용 — 점선 화살표

```
[예시] 사람이 자동차를 이용해서 이동한다
Person - - ▷ Car
```

```java
class Person {
    // Car 필드 없음 — 멤버로 두지 않음
    void move(Car car) {    // 메서드 파라미터로 잠깐 사용
        car.drive();
    }
}
```

> 연관과 의존의 차이: 연관은 `Car myCar` 필드로 상태를 유지 / 의존은 `move(Car car)` 처럼 파라미터로만 전달

### 5) 상속 예시 다이어그램

> `직원 <<abstract>>` → `사람` 상속 / `계약직·정규직` → `직원` 상속

```
사람
 ▲
직원 <<abstract>>
 ├── 계약직
 └── 정규직
```

```java
class 사람 { String name; int age; String addr; }

abstract class 직원 extends 사람 {
    abstract void setSalary(type param);
}

class 계약직 extends 직원 { @Override void setSalary(type param) { ... } }
class 정규직 extends 직원 { @Override void setSalary(type param) { ... } }
```
