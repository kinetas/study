# 5월 15일 필기 — JAVA

## 핵심요약

람다는 `(매개변수) -> { 본문 }` 형태로 익명 함수를 간결하게 표현하며, 추상 메서드가 1개인 함수형 인터페이스에 대입된다. 메서드 참조(`클래스::메서드`)는 람다를 더 짧게 쓰는 문법으로, 정적 메서드·특정 인스턴스·임의 인스턴스·생성자 참조 4가지 유형이 있다. 클로저는 람다가 외부 스코프 변수를 캡처하는 것으로, 지역 변수는 effectively final이어야 하며 배열 트릭·인스턴스 변수·static 변수로 가변 상태를 우회할 수 있다. 스트림은 Java 8에서 도입된 파이프라인 방식의 컬렉션 처리 API로, `filter/map/flatMap/distinct/sorted/limit/skip` 같은 중간 연산과 `forEach/collect/reduce/count/min/max/anyMatch/allMatch/noneMatch/findFirst` 같은 최종 연산으로 구성되며 최종 연산이 호출될 때 비로소 지연(lazy) 실행된다.

## JAVA03 — Ch16 람다(Lambda) & 메서드 참조(Method Reference)

---

### 1) 핵심 개념

| 개념 | 설명 |
|---|---|

| **메서드 참조** | 람다를 더 짧게 쓰는 문법 — `클래스::메서드` |

### 2) C01 — 람다로 List 정렬 & 순회

```java
List<Person> list = new ArrayList<>();
list.add(new Person("홍길동", 55, "대구"));
list.add(new Person("남길동", 45, "창원"));
list.add(new Person("서길동", 35, "울산"));

// 나이 오름차순 정렬 (람다)
list.sort((a, b) -> { return a.getAge() - b.getAge(); });

// forEach 순회 — 4가지 동등한 표현
list.forEach((el) -> { System.out.println(el); }); // 완전형
list.forEach(el -> { System.out.println(el); });    // 매개변수 괄호 생략
list.forEach(el -> System.out.println(el));         // 중괄호 생략
list.forEach(System.out::println);                  // 메서드 참조
```

**람다 축약 규칙**

| 조건 | 생략 가능 항목 |
|---|---|
| 매개변수 1개 | `( )` 괄호 |
| 본문이 한 줄 | `{ }` 중괄호 + `return` |
| 본문이 기존 메서드 호출 1번 | `클래스::메서드` 메서드 참조로 대체 |

### 4) C03 — 함수형 인터페이스 활용 예제 (사칙연산)

```java
interface Calculator {
    int calculate(int num1, int num2);
}

Calculator add = (a, b) -> { return a + b; };
Calculator sub = (a, b) -> { return (a > b) ? a - b : b - a; };
Calculator mul = (a, b) -> { return a * b; };
Calculator div = (a, b) -> { return a / b; };

System.out.println(add.calculate(10, 20));  // 30
System.out.println(sub.calculate(30, 10));  // 20
System.out.println(mul.calculate(10, 20));  // 200
System.out.println(div.calculate(100, 5));  // 20
```

### 5) C04 — 메서드 참조 4가지 유형

```java
// (1) 정적 메서드 참조 — 클래스::정적메서드
StrToInt t1 = Integer::parseInt;

// (2) 특정 인스턴스 메서드 참조 — 인스턴스::메서드
StrConsumer t2 = System.out::println;

// (3) 임의 인스턴스 메서드 참조 — 클래스::인스턴스메서드
StrToStr t3 = String::toUpperCase;   // 호출 대상 자체가 매개변수

// (4) 생성자 참조 — 클래스::new
PersonFactory  t4 = C04Person::new;  // 기본 생성자
ListFactory    t5 = ArrayList::new;  // 제네릭 컬렉션도 동일
```

**메서드 참조 유형 정리**

| 유형 | 문법 | 예시 |
|---|---|---|
| 정적 메서드 | `클래스::정적메서드` | `Integer::parseInt` |
| 특정 인스턴스 메서드 | `인스턴스::메서드` | `System.out::println` |
| 임의 인스턴스 메서드 | `클래스::인스턴스메서드` | `String::toUpperCase` |
| 생성자 | `클래스::new` | `ArrayList::new` |

### 6) C05/C06 — 클로저 (Closure)

> 람다가 자신이 선언된 **외부 스코프의 변수를 캡처(capture)** 하는 것

```java
// (1) effectively final — 캡처 후 변경 불가
String str = "Hello?";
Greeter t1 = () -> { return str; };   // str 캡처 (이후 str 변경 시 컴파일 에러)

// (2) 메서드가 람다를 반환 → 클로저
private static Greeter makeGreeter(String name) {
    return () -> { return "Closure : " + name; };  // name 캡처
}
Greeter t3 = makeGreeter("hihi");    // Closure : hihi
Greeter t4 = makeGreeter("hello");   // Closure : hello  (독립)
```

**가변 상태 우회 방법**

```java
// 배열 트릭 — 참조는 final, 내용은 변경 가능
int[] n = {0};
Task addOne = () -> { n[0]++; };

// 인스턴스 변수 / static 변수 — final 제약 없음
Task addOne2 = () -> { ob.n++; };
Task addOne3 = () -> { C06ClosureBasic.n++; };
```

| 방법 | 특징 |
|---|---|
| `int[] n = {0}` 배열 트릭 | 지역 변수 final 제약 우회, 람다별 독립 공간 |
| 인스턴스 변수 (`ob.n`) | 객체 공유 — 여러 람다가 같은 값 참조 |
| `static` 변수 | 클래스 전역 공유 |

## JAVA03 — Ch17 스트림 (Stream)

---

### 1) 핵심 개념

| 개념 | 설명 |
|---|---|
| **중간 연산** | 스트림 변환·필터링. 최종 연산 전까지 실행되지 않음 (lazy) |
| **최종 연산** | 파이프라인을 소비해 결과(값/컬렉션/boolean) 반환. 1회만 사용 가능 |

```
컬렉션 → .stream() → [중간 연산...] → 최종 연산
```

---

### 2) 중간 연산

| 메서드 | 설명 |
|---|---|
| `filter(Predicate)` | 조건에 맞는 요소만 통과 ★ |
| `map(Function)` | 요소를 다른 형태로 변환 ★ |
| `flatMap(Function)` | 각 요소를 스트림으로 변환 후 하나로 평면화 |
| `distinct()` | 중복 제거 (equals/hashCode 기준) |
| `sorted()` / `sorted(Comparator)` | 자연 순서 또는 사용자 지정 정렬 |
| `limit(n)` | 앞에서 n개만 통과 |
| `skip(n)` | 앞 n개 건너뛰기 |

---

### 3) filter / map / flatMap

```java
// filter + map
List<Integer> li3 = li1.stream()
    .filter(n -> n % 2 == 1)   // 1, 3, 5
    .map(n -> n * n)            // 1, 9, 25
    .collect(Collectors.toList());

// 객체 타입 변환 (Person → Employee)
List<Employee> employees = persons.stream()
    .map(Employee::new)
    .collect(Collectors.toList());

// flatMap — List<List<Integer>> → List<Integer>
List<Integer> flat = numbers.stream()
    .flatMap(List::stream)
    .collect(Collectors.toList());   // [1,2,3,4,5,6]

// 문장 → 단어 분리
List<String> words = sentences.stream()
    .flatMap(s -> Arrays.stream(s.split(" ")))
    .collect(Collectors.toList());
```

> `map`은 1:1 변환, `flatMap`은 1:N 변환 후 하나의 스트림으로 합침

---

### 4) distinct / sorted / limit / skip

```java
// distinct — equals/hashCode 없으면 객체 주소 기준이라 의도대로 동작 안 할 수 있음
nums.stream().distinct()

// sorted
nums.stream().sorted()                          // 오름차순
nums.stream().sorted(Comparator.reverseOrder()) // 내림차순
nums.stream().sorted((a, b) -> b - a)          // 람다 Comparator
products.stream()
    .sorted(Comparator
        .comparingInt((Product p) -> p.price)
        .thenComparing(p -> p.name))            // 다중 정렬

// limit + skip → 페이지네이션
list.stream()
    .skip((long)(page - 1) * pageSize)
    .limit(pageSize)
    .collect(Collectors.toList());

// 무한 스트림 종료
Stream.iterate(0, n -> n + 2).limit(5)         // [0, 2, 4, 6, 8]
```

---

### 5) 최종 연산

| 메서드 | 반환 | 설명 |
|---|---|---|
| `forEach(Consumer)` | void | 각 요소 반복 처리 ★ |
| `collect(Collector)` | T | 컬렉션으로 수집 ★ |
| `reduce(BinaryOperator)` | `Optional<T>` | 누적해서 하나로 줄이기 ★ |
| `count()` | long | 개수 |
| `min(Comparator)` | `Optional<T>` | 최솟값 |
| `max(Comparator)` | `Optional<T>` | 최댓값 |
| `anyMatch(Predicate)` | boolean | 하나라도 조건 만족하면 true |
| `allMatch(Predicate)` | boolean | 모두 조건 만족하면 true |
| `noneMatch(Predicate)` | boolean | 모두 조건 불만족하면 true |
| `findFirst()` | `Optional<T>` | 첫 번째 요소 |
| `findAny()` | `Optional<T>` | 임의의 요소 |

```java
// collect
List<String>  list   = names.stream().collect(Collectors.toList());
Set<String>   set    = names.stream().collect(Collectors.toSet());
String        joined = names.stream().collect(Collectors.joining(", "));
Map<Integer, List<String>> byLen = names.stream()
    .collect(Collectors.groupingBy(String::length));

// reduce
Optional<Integer> sumOpt  = nums.stream().reduce((a, b) -> a + b);
int sumWithSeed = nums.stream().reduce(0, (a, b) -> a + b);  // 시드 제공 시 Optional 아님

// 매치
boolean hasEven     = nums.stream().anyMatch(n -> n % 2 == 0);
boolean allPositive = nums.stream().allMatch(n -> n > 0);
boolean noneNeg     = nums.stream().noneMatch(n -> n < 0);
```

### 6) 종합 예제 — 직원 데이터 처리

```java
// IT 부서 이름순 정렬
List<String> itNames = emps.stream()
    .filter(p -> p.dept.equals("IT"))
    .map(p -> p.name)
    .sorted()
    .collect(Collectors.toList());

// 부서별 평균 연봉
Map<String, Double> deptAvg = emps.stream()
    .collect(Collectors.groupingBy(p -> p.dept,
             Collectors.averagingInt(p -> p.salary)));

// 연봉 상위 3명
List<String> top3 = emps.stream()
    .sorted((a, b) -> b.salary - a.salary)
    .limit(3)
    .map(p -> p.name)
    .collect(Collectors.toList());

// 전체 연봉 합계
int totalSal = emps.stream().mapToInt(p -> p.salary).sum();

// 부서별 직원 수
Map<String, Long> deptCount = emps.stream()
    .collect(Collectors.groupingBy(p -> p.dept, Collectors.counting()));
```
