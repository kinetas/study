# 5월 18일 필기 — JAVA

## 핵심요약

`@FunctionalInterface`는 추상 메서드가 정확히 1개인 인터페이스로, 람다 표현식이 그 익명 구현체가 된다. 가변인자(`Integer...`, `Object...`)를 받는 커스텀 함수형 인터페이스도 정의 가능하며, `Function`의 `andThen`으로 함수를 체이닝하고 `x -> n -> n + x` 형태의 커링(함수를 반환하는 함수)도 표현할 수 있다. 표준 함수형 인터페이스 5종은 `Function<T,R>(T→R·apply)·Consumer<T>(T→void·accept)·Supplier<T>(()→T·get)·Predicate<T>(T→boolean·test)·BiFunction<T,U,R>(T,U→R·apply)`이며, `Predicate`는 `and·or·negate`로 조건을 조합해 `Stream.filter`와 결합한다. 어노테이션(`@`)은 컴파일러·빌드 도구·런타임에 메타데이터를 제공하며, 표준(`@Override·@Deprecated·@FunctionalInterface`)과 메타(`@Target·@Retention·@Inherited`) 어노테이션으로 나뉜다. 사용자 정의 어노테이션은 `@interface`로 선언하고, `@Retention(RUNTIME)` + `@Target`으로 적용 범위를 지정한 뒤 `getClass().getAnnotation()`으로 런타임에 값을 읽는다.

## JAVA03 — Ch18 함수형 인터페이스 (Functional Interface)

---

### 1) 핵심 개념

| 개념 | 설명 |
|---|---|
| `@FunctionalInterface` | 추상 메서드가 정확히 1개인 인터페이스 — 람다 표현식으로 구현 |
| **커스텀 인터페이스** | 직접 `@interface`로 정의. 가변인자·제네릭 반환도 가능 |
| **표준 인터페이스** | `java.util.function.*` 패키지에 미리 정의된 5종 |
| **함수 합성** | `andThen`으로 여러 Function을 체이닝 |
| **커링** | 함수를 반환하는 함수 — `x -> n -> n + x` |

### 2) C01 — 커스텀 함수형 인터페이스

```java
// (1) 인자O / 반환X  — Consumer<String>과 같은 형태
@FunctionalInterface
interface Func1 {
    void say(String message);
}

// (2) 가변인자 받아 int 반환
@FunctionalInterface
interface Func2 {
    int sum(Integer ...args);
}

// (3) 가변인자 받아 List<Integer> 반환
@FunctionalInterface
interface Func3 {
    List<Integer> sum(Object ...args);
}

// 람다 대입
Func1 func1 = message -> System.out.println(message + "_!");
func1.say("Hello");   // Hello_!

Func2 func2 = arr -> Arrays.stream(arr).reduce(0, (sum, item) -> sum + item);
System.out.println(func2.sum(10, 20, 30, 40, 50, 60));  // 210

Func3 func3 = arr -> Arrays.stream(arr)
    .filter(el -> el instanceof Integer)
    .map(el -> (Integer) el)
    .collect(Collectors.toList());
System.out.println(func3.sum(10, 20, 30.5, "홍길동", true));  // [10, 20]
```

### 3) C02 — 사칙연산 함수형 인터페이스 예제

```java
@FunctionalInterface
interface Functional {
    Integer execute(int ...args);
}

class Calc {
    Functional sum;   // 합
    Functional sub;   // 차 (내림차순 정렬 후 누적 감산)
    Functional mul;   // 곱
    Functional div;   // 나누기

    Calc() {
        sum = arr -> Arrays.stream(arr).sum();
        sub = arr -> Arrays.stream(arr).boxed()
                    .sorted((a, b) -> b - a)
                    .reduce((s, item) -> s - item).orElse(0);
        mul = arr -> Arrays.stream(arr).reduce(1, (m, item) -> m * item);
        div = arr -> Arrays.stream(arr).reduce((d, item) -> d / item).orElse(0);
    }
}

Calc calc = new Calc();
System.out.println("합    : " + calc.sum.execute(10, 20, 30, 40, 50, 60));  // 210
System.out.println("차    : " + calc.sub.execute(12, 10));                  // 2
System.out.println("곱    : " + calc.mul.execute(10, 20, 30, 40, 50, 60));  // 720000000
System.out.println("나눗셈 : " + calc.div.execute(10, 20, 30, 40, 50, 60)); // 0
```

> 뺄셈·나눗셈은 큰 수 → 작은 수 순서로 정렬 후 누적 연산한다.

### 4) C03 — Function andThen 합성 & 커링

```java
// 기본 Function
Function<Integer, Integer> func1 = x -> x * x;  // 제곱
Function<Integer, Integer> func2 = x -> x + x;  // 2배
Function<List<Integer>, Integer> func3 = arr -> arr.stream().reduce(0, (sum, el) -> sum + el);
Function<List<Object>, List<String>> func4 = arr -> arr.stream()
    .filter(el -> el instanceof String)
    .map(el -> (String) el)
    .collect(Collectors.toList());

// andThen — 앞 함수 결과를 뒤 함수에 전달
Function<Integer, Integer> func5 = func1.andThen(func2);
// 10 → 100(제곱) → 200(2배)

Function<List<Integer>, Integer> func6 = func3.andThen(func2).andThen(func1);
// [10,20,30] → 60 → 120 → 14400

// 커링 — 함수를 반환하는 함수
Function<Integer, Function<Integer, Integer>> func7 = (x) -> {
    return (n) -> n + x;  // x를 클로저로 캡처
};

Function<Integer, Function<Integer, Integer>> func8 = x -> n -> n + x;  // 축약형

System.out.println(func5.apply(10));                         // 200
System.out.println(func6.apply(Arrays.asList(10, 20, 30)));  // 14400
System.out.println(func7.apply(10).apply(20));               // 30
System.out.println(func8.apply(10).apply(20));               // 30
```

### 5) C04 — Function + Stream 활용 예제

```java
// func1 : List<String>에서 길이 5 이상인 문자열 개수 반환
Function<List<String>, Integer> func1 = x ->
    (int) x.stream().filter(s -> s.length() >= 5).count();

// func2 : List<Integer> 각 숫자를 제곱해서 누적합 반환
Function<List<Integer>, Integer> func2 = x ->
    x.stream().map(n -> (int) Math.pow(n, 2)).reduce(0, Integer::sum);

// func3 : List<String> 각 문자열의 첫 글자만 대문자로 추출한 리스트 반환
Function<List<String>, List<String>> func3 = x ->
    x.stream().map(s -> s.substring(0, 1).toUpperCase()).collect(Collectors.toList());

System.out.println(func1.apply(Arrays.asList("apple", "hi", "banana", "cat", "elephant"))); // 3
System.out.println(func2.apply(Arrays.asList(1, 2, 3, 4, 5)));                              // 55
System.out.println(func3.apply(Arrays.asList("kim", "park", "lee", "choi")));               // [K, P, L, C]
```

### 6) C05 — 표준 함수형 인터페이스 카탈로그

| 인터페이스 | 시그니처 | 메서드 | 설명 |
|---|---|---|---|
| `Function<T, R>` | T → R | `apply` | 변환 |
| `Consumer<T>` | T → void | `accept` | 부수효과 (출력·저장) |
| `Supplier<T>` | () → T | `get` | 팩토리 (인자 없이 생성) |
| `Predicate<T>` | T → boolean | `test` | 조건 검사 |
| `BiFunction<T,U,R>` | T, U → R | `apply` | 인자 2개 변환 |

```java
// (1) Function<T, R> — 변환
Function<String, Integer> toInt = Integer::parseInt;
toInt.apply("123");   // 123

Function<String, Integer> toIntThenTwice = toInt.andThen(n -> n * 2);
toIntThenTwice.apply("50");   // 100

// (2) Consumer<T> — 부수효과
Consumer<String> printer = System.out::println;
printer.accept("출력");

Consumer<String> upperPrinter = s -> System.out.println(s.toUpperCase());
printer.andThen(upperPrinter).accept("hello");   // 연결: 두 Consumer 순서대로 실행

// (3) Supplier<T> — 팩토리
Supplier<ArrayList<String>> listFactory = ArrayList::new;
ArrayList<String> list = listFactory.get();
list.add("AAA"); list.add("BBB");

// (4) Predicate<T> — 조건 검사
Predicate<String> isEmpty = String::isEmpty;
Predicate<String> isShort = s -> s.length() < 3;

isEmpty.test("");        // true
isShort.test("hi");      // true

Predicate<String> notEmptyAndShort = isEmpty.negate().and(isShort);
words.stream().filter(isEmpty.negate()).filter(isShort).forEach(printer);

// (5) BiFunction<T, U, R> — 인자 2개
BiFunction<String, String, Integer> cmp = String::compareTo;
BiFunction<Integer, Integer, Integer> add = (a, b) -> a + b;
add.apply(3, 5);   // 8
```

> 숫자 전용 변형(`IntFunction·IntConsumer·IntSupplier·IntPredicate` 등)도 존재 — boxing 비용 절감 목적

### 7) C06 — Predicate 조합 메서드 활용 예제

| 메서드 | 설명 |
|---|---|
| `and(Predicate)` | 논리 AND |
| `or(Predicate)` | 논리 OR |
| `negate()` | 논리 NOT |

```java
List<Integer> nums  = Arrays.asList(-5, -3, 0, 2, 7, 8, 11, 13, 15, 20);
List<String>  words = Arrays.asList("apple", "ant", "banana", "cherry", "", "kiwi", "berry");

// 정수 Predicate 조합
Predicate<Integer> isPositive = n -> n > 0;
Predicate<Integer> isEven     = n -> n % 2 == 0;
Predicate<Integer> lessThan10 = n -> n < 10;

nums.stream().filter(isPositive)                               // [2, 7, 8, 11, 13, 15, 20]
nums.stream().filter(isEven)                                   // [-4는없음, 0, 2, 8, 20]
nums.stream().filter(lessThan10)                               // [-5,-3,0,2,7,8]
nums.stream().filter(isPositive.and(isEven))                   // [2, 8, 20]
nums.stream().filter(isPositive.or(isEven))                    // [0, 2, 7, 8, 11, 13, 15, 20]
nums.stream().filter(isPositive.negate())                      // [-5, -3, 0]
nums.stream().filter(isPositive.and(isEven).and(lessThan10))   // [2, 8]

// 문자열 Predicate 조합
Predicate<String> isEmpty    = s -> s.equals("");
Predicate<String> startsWithA = s -> s.startsWith("a");
Predicate<String> lenGE5     = s -> s.length() >= 5;

words.stream().filter(isEmpty.negate())    // 비어있지 않은 것
words.stream().filter(startsWithA)         // a로 시작
words.stream().filter(lenGE5)              // 길이 5 이상

Predicate<String> isSameApple = s -> s.equals("apple");
words.stream().filter(isSameApple)         // ["apple"]
words.stream().filter(isSameApple.negate()) // apple 제외
```

## JAVA03 — Ch19 어노테이션 (Annotation)

---

### 1) 핵심 개념

| 분류 | 설명 |
|---|---|
| **표준 어노테이션** | 자바가 기본 제공 (`@Override`, `@Deprecated` 등) |
| **메타 어노테이션** | 어노테이션을 정의할 때 사용 (`@Target`, `@Retention` 등) |
| **사용자 정의 어노테이션** | `@interface`로 직접 정의 |

어노테이션의 역할

- **컴파일러**: 문법 오류 체크 정보 제공 (`@Override`)
- **빌드 시**: 코드 자동 생성 (Lombok 등)
- **런타임**: 리플렉션으로 특정 동작 실행

---

### 2) 표준 어노테이션

| 어노테이션 | 설명 |
|---|---|
| `@Override` | 메서드 오버라이딩임을 컴파일러에 알림 |
| `@Deprecated` | 더 이상 사용하지 않을 대상 표시 |
| `@FunctionalInterface` | 함수형 인터페이스임을 명시 |
| `@SuppressWarnings` | 컴파일 경고 무시 |
| `@SafeVarargs` | 제네릭 가변인자 경고 무시 |

---

### 3) 메타 어노테이션

| 어노테이션 | 설명 |
|---|---|
| `@Target` | 어노테이션 적용 대상 지정 |
| `@Retention` | 어노테이션 유지 기간 지정 |
| `@Documented` | javadoc 문서에 포함 |
| `@Inherited` | 하위 클래스에 상속 |
| `@Repeatable` | 동일 대상에 반복 적용 허용 |

**`@Target` 주요 값**

| 값 | 적용 대상 |
|---|---|
| `ElementType.TYPE` | 클래스, 인터페이스, enum |
| `ElementType.FIELD` | 멤버 변수 |
| `ElementType.METHOD` | 메서드 |

**`@Retention` 주요 값**

| 값 | 유지 시점 |
|---|---|
| `RetentionPolicy.SOURCE` | 소스 코드에만 (컴파일 시 제거) |
| `RetentionPolicy.CLASS` | 클래스 파일까지 (기본값) |
| `RetentionPolicy.RUNTIME` | 런타임까지 (리플렉션으로 읽기 가능) |

### 4) C01 — 사용자 정의 어노테이션 선언 & 사용

**어노테이션 선언**

```java
@Retention(RetentionPolicy.RUNTIME)               // 런타임까지 유지
@Target({ElementType.TYPE, ElementType.METHOD})   // 클래스 & 메서드 적용 가능
public @interface C01CustomAnnotation {
    String  value()  default "HELLO WORLD";
    int     number() default 1;
    boolean isOpen() default false;
}
```

**어노테이션 적용 & 리플렉션으로 값 읽기**

```java
@C01CustomAnnotation(value = "홍길동", number = 10000, isOpen = true)
class Simple {
    String  value;
    int     number;
    boolean isOpen;

    Simple() {
        // 런타임에 어노테이션 정보 읽기
        C01CustomAnnotation ref = this.getClass().getAnnotation(C01CustomAnnotation.class);
        System.out.println("VALUE  : " + ref.value());    // 홍길동
        System.out.println("NUMBER : " + ref.number());   // 10000
        System.out.println("ISOPEN : " + ref.isOpen());   // true

        this.value  = ref.value();
        this.number = ref.number();
        this.isOpen = ref.isOpen();
    }
}
```

> `@Retention(RUNTIME)` 이 없으면 런타임에 `getAnnotation()`이 `null`을 반환한다.

| 항목 | 설명 |
|---|---|
| `getClass().getAnnotation(X.class)` | 클래스에 붙은 어노테이션 읽기 |
| `getMethod(...).getAnnotation(X.class)` | 메서드에 붙은 어노테이션 읽기 |
| `default` 값 | 어노테이션 적용 시 생략하면 default 값 사용 |
