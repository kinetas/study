# 5월 29일 필기 — Spring Boot / 요청 파라미터 & 뷰 데이터 전달

## 핵심요약

03_PARAM에서는 Spring MVC의 파라미터 수신 방법과 뷰 데이터 전달 방법을 학습했다. 파라미터 수신은 크게 4가지로 나뉜다. `@RequestParam`은 쿼리스트링·폼 파라미터를 개별 변수에 매핑하며 `required=false`로 필수 여부를, `name=`으로 파라미터명-변수명 불일치를 해결한다. `@ModelAttribute`(생략 가능)는 DTO 필드명과 파라미터명이 일치하면 자동 바인딩한다. `@PathVariable`은 URL 경로 일부를 변수로 추출하며 DTO에도 바인딩된다. `@RequestBody`는 JSON 요청(`application/json`)을 DTO로 역직렬화할 때 사용하며, 폼 데이터에는 쓰지 않는다. 뷰 데이터 전달은 `Model.addAttribute()`가 가장 일반적이고, `ModelAndView`는 뷰명과 데이터를 한 객체로 묶어 반환한다. `HttpServletRequest.setAttribute()`는 순수 서블릿 API로 직접 `forward()`까지 처리한다. Forward는 서버 내부 이동으로 같은 request가 이어지므로 이전 Model 데이터가 누적되고 URL이 바뀌지 않는다. Redirect는 새 요청을 발생시키므로 Model 데이터가 소멸하며, 1회성 데이터를 넘기려면 `RedirectAttributes.addFlashAttribute()`를 사용한다. `addAttribute()`는 QueryString에 노출되고 `addFlashAttribute()`는 세션에 저장 후 1회 소멸한다. ExController(ex01~ex09)는 파라미터 수신 방법 전체를 반환타입 `void`로 콘솔 로그만 출력하는 실습 문제다. EX05의 `Map<String,Object>` 수신은 파라미터명을 모를 때 유용하고, EX09의 `@RequestBody String`은 body 전체 문자열을 변수에 담으므로 form 파라미터 분리가 필요하면 `@RequestParam`을 써야 한다. build.gradle에는 JSP EL 파서(`jakarta.el:jakarta.el-api:6.0.1`)와 DevTools(`spring-boot-devtools`)가 추가되었다.

## Spring Boot — 03_PARAM 파라미터 수신 방법

---

### 1) 핵심 개념

| 어노테이션 | 대상 | 특징 |
|---|---|---|
| `@RequestParam` | 쿼리스트링, 폼 파라미터 | 개별 변수 매핑. `required`, `defaultValue`, `name` 옵션 |
| `@ModelAttribute` | 쿼리스트링, 폼 파라미터 | DTO 필드명 = 파라미터명이면 자동 바인딩. 생략 가능 |
| `@PathVariable` | URL 경로 변수 | `@GetMapping("/p07/{name}/{age}")` 처럼 경로에 `{}` 선언 |
| `@RequestBody` | HTTP 요청 body (JSON) | `Content-Type: application/json` 전용. DTO 역직렬화 |

---

### 2) 파라미터 수신 흐름

```
GET /param/p07/홍길동/25/서울
         └─────────────────── @PathVariable

GET /param/p01?name=홍길동
                   └──────── @RequestParam

POST /param/p09  (body: name=홍길동&age=25&addr=서울)
                  └────────── @ModelAttribute  (form-urlencoded)

POST /param/p10  (body: {"name":"홍길동","age":25,"addr":"서울"})
                  └────────── @RequestBody  (application/json)
```

---

### 3) `@RequestParam` 상세

```java
// p01: required=false — 파라미터 없으면 null (기본 true면 없을 때 400)
@GetMapping("/p01")
public String p01(@RequestParam(required = false) String name) { ... }

// p02: name= — 요청 파라미터명(username)과 변수명(name)이 다를 때
@PostMapping("/p02")
public String p02(@RequestParam(name = "username") String name) { ... }

// p03: 어노테이션 생략 — 파라미터명 = 변수명이면 자동 바인딩
@GetMapping("/p03")
public String p03(String name) { ... }

// p04: 여러 개 개별 수신 + 자동 형변환 (String → int)
@GetMapping("/p04")
public String p04(@RequestParam String name, @RequestParam int age, @RequestParam String addr) { ... }
```

---

### 4) `@ModelAttribute` / DTO 자동 바인딩

```java
// p05: @ModelAttribute 명시
@GetMapping("/p05")
public String p05(@ModelAttribute PersonDTO dto) { ... }

// p06: 어노테이션 생략 — DTO 타입이면 자동으로 @ModelAttribute 적용
@GetMapping("/p06")
public String p06(PersonDTO dto) { ... }
// 요청: ?name=홍길동&age=25&addr=서울 → PersonDTO 필드에 자동 바인딩
```

---

### 5) `@PathVariable`

```java
// p07: 경로 변수를 개별 변수로 수신
@GetMapping("/p07/{name}/{age}/{addr}")
public String p07(@PathVariable String name, @PathVariable int age, @PathVariable String addr) { ... }

// p08: 경로 변수를 DTO에 자동 바인딩
@GetMapping("/p08/{name}/{age}/{addr}")
public String p08(PersonDTO dto) { ... }
// URL: /param/p08/홍길동/25/서울
```

---

### 6) POST 수신 방식 비교 — `@ModelAttribute` vs `@RequestBody`

```java
// p09: HTML form / form-urlencoded 처리
@PostMapping("/p09")
public String p09(@ModelAttribute PersonDTO dto) { ... }

// p10: JSON 처리
@PostMapping("/p10")
public String p10(@RequestBody PersonDTO dto) { ... }
```

| | `@RequestParam` / `@ModelAttribute` | `@RequestBody` |
|---|---|---|
| **처리 대상** | HTML form, Query String | JSON |
| **Content-Type** | `application/x-www-form-urlencoded` | `application/json` |
| **테스트 도구** | 브라우저 form | Postman (Body → raw → JSON) |

## Spring Boot — 03_PARAM 뷰 데이터 전달

---

### 1) `Model.addAttribute()` — 개별 속성 / DTO 전달

```java
// p11: 개별 속성 추가
@GetMapping("/p11")
public String p11(PersonDTO dto, Model model) {
    model.addAttribute("name", dto.getName());
    model.addAttribute("age",  dto.getAge());
    model.addAttribute("addr", dto.getAddr());
    model.addAttribute("now",  LocalDateTime.now());
    return "param/page11";
}
```
```jsp
${name} ${age} ${addr} ${now}   <%-- EL로 출력 --%>
```

```java
// p12: DTO 통째로 전달
@GetMapping("/p12")
public String p12(PersonDTO dto, Model model) {
    model.addAttribute("dto", dto);
    model.addAttribute("now", LocalDateTime.now());
    return "param/page12";
}
```
```jsp
${dto}        <%-- toString() 출력 --%>
${dto.name}   <%-- 필드 접근 --%>
```

---

### 2) `ModelAndView` — 뷰명 + 데이터 한 객체로 반환

```java
// p13
@GetMapping("/p13")
public ModelAndView p13(PersonDTO dto) {
    ModelAndView mav = new ModelAndView();
    mav.addObject("dto", dto);
    mav.addObject("now", LocalDateTime.now());
    mav.setViewName("param/page13");
    return mav;
}
```

---

### 3) `HttpServletRequest` — 순수 Servlet API

```java
// p14: 반환타입 void — 직접 forward 처리
@GetMapping("/p14")
public void p14(HttpServletRequest request, HttpServletResponse response)
        throws ServletException, IOException {
    String name = request.getParameter("name");
    int age = request.getParameter("age") != null
            ? Integer.parseInt(request.getParameter("age")) : -1;
    String addr = request.getParameter("addr");

    request.setAttribute("dto", new PersonDTO(name, age, addr));
    request.setAttribute("now", LocalDateTime.now());
    request.getRequestDispatcher("/WEB-INF/views/param/page14.jsp")
           .forward(request, response);
}
```

---

### 4) 뷰 데이터 전달 방법 비교

| 방법 | 특징 |
|------|------|
| `Model.addAttribute()` | 가장 일반적. 개별 또는 객체 전달 |
| `ModelAndView` | 뷰명과 데이터를 하나의 객체로 묶어 반환 |
| `HttpServletRequest.setAttribute()` | 순수 Servlet API, 직접 forward 처리 |

## Spring Boot — 03_PARAM Forward vs Redirect

---

### 1) 핵심 차이

| | Forward | Redirect |
|---|---|---|
| **URL 변화** | 없음 | 변경됨 |
| **요청 횟수** | 1번 | 2번 (새 요청) |
| **Model 데이터** | 유지 (누적) | 소멸 |
| **데이터 전달** | `Model` | `RedirectAttributes.addFlashAttribute()` |
| **주 용도** | 내부 단계 처리 | PRG 패턴, 외부 이동 |

---

### 2) Forward — 서버 내부 이동

```java
@GetMapping("/forward/init")
public String forward_init(Model model) {
    model.addAttribute("init", "init_value");
    return "forward:/param/forward/step1";   // URL 안 바뀜
}
@GetMapping("/forward/step1")
public String forward_step1(Model model) {
    model.addAttribute("step1", "step1_value");
    return "forward:/param/forward/step2";
}
@GetMapping("/forward/step2")
public String forward_step2(Model model) {
    model.addAttribute("step2", "step2_value");
    return "param/forward/step2";            // 최종 뷰
}
```
```jsp
<%-- step2.jsp — init, step1, step2 모두 출력 가능 (같은 request가 이어짐) --%>
${init} ${step1} ${step2}
```

---

### 3) Redirect — 새 요청 발생

```java
@GetMapping("/redirect/init")
public String redirect_init(RedirectAttributes redirectAttributes) {
    redirectAttributes.addFlashAttribute("init", "init_value");
    // addAttribute()      → QueryString으로 노출 (?init=init_value)
    // addFlashAttribute() → 세션에 1회 저장 후 소멸 (URL에 노출 안 됨)
    return "redirect:/param/redirect/step1"; // URL 바뀜, 새 요청
}
```

```
Forward 흐름:  요청 1번  →  init → step1 → step2 (연속, request 공유)
Redirect 흐름: 요청 1번  →  init  →  [새 요청]  →  step1  →  [새 요청]  →  step2
```

## Spring Boot — 03_PARAM ExController 실습 (ex01~ex09)

---

### 1) 구조

`@RequestMapping("/ex")` 기반. 모든 핸들러 반환타입 `void` → 뷰 없이 콘솔 로그로 결과 확인.

---

### 2) ex01~ex05 — `@RequestParam` 변형

```java
// EX01: 기본
@GetMapping("/ex01")
public void ex01(@RequestParam String keyword) {
    log.info("EX01 keyword : " + keyword);
}
// GET /ex/ex01?keyword=spring  →  EX01 keyword : spring

// EX02: required=false + defaultValue
@GetMapping("/ex02")
public void ex02(@RequestParam(required = false, defaultValue = "1") int page) {
    log.info("EX02 page : " + page);
}
// GET /ex/ex02          →  EX02 page : 1
// GET /ex/ex02?page=3   →  EX02 page : 3

// EX03: 파라미터명 ≠ 변수명
@GetMapping("/ex03")
public void ex03(@RequestParam(name = "user_name") String name) {
    log.info("EX03 name : " + name);
}
// GET /ex/ex03?user_name=hong  →  EX03 name : hong

// EX04: 여러 개 + 자동 형변환
@GetMapping("/ex04")
public void ex04(@RequestParam String name, @RequestParam int age) {
    log.info("EX04 name : " + name + ", age : " + age);
}
// GET /ex/ex04?name=hong&age=20  →  EX04 name : hong, age : 20

// EX05: Map으로 전체 수신
@GetMapping("/ex05")
public void ex05(@RequestParam Map<String, Object> params) {
    log.info("EX05 params : " + params);
}
// GET /ex/ex05?a=1&b=2&c=3  →  EX05 params : {a=1, b=2, c=3}
```

---

### 3) ex06~ex08 — `@PathVariable` / DTO 바인딩

```java
// EX06: @PathVariable 1개
@GetMapping("/ex06/{id}")
public void ex06(@PathVariable int id) {
    log.info("EX06 id : " + id);
}
// GET /ex/ex06/100  →  EX06 id : 100

// EX07: @PathVariable 여러 개
@GetMapping("/ex07/{category}/{id}")
public void ex07(@PathVariable String category, @PathVariable int id) {
    log.info("EX07 category : " + category + ", id : " + id);
}
// GET /ex/ex07/book/100  →  EX07 category : book, id : 100

// EX08: DTO 자동 바인딩
@GetMapping("/ex08")
public void ex08(PersonDTO dto) {
    log.info("EX08 dto : " + dto);
}
// GET /ex/ex08?name=hong&age=20&addr=seoul  →  EX08 dto : PersonDTO(name=hong, age=20, addr=seoul)
```

---

### 4) ex09 — POST + `@RequestBody`

```java
// EX09: @RequestBody — body 전체 문자열로 수신
@PostMapping("/ex09")
public void ex09(@RequestBody String username) {
    log.info("EX09 username : " + username);
}
// PowerShell: curl -Method POST "http://localhost:8090/ex/ex09" -Body "username=hong"
// → EX09 username : username=hong  (body 전체가 변수에 들어옴)
```

> `@RequestBody String`은 body를 통째로 문자열로 받음.  
> `username=hong`에서 `hong`만 꺼내려면 `@RequestParam`으로 바꿔야 함.

---

### 5) ExController 포인트 정리

| 문제 | 핵심 | 요청 예시 |
|------|------|-----------|
| EX01 | `@RequestParam` 기본 | `GET /ex/ex01?keyword=spring` |
| EX02 | `required=false` + `defaultValue` | `GET /ex/ex02` (파라미터 생략) |
| EX03 | `name=` 으로 파라미터명 매핑 | `GET /ex/ex03?user_name=hong` |
| EX04 | 다중 파라미터 + String→int 자동변환 | `GET /ex/ex04?name=hong&age=20` |
| EX05 | `Map<String,Object>` 로 전체 수신 | `GET /ex/ex05?a=1&b=2&c=3` |
| EX06 | `@PathVariable` 1개 | `GET /ex/ex06/100` |
| EX07 | `@PathVariable` 다중 | `GET /ex/ex07/book/100` |
| EX08 | DTO 커맨드 객체 바인딩 | `GET /ex/ex08?name=hong&age=20&addr=seoul` |
| EX09 | `@RequestBody` — raw body 수신 | `POST /ex/ex09` |

## Spring Boot — 03_PARAM build.gradle / DevTools 변경점

---

### 1) 추가된 의존성

```groovy
// JSP EL PARSER
implementation 'jakarta.el:jakarta.el-api:6.0.1'

// DEVTOOLS
implementation 'org.springframework.boot:spring-boot-devtools'
```

| 라이브러리 | 역할 |
|---|---|
| `jakarta.el:jakarta.el-api:6.0.1` | JSP EL(`${...}`) 파서. Embedded Tomcat 단독 실행 시 EL 미동작 방지용 |
| `spring-boot-devtools` | 코드 변경 시 자동 재시작 + LiveReload |

---

### 2) application.properties — DevTools 설정

```properties
# DevTools
spring.devtools.restart.enabled=true
spring.devtools.restart.poll-interval=2s
spring.devtools.restart.quiet-period=1s
spring.devtools.livereload.enabled=true
```

| 설정 키 | 설명 |
|---|---|
| `restart.enabled` | 자동 재시작 활성화 |
| `restart.poll-interval` | 클래스파일 변경 감지 주기 |
| `restart.quiet-period` | 마지막 변경 후 재시작까지 대기 시간 (연속 저장 시 중복 재시작 방지) |
| `livereload.enabled` | 브라우저 자동 새로고침 활성화 |

---

### 3) `@RequestMapping` vs `@GetMapping` / `@PostMapping`

```java
// 구형 방식 (method 명시)
@RequestMapping(value = "/p01", method = RequestMethod.GET)

// 신형 방식 (축약)
@GetMapping("/p01")
@PostMapping("/p02")
```

> 클래스 레벨 `@RequestMapping("/param")` + 메서드 레벨 `@GetMapping("/p01")` → `/param/p01` 완성
