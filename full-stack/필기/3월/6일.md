# 3월 6일 필기 — screen-implement (화면 구현)

## 핵심요약

Markup은 텍스트에 태그로 의미를 붙여 문서의 구조를 드러내는 방식으로, 같은 글자라도 어떤 태그로 감싸느냐에 따라 역할이 달라진다. 태그는 시작과 종료 태그를 짝으로 쓰며, 속성을 추가해 부가 정보를 전달할 수 있다. 요소는 부모·자식 계층 관계를 가지며 이 구조가 화면 레이아웃의 기반이 된다. 구조가 명확해야 CSS 스타일 적용 대상이 분명해지고, 검색 엔진과 보조기기도 태그를 기준으로 문서를 해석한다.

## Markup

- Markup은 텍스트에 의미를 붙여 구조를 드러내는 표현 방식
- 예를 들어 `<h1>`은 가장 중요한 제목, `<p>`는 문단, `<a>`는 링크를 뜻
- 즉, 같은 글자라도 어떤 태그로 감싸느냐에 따라 문서의 역할이 달라짐

## 왜 중요한가

- 구조가 명확해야 CSS로 스타일을 줄 때도 대상이 분명
- 검색 엔진이나 보조기기도 태그를 기준으로 문서의 의미를 해석
- 화면을 빨리 만드는 것만큼, 의미 있게 마크업하는 습관이 중요

## 기본 원칙

- 시작 태그와 종료 태그를 짝으로 쓰는 요소가 많음
- 태그 안에는 속성을 넣어 추가 정보를 줄 수 있음
- 요소는 부모와 자식 관계를 가지며, 이 계층이 화면 구조가 됨

## 자주 보는 예

- `<h1>` : 문서의 큰 제목
- `<p>` : 하나의 문단
- `<div>` : 영역을 나누는 블록
- `<span>` : 문장 일부를 묶는 인라인 요소
- `<a>` : 다른 위치로 이동하는 링크
