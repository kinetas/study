# 3월 30일 필기 — screen-implement (화면 구현)

## 핵심요약

JSLIB는 npm 없이 CDN으로 붙이는 외부 라이브러리 연습 구역으로, Swiper·Lodash·ScrollMagic·GSAP을 다룬다. Swiper는 슬라이드 UI를 위한 라이브러리로 autoplay·pagination·navigation·멀티뷰·수직 방향 등 다양한 옵션을 제공한다. Lodash의 `_.throttle`은 scroll처럼 빈번한 이벤트에서 콜백 실행 빈도를 제한해 성능을 개선하는 용도로 사용한다. ScrollMagic은 Controller와 Scene으로 특정 요소가 뷰포트 지점에 도달할 때 클래스를 토글하는 스크롤 연동 연출을 구현하며, GSAP은 gsap.to()로 CSS 값을 시간에 따라 변화시키는 타임라인 애니메이션 라이브러리이다.


- 17. **예제 파일** : 핵심 문법·DOM은 `JAVASCRIPT` 폴더(`01Basic` ~ `08Sync&Async`), **외부 라이브러리 연동**은 `JSLIB` 폴더 참고
## JSLIB
- 메인 `JAVASCRIPT`와 달리 **npm 없이 CDN·로컬 스크립트로 붙이는 외부 라이브러리** 연습 구역. (과거 `SUB` 폴더를 `JSLIB`으로 옮긴 경우 경로만 **`JSLIB/...`** 로 맞추면 됨.)
- 아래는 **각 라이브러리가 무엇을 해 주는지, 핵심 호출이 어떤 모양인지**를 `콜백 함수` 절처럼 정리한 내용. 예제 HTML은 맨 아래 **「예제 위치」** 표에만 모음.
### Swiper — 터치·휠 슬라이드 UI
- **역할:** 컨테이너 안의 슬라이드를 넘기기, 자동 재생, 페이지 표시, 이전/다음 버튼을 한 번에 묶어 준다. 인스턴스는 **CSS 선택자(또는 요소)** 와 **옵션 객체**로 만든다.

| 항목 | 설명 |
|------|------|
| 인스턴스 | `new Swiper(선택자, { …옵션 })` |
| 자동 재생 | `autoplay: { delay: 2000 }` — ms마다 다음 슬라이드 |
| 루프 | `loop: true` — 끝에서 처음으로 이어 붙임 |
| 페이지 | `pagination: { el, type: 'fraction', clickable: true }` — 예: 현재/전체 분수 표시 |
| 화살표 | `navigation: { prevEl, nextEl }` — DOM과 연결 |
| 멀티 뷰 | `slidesPerView`, `centeredSlides` — 한 화면에 여러 장 + 가운데 정렬 |
| 수직 | `direction: 'vertical'` — 상단 공지 롤링 등 |
| API | `swiper.autoplay.stop()` / `start()` — 재생 제어, `slideToLoop(i)` — 루프 모드에서 인덱스 이동 |
```javascript
const swiper = new Swiper('.my-swiper', {
    direction: 'horizontal',
    loop: true,
    autoplay: { delay: 2000 },
    pagination: { el: '.swiper-pagination', type: 'fraction', clickable: true },
    navigation: { prevEl: '.swiper-button-prev', nextEl: '.swiper-button-next' },
});
// swiper.autoplay.stop();
```
- **예제:** 단일 배너, 배너+수직 공지, 멀티 슬라이드처럼 Swiper 옵션 변화에 따른 동작 비교용.
### Lodash — `throttle` (호출 빈도 줄이기)
- **역할:** `scroll` 같이 아주 자주 도는 이벤트에서 **같은 함수를 너무 많이 호출하지 않도록** 막는다. Lodash의 **`_.throttle(함수, 간격ms)`** 가 **새 함수**를 반환하고, 그걸 리스너에 넣는다.

| 항목 | 설명 |
|------|------|
| 의도 | 연속 이벤트 → 최대 N ms에 한 번만 본문 실행 |
| 반환 | throttled 함수 (원본과 시그니처 동일하게 쓰면 됨) |
```javascript
window.addEventListener('scroll', _.throttle(() => {
    console.log(window.scrollY);
}, 100));
```
- **예제 주소:** Lodash throttle 예제
### ScrollMagic — 스크롤 구간과 연동된 연출
- **역할:** 특정 요소가 뷰포트의 어느 지점에 **걸렸을지** 감시하고, 그때마다 **클래스 넣기/빼기** 등 후속 동작 연결 가능. CSS `transition`과 같이 쓰는 "스크롤 반응 모션" 구성 방식.

| 항목 | 설명 |
|------|------|
| Controller | `new ScrollMagic.Controller()` — 스크롤 관측의 허브 |
| Scene | `new ScrollMagic.Scene({ triggerElement, triggerHook })` — 트리거 대상·위치(0~1, 예: 0.5는 화면 중앙) |
| 토글 | `.setClassToggle(대상El, '클래스명')` — 씬 활성/비활성에 맞춰 클래스 스위치 |
| 등록 | `.addTo(controller)` |
```javascript
const controller = new ScrollMagic.Controller();
new ScrollMagic.Scene({ triggerElement: spySection, triggerHook: 0.5 })
    .setClassToggle(ballEl, 'move')
    .addTo(controller);
```
- **예제:** 스크롤 위치에 따라 한 요소만 움직이게 하거나, 구간별로 다른 클래스를 토글하는 흐름 비교용.
### GSAP — 타임라인·트윈 애니메이션
- **역할:** 선택자(또는 요소)의 **CSS 값·변환**을 시간에 따라 바꾼다. **`gsap.to(대상, { 속성들 })`** 에 `duration`, `ease`, `delay` 등을 넣는다.
```javascript
gsap.to('.box', {
    duration: 3,
    x: 200,
    y: 200,
    opacity: 0.2,
    rotate: 360,
    ease: 'elastic.out(1, 0.3)',
});
```
- **예제 주소:** GSAP 기초 예제
### Chart.js — `<canvas>` 차트
- **역할:** `canvas` + **데이터 객체**로 막대·선·파이 등을 그리는 방식. 패턴은 **`new Chart(캔버스컨텍스트 또는 요소, { type, data, options })`** 한 가지 정리.

| 항목 | 설명 |
|------|------|
| `type` | `'bar'`, `'line'`, `'pie'`, `'doughnut'`, `'radar'`, `'scatter'` 등 |
| `data` | `labels`, `datasets` (배열·색·값) |
| `options` | `scales`(축, 스택), `plugins`(제목·범례), `onClick` 등 |
```javascript
const ctx = document.getElementById('myChart');
new Chart(ctx, {
    type: 'bar',
    data: {
        labels: ['A', 'B', 'C'],
        datasets: [{ label: '값', data: [1, 2, 3], backgroundColor: ['#f44', '#4af', '#4f4'] }],
    },
    options: { scales: { y: { beginAtZero: true } } },
});
```
- 앱 데이터는 **`reduce` / `Object.keys` / `Object.entries` / `filter`** 로 가공한 뒤 `labels`·`datasets.data`에 넣는 식으로 이어 붙인다.
- **예제:** 스택 차트, 차트 종류 비교, 데이터 집계, 분할 통계처럼 데이터를 시각화하는 여러 흐름 비교용.
### QR 코드 (qrcode-generator)
- **역할:** 문자열을 QR 규격으로 인코딩해 **SVG 등 DOM**으로 내보내는 방식. 흐름은 **버전·오류보정 → 데이터 넣기 → 생성 → 태그 삽입** 순서 정리.
```javascript
const qr = qrcode(0, 'L');       // 버전, 오류보정 레벨
qr.addData('https://example.com');
qr.make();
document.getElementById('qrcode').innerHTML = qr.createSvgTag({ scalable: true });
```
- **예제 주소:** QR 코드 생성 예제
### Canvas 2D API — 그림판 (라이브러리 아님, `JSLIB`에 실습만 있음)
- **역할:** **`getContext('2d')`** 로 붓을 잡고, `lineTo` / `stroke` / `beginPath` / `moveTo` 로 선을 그린다. 화면 비우기는 **`clearRect`** .
```javascript
const canvas = document.getElementById('canvas');
const ctx = canvas.getContext('2d');
ctx.lineWidth = 4;
ctx.strokeStyle = '#000';
ctx.lineCap = 'round';
ctx.lineTo(x, y);
ctx.stroke();
ctx.clearRect(0, 0, canvas.width, canvas.height);
```
- 마우스는 `mousedown` / `mousemove` / `mouseup` 으로 "그리는 중" 플래그와 조합하는 방식.
- **예제 주소:** Canvas 그림판 예제 + 스크립트 파일
### Leaflet — 웹 지도
- **역할:** **`L.map`** 으로 지도 루트를 만들고, **`L.tileLayer(타일URL규칙).addTo(map)`** 으로 배경 타일을 깐다. **마커**는 `L.marker([lat, lng]).addTo(map)`, 팝업은 `bindPopup` / `openPopup`. **한국 전용 타일(EPSG 변환)** 은 `proj4`, `proj4leaflet`, `Leaflet.KoreanTmsProviders` 를 붙여 **`crs: L.Proj.CRS.Daum`** 같은 식으로 쓴다.
```javascript
const map = L.map('map').setView([35.87, 128.60], 13);
L.tileLayer('https://tile.openstreetmap.org/{z}/{x}/{y}.png', { maxZoom: 19 }).addTo(map);
const m = L.marker([35.87, 128.60]).addTo(map);
m.bindPopup('설명').openPopup();
```
  - **역지오코딩:** 위경도 → 주소는 외부 API(예: Nominatim)에 **`fetch`** 후 JSON 파싱, **`L.popup().setLatLng(latlng).setContent(html).openOn(map)`** 으로 표시하는 흐름.
  - **클러스터:** 많은 마커를 묶어 보이려면 **`L.markerClusterGroup()`** 에 마커를 `addLayer` 한 뒤 `map.addLayer(그룹)` 하는 방식.
  - **CSV → 점:** 플러그인 조합으로 `L.geoCsv` + `MarkerClusterGroup` + 필터 UI를 쓰는 패턴 정리.
- **예제:** 기본 지도, 타일 변환, 마커 추가, 역지오코딩, 클러스터링처럼 지도 기능이 단계별로 확장되는 흐름 비교용.
- `05Clustering/ex` 는 외부 템플릿 복제로 메인 문서에 Jekyll 문법(`{{ }}`)이 있어 그대로 열기 어려울 수 있으며, 참고 대상은 `ex/assets` 쪽 스크립트·GeoJSON.

### JSLIB 예제 읽는 순서
- **Swiper**: 기본 단일 슬라이더에서 시작해, 이중 인스턴스와 멀티 슬라이드 구성으로 확장되는 흐름. 뒤 예제로 갈수록 옵션이 더 많아지는 구조.
- **ScrollMagic**: 첫 예제는 트리거와 클래스 토글의 기본 구조 확인용, 다음 예제는 같은 원리를 여러 구간이나 다른 대상에 확장해 보는 흐름.
- **Chart.js**: 차트 생성 문법은 거의 같고, 달라지는 것은 `type`, `data`, `options` 구성. 예제가 바뀔수록 차트 종류와 데이터 가공 방식 비교용.
- **Leaflet**: 지도 표시에서 출발해 타일 체계 변경, 마커 추가, 주소 해석, 대량 마커 묶기까지 단계적으로 확장되는 구성. 기능이 쌓이는 순서 파악용.
