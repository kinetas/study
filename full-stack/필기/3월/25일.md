# 3월 25일 필기 — screen-implement (화면 구현)

## 핵심요약

DOM CRUD 실습에서 querySelector·createElement·appendChild로 동적 리스트와 테이블을 생성하고, 더블클릭 수정·드래그 정렬까지 구현하였다. 동기는 순차 실행으로 긴 작업 시 지연이 생기며, 비동기는 setTimeout·setInterval·Promise·async/await로 후속 처리를 연결하는 방식이다.


### DOM 노드 탐색·CRUD

- 탐색 예제 1 — `getElementById`, `querySelector` / `querySelectorAll`, `getElementsByTagName`, 유사 배열 `Array.from`·`Object.entries`
- 탐색 예제 2 — `document.forms`, 인덱스·`name`으로 폼·폼 내 `input` 접근
- 탐색 예제 3 — `querySelectorAll` + `mouseover` / `mouseleave`로 클래스 `btn--style1~8` 동적 부여
- 탐색 예제 4 — Enter·입력 버튼으로 값 추가, `createElement`로 행 UI 생성, 삭제 링크로 `remove()` (유효성: 빈 값·첫 글자 숫자 금지 등)
- 테이블 실습 초안 — 폼 → `tbody`에 행 추가·삭제, 더블클릭으로 폼에 재로딩 후 수정, 행 `dragstart`/`drop`으로 순서 바꾸기
- 테이블 실습 완성 — 동일 주제 완성본: 각 셀 `input readonly`, 수정/삭제 링크, 입력 유효성 (초안과 마크업 방식 차이)
- 드래그 정렬 예제 — 위 흐름과 유사 + **드래그 앤 드롭**으로 항목 순서 재배치(`dragstart` / `dragover` / `drop`)

### 동기·비동기
- **Info.txt** 요약: **동기**는 위→아래 순차 실행(긴 작업 시 지연), **비동기**는 기다리지 않고 다음 코드가 먼저 실행되며 콜백·Promise·async/await로 후속 처리.

- 비동기 예제 1 — 메인 스레드 `for`와 `setTimeout(..., 1)` 안 루프 실행 순서 비교(콘솔 로그)
- 비동기 예제 2 — `setTimeout` 지연 실행, `clearTimeout`으로 예약 취소
- 비동기 예제 3 — `setInterval` 반복, `clearInterval`로 중지
- 비동기 예제 4 — `setInterval`로 메인 패널 갱신 + `Promise` + `async`/`await`로 `sub_process_executor` 두 단계 순서 배치 예제

- 15. **DOM CRUD·폼** : `07NodeCRUD`(탐색, `forms`, 동적 리스트/테이블, 드래그 정렬)
- 16. **동기·비동기** : `08Sync&Async`(`setTimeout`/`setInterval`, Promise, `async`/`await`)
- 17. **예제 파일** : JAVASCRIPT 폴더 내 `07NodeCRUD`, `08Sync&Async` 참고
