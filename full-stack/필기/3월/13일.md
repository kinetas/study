# 3월 13일 필기 — screen-implement (화면 구현)

## 핵심요약

Transition은 속성 값이 바뀔 때 일정 시간에 걸쳐 부드럽게 변하게 하는 기능으로, duration·timing-function·delay로 세밀하게 제어한다. Transform은 레이아웃 흐름에 영향 없이 요소의 위치·크기·회전·기울기를 변경하며, perspective와 rotateX/Y로 3D 효과도 표현할 수 있다. Animation은 @keyframes로 구간별 스타일을 정의하고 iteration-count·direction으로 반복과 왕복을 제어하며, hover 시 play-state: paused로 멈출 수 있다. 미디어 쿼리(@media)로 화면 너비 조건에 따라 다른 CSS를 적용해 데스크톱·태블릿·모바일 반응형 레이아웃을 구현한다.
### 예제 파일별 추가 메모
- 이동 전환 예제: `width` 나 `background-color` 변화뿐 아니라, `top` / `left` 값 변화로 요소가 실제로 이동하는 전환 확인용. 요소 이동 전 `position` 지정 필요성 확인용.
- 사이트 메뉴 예제: hover 시 서브 메뉴가 열리거나 강조선이 움직이는 등 "상호작용이 있는 메뉴"를 만드는 흐름 확인용. transition 의 부드러운 시간 변화 역할과 transform 의 이동·확대 시각 효과 역할 구분용.
- 3D 변형 응용: 2D 회전과 달리 원근감 때문에 앞뒤가 뒤집혀 보이는 느낌 확인용. "부모에 원근감 부여 + 자식에 회전 적용" 구조 확인용.
- 기본 애니메이션 / 애니메이션 응용: 기본 `@keyframes` 문법 이후, 반복 재생과 다단계 변화의 실제 화면 연결 흐름 정리. 같은 animation 이라도 duration, direction, timing 값에 따른 체감 차이 비교용.
- 슬라이더 응용 1 / 슬라이더 응용 2: 슬라이더 기초 원리를 바탕으로 배치 방식과 강조 방식 확장 비교용. 단순 이동보다 "어떤 아이템을 더 눈에 띄게 만들 것인가"까지 포함한 확인용.
- 데스크톱 / 태블릿 / 모바일 스타일 분리: 같은 선택자라도 화면 폭에 따라 다른 값을 덮어쓰는 구조 연습용. 반응형에서 문서 구조 유지 + CSS 분기 감각 정리용.

### Transition (전환)
- 속성 값이 바뀔 때 **일정 시간에 걸쳐** 부드럽게 변하게 함.

| 속성 | 설명 |
|------|------|
| `transition-property` | 전환할 속성 (all 또는 width, background-color 등) |
| `transition-duration` | 지속 시간 (예: `.5s`, `1s`) |
| `transition-timing-function` | 가속도: `ease`, `linear`, `ease-in`, `ease-out` |
| `transition-delay` | 시작 지연 시간 |
```css
div {
    transition: 1s;                    /* duration만 */
    transition: .5s ease;              /* duration + timing */
    transition: width .5s, background .3s;
}
div:hover { width: 500px; background-color: aqua; }
```
### Transform (변형)
- 요소의 **위치·크기·회전·기울기**를 변경. 레이아웃 흐름에는 영향 없음.

| 함수 | 설명 |
|------|------|
| `translate(x, y)` / `translateX`, `translateY` | 이동 |
| `scale(x, y)` | 크기 비율 (2 = 2배) |
| `rotate(deg)` | 2D 회전 (단위: deg) |
| `skew(x, y)` | 기울이기 |
| `perspective(npx)` | 3D 원근 (부모에 지정) |
| `rotateX`, `rotateY`, `rotateZ` | 3D 회전 |
```css
.son:hover {
    transform: translate(400px, 400px) scale(2) rotate(360deg);
}
/* 3D */
.container:hover > .item {
    transform: perspective(100px) rotateY(180deg);
}
```
### Animation (@keyframes)
- **키프레임**으로 구간별 스타일을 정의하고, 요소에 `animation-*` 속성으로 적용.

| 속성 | 설명 |
|------|------|
| `animation-name` | @keyframes 이름 |
| `animation-duration` | 한 사이클 시간 |
| `animation-iteration-count` | 반복 횟수 (`infinite` 가능) |
| `animation-direction` | `normal` / `alternate`(왕복) |
| `animation-timing-function` | `linear`, `ease-in`, `ease-out` 등 |
| `animation-play-state` | `running` / `paused` (hover에서 정지 등) |
```css
@keyframes moving {
    from { margin-left: 100%; }
    to   { margin-left: 0%; }
}
div {
    animation-name: moving;
    animation-duration: 2s;
    animation-iteration-count: infinite;
    animation-direction: alternate;
}
```
### 슬라이더 예제
  - `overflow: hidden` + 넓은 wrapper + `@keyframes`로 좌우 이동
  - `animation-play-state: paused` on hover 로 멈춤
  - 자식 아이템에 `transform: scale(1.5)` 등으로 강조 가능
### 미디어 쿼리 (Media Query, 07MQ)
- 화면 너비 등 조건에 따라 **다른 CSS**를 적용. 반응형 레이아웃의 기본.
- **방법 1: 같은 파일 안에서 @media**
```css
/* 기본: 데스크톱 */
div { height: 150px; background-color: orange; }
@media all and (max-width: 1024px) {
    div { background-color: green; height: 100px; }
}
@media all and (max-width: 400px) {
    div { background-color: royalblue; height: 50px; }
}
```
- **방법 2: link 태그에 media 속성**
```html
<link rel="stylesheet" href="./css/desktop.css">
<link rel="stylesheet" href="./css/tablet.css" media="all and (max-width:1024px)">
<link rel="stylesheet" href="./css/mobile.css" media="all and (max-width:400px)">
```
  - 한 파일 안에 `<style>` 내부에 `@media` 작성
  - desktop / tablet / mobile CSS 파일 분리 후 `link`의 `media`로 로드
- 17. **Transition** : transition-duration, timing-function
- 18. **Transform** : translate, scale, rotate, skew, 3D perspective/rotateY
- 19. **Animation** : @keyframes, animation-name/duration/iteration-count/direction, play-state
- 20. **미디어 쿼리** : @media (max-width), link media
