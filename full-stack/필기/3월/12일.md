# 3월 12일 필기 — screen-implement (화면 구현)

## 핵심요약

sticky는 스크롤 전에는 일반 흐름에 있다가 지정한 top 값에 도달하면 그 위치에 고정되는 포지션이다. Flexbox는 한 방향(행/열) 기준의 1차원 레이아웃으로, justify-content·align-items·gap·wrap으로 배치를 제어하고 자식의 flex-grow·shrink·basis로 크기를 세밀하게 조정한다. Grid는 행과 열을 동시에 다루는 2차원 레이아웃으로, grid-template-areas를 이용해 header·sidebar·main·footer 같은 영역을 이름으로 배치할 수 있다. KREAM 스타일 실전 레이아웃 실습으로 실제 서비스 화면 구조를 구현해 보았다.



### 01BASIC — 표현·단위·타이포·색

- block / inline / inline-block: 요소 배치 방식과 `display` 변화 확인용.
- 스타일 우선순위: 여러 규칙이 겹칠 때 최종 적용 값 확인용.
- width, height, min/max: 박스 크기 지정과 화면 깨짐 방지 방식 정리.
- 색상 표현: 키워드, RGB, RGBA, HEX 같은 색상 표현 방식 정리.
- px, %, vw, vh 등: 고정 크기와 가변 크기 차이 정리.
- 배경 속성: 배경색, 배경이미지, 반복, 크기 조절 방식 정리.
- 텍스트 스타일: 정렬, 줄 높이, 글자 간격 같은 문장 배치 방식 정리.
- font-size, em, rem: 글자 크기 지정 방식과 상대 단위 기준 정리.
- font-family, @font-face: 폰트 지정과 사용자 폰트 불러오기 흐름 정리.
- 공통 스타일: 반복 적용용 기본 규칙 묶기 방식 정리.

### 02BOX — margin / padding / border / overflow

- margin, 가운데 정렬: 요소 바깥 여백과 `margin: 0 auto` 정렬 원리 정리.
- padding: 내용과 테두리 간격 조절 방식 정리.
- border, border-radius: 선의 두께, 색상, 모양, 모서리 둥글기 설정 방식 정리.
- content-box / border-box: width 계산 기준 비교용.
- overflow: 내용이 넘칠 때 처리 방식 비교용.

### 03SELECTOR — 선택자

- 전체/요소/id/class/그룹 선택자: 요소 선택 기준 정리.
- 자식(>), 자손(공백) 선택자: 바로 아래 요소와 하위 전체 요소 구분용.
- 자식 선택자 레이아웃 예제: 구조 변화에 따른 선택 결과 차이 확인용.
- 동위 선택자 (+, ~): 형제 선택 범위 차이 정리.
- 속성 선택자, :checked + label: 입력 상태에 따라 인접 라벨까지 함께 스타일링하는 흐름 정리.
- 체크박스 커스텀 스타일: 기본 체크박스를 원하는 UI처럼 보이게 만드는 방식 정리.
- 의사 클래스 (:hover, :active, :visited): 사용자 반응과 상태 변화별 화면 변화 정리.
- 의사 요소 (::before, ::after): 실제 태그 추가 없이 꾸밈 요소를 붙이는 방식 정리.
- :nth-child(2n), :nth-child(2n+1): 특정 순번 요소 선택 방식 정리.

### 04POSITION — 좌표·GNB·sticky

- position: relative: 원래 위치 기준 이동 방식 정리.
- position: absolute: 부모 또는 화면 기준 좌표 배치 방식 정리.
- GNB 기본 구조: 메인 메뉴와 서브 메뉴를 가진 네비게이션 뼈대 정리.
- GNB 단계별 예제: hover, 서브 메뉴 배치, 표시/숨김 흐름 확장 과정 정리.
- fixed GNB 레이아웃: 스크롤과 무관한 상단 고정 메뉴 구성 방식 정리.
- position: sticky 예제: 특정 지점부터 고정되는 동작 확인용.
- position 종합: static, relative, absolute, fixed, sticky 기준점 비교 정리.
- GNB 연습 예제: position 규칙을 메뉴 구조에 직접 적용하는 단계.
- GNB 연습: 서브 메뉴 방향과 배치를 조정해 완성형 메뉴로 다듬는 단계.

### 05LAYOUT — Flex / Grid / 실전

- Flex 기초 (direction, justify, align, gap, wrap): 한 방향 레이아웃의 정렬과 간격 제어 기본 정리.
- flex-grow, flex-shrink, flex-basis, align-self: 자식 요소 크기와 개별 정렬 세부 제어 속성 정리.
- Flex 연습: 여러 박스를 원하는 방향과 간격으로 배치하는 문제 풀이용.
- Flex 연습 정답: 같은 배치를 더 간단한 속성 조합으로 만드는 방법 비교용.
- Grid (template-columns/rows, areas): 행과 열을 동시에 다루는 2차원 레이아웃 방식 정리.
- KREAM 스타일 실전 레이아웃: 실제 서비스 화면 구조를 따라 만드는 실전 연습용.

### 기타 (CSS 루트 실습)

- 기타 연습용: 앞서 배운 속성을 자유롭게 조합해 보는 보충 실습용.

## Flexbox (플렉스 레이아웃)
- 단일 행/열 기준 배치.

| 속성 (컨테이너) | 설명 |
|-----------------|------|
| `display: flex` | Flex 컨테이너로 지정 |
| `flex-direction` | `row` / `column` / `row-reverse` / `column-reverse` |
| `justify-content` | 주축 정렬: `start`, `end`, `center`, `space-between`, `space-around`, `space-evenly` |
| `align-items` | 교차축 정렬: `start`, `end`, `center`, `stretch` |
| `flex-wrap` | `nowrap`(한 줄) / `wrap`(줄바꿈) |
| `gap` | 자식 간 간격 |

| 속성 (자식) | 설명 |
|-------------|------|
| `flex-grow` | 남는 공간 비율로 확장 |
| `flex-shrink` | 부족 시 축소 비율 |
| `flex-basis` | 기본 크기 (예: `200px`) |
| `align-self` | 해당 자식만 교차축 정렬 (`flex-start`, `flex-end`, `center`, `stretch`) |

## Grid (그리드 레이아웃)
- 행·열 기준 배치.
```css
.grid-container {
    display: grid;
    grid-template-columns: 1fr 2fr 1fr;   /* repeat(3, 1fr) 가능 */
    grid-template-rows: 1fr 1fr 1fr;
    gap: 10px;
}
/* 영역 이름으로 배치 */
.grid-areas {
    display: grid;
    grid-template-areas:
        'header header header'
        'sidebar main main'
        'footer footer footer';
    gap: 10px;
}
.header { grid-area: header; }
.sidebar { grid-area: sidebar; }
.main { grid-area: main; }
.footer { grid-area: footer; }
```
