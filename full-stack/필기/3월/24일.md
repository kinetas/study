# 3월 24일 필기 — screen-implement (화면 구현)

## 핵심요약

scroll 이벤트에서 window.scrollY로 스크롤 위치를 감지해 nav를 고정 상태로 전환하고, resize 이벤트에서 outerWidth 구간별로 클래스를 교체하는 반응형 처리를 구현한다. DOMContentLoaded는 HTML 파싱 완료 시점, load는 모든 리소스 로드 완료 시점에 발생하므로 초기 레이아웃 설정은 DOMContentLoaded 안에서 처리한다.


### 표 동적 생성·셀 선택
  - `selectedStyle` / `deSelectedStyle` : 현재 선택된 셀과 이전 셀의 상태를 눈에 보이게 구분하는 스타일 함수.
  - `makemap` : 입력한 행·열 수만큼 `<tr>` / `<td>`를 만들어 표 구조를 구성하는 생성 로직.
  - `data-row`, `data-col` : 각 셀이 표에서 몇 번째 행·열인지 기억하는 좌표값. 클릭 후 같은 위치를 다시 찾거나 비교할 때 사용하는 값.
  - `querySelector` : 저장해 둔 좌표값과 일치하는 셀을 다시 찾아 현재 선택 대상으로 바꿀 때 사용하는 도구.
  - `maxRow` / `maxCol` : 화살표 키로 이동할 때 표 바깥으로 넘어가지 않도록 막는 경계값.
### 스크롤 이벤트
  - `document`에 `scroll` 리스너. `window.scrollY`(또는 `document.documentElement.scrollTop`)로 세로 스크롤 위치 확인.
  - `.wrapper>header>nav`에 `scrollY >= 50`이면 `nav-fixed`(상단 고정·전체 너비) 클래스 토글, 헤더 레이아웃 전환 예제.
### 리사이즈 이벤트
  - `window`에 `resize` 리스너. `outerWidth` 구간에 따라 `.ball`에 `ball-1280px` / `ball-860px` / `ball-380px` 클래스 교체(원 크기·배경색 반응형).
### DOM 로드 이벤트
  - **`DOMContentLoaded`** : HTML 파싱 완료 후 DOM 준비 시 1회 (CSS·이미지 등 리소스 완료와 무관).
  - **`load`** : 문서·하위 리소스까지 모두 로드 후 1회.
  - 예제: `DOMContentLoaded` 안에서 `innerWidth`로 초기 `ball-*` 클래스 설정, 리사이즈와 동일한 브레이크포인트 로직.

- 14. **이벤트** : 마우스·드래그, 키보드(`keyup`/`keydown`/`keypress`), 입력·IME(`input`, `composition*`), 타자 연습, 체크박스(`change`), 표·선택, 스크롤, 리사이즈, 로드
- 17. **예제 파일** : JAVASCRIPT 폴더 내 `01Basic`, `02Type`, `03Operator`, `04FlowControl`, `05Function`, `06EVENT` 참고
