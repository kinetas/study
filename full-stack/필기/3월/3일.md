# 3월 3일 필기 — 화면설계 (screen-design)

## 핵심요약

화면설계가 무엇을 만들지 정하는 단계라면, 화면 구현은 설계 문서를 실제 HTML·CSS·JavaScript로 옮기는 단계다. HTML은 구조, CSS는 표현, JavaScript는 동작을 담당하며 세 역할을 분리하여 개발한다. 구현 초반에는 색상보다 먼저 문서 구조(제목, 메뉴, 본문, 버튼 영역)를 잡아야 이후 스타일 작업이 수월해진다. 유스케이스에서 정리한 흐름은 클릭 순서와 페이지 이동 구조로, 프로토타입의 버튼과 입력창은 태그와 스타일로 이어진다.

# 화면 구현

## 시작 개념

- 화면설계가 "무엇을 만들지" 정하는 단계라면 화면 구현은 "어떻게 보이게 만들지" 다루는 단계
- 이 과정에서 구조는 HTML, 표현은 CSS, 동작은 JavaScript가 담당
- 즉, 설계 문서의 표와 프로토타입이 실제 브라우저 화면으로 바뀌는 출발점

## 왜 먼저 구조를 잡는가

- 구현 초반에는 디자인보다도 문서 구조를 먼저 잡아야 수정이 쉬움
- 제목, 메뉴, 본문, 버튼 같은 영역을 나누어 두면 이후 스타일을 입히기 편리
- 구조 없이 색상부터 손대면 화면은 보이지만 유지보수에 어려움

## 화면설계와의 연결

- 도서 서비스, 인증 서비스, 대여 서비스처럼 나눈 기능은 결국 각 화면 메뉴와 폼으로 이어짐
- 유스케이스에서 정리한 흐름은 클릭 순서와 페이지 이동 구조로 연결
- 프로토타입에서 본 버튼과 입력창도 구현 단계에서는 태그와 스타일로 다시 표현
